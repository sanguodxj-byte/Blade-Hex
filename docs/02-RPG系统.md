# RPG系统

## 命运检定系统（内部机制，不向玩家暴露骰子细节）

> **设计原则**：所有骰子计算在引擎后台完成。玩家在UI上只看到：
> - 命中概率百分比
> - 结果反馈（命中/未命中/暴击/失误）
> - 伤害数值
>
> 骰子公式仅供内部设计参考，不在UI中显示"d20"、"1d8"等术语。

## 优势/劣势
- **优势**：掷两次取较高（UI表现：命中率提升约25%）
- **劣势**：掷两次取较低（UI表现：命中率降低约25%）
- 优势劣势互相抵消

## 核心检定类型（内部计算）

### 命中检定
```
命中检定 = 随机数(1-20) + 属性修正(STR或DEX) + 专精加值
vs 目标AC

灵巧武器(finesse)且DEX修正>STR修正时，使用DEX修正代替STR。
远程武器始终使用DEX修正。
```

### 伤害计算
```
基础伤害 = 武器骰(XdY) + 等级额外骰(Nd20) + STR修正 + 技能盘加成

等级额外骰数 = min(6, 1 + floor((等级-1)/20))
  1-20级 → +0d20, 21-40级 → +1d20, ..., 101-120级 → +5d20
```

> 注：等级额外骰为代码实现中的 Nd20 缩放机制，确保高等级角色伤害显著增长。

### 豁免检定
```
豁免检定 = 随机数(1-20) + 属性修正 + 专精加值(若精通)
vs 法术DC = 8 + 施法属性修正 + 专精加值
```

## 生命值(HP)系统
```
最大HP = baseMaxHp + 技能盘HP加值

其中 baseMaxHp 由单位生成器计算（包含CON修正），代码不再重复乘CON。
```

- 所有角色基础HP为10，CON修正由生成器预先计入 `baseMaxHp`
- 技能盘上的HP小节点提供额外HP（`CharacterSkillTree.get_hp_bonus()`）
- 最低HP为1

## 护甲等级(AC)系统

### AC计算（闪避值，不含护甲硬度）

```
闪避AC = 10 + floor(sqrt(DEX / 2)) + 盾牌AC加成 + 技能盘AC加成

有效AC = 闪避AC + 被动技能AC加成 + 防御模式(+2) + 士气修正 - Keystone惩罚
```

> **护甲不参与AC计算。** 护甲的 `AcBonus` 字段仅用于UI描述和旧版代码，战斗中不使用。
> 护甲对防御的唯一影响是：通过 `max_dex_bonus` 限制DEX对AC的贡献上限。

DEX贡献 = `floor(sqrt(DEX / 2))`：
- DEX 10 → +2, DEX 18 → +3, DEX 30 → +3, DEX 40 → +4

DEX上限（由护甲的 `max_dex_bonus` 控制）：
- 轻甲：无限制
- 中甲：上限1-2
- 重甲：上限0（完全无法闪避，纯靠装甲）

盾牌AC加成是**唯一从装备直接加入AC的来源**——盾牌是主动格挡，属于闪避行为。

护甲提供的防御通过**装甲耐久(DR)**系统实现，详见 06-装备与物品.md。

### 属性倾向与技能盘
角色没有固定职业。属性分布自然决定角色的**倾向方向**：
- 高STR → 近战伤害高 → 技能盘上近战节点效果更强
- 高DEX → 远程命中高/AC高 → 射击和闪避分支更有效
- 高CON → HP多 → 防御分支收益更大
- 高INT → 法术DC高 → 法术分支伤害/效果更强
- 高WIS → 治疗量高/侦察强 → 辅助分支更有效
- 高CHA → 士气高/招募便宜 → 指挥分支效果更好

详见 05-角色与技能系统.md

## 等级与成长（120级系统）

### 核心参数
- **等级上限**：120
- **初始属性点**：25点（1级时分配至6项属性）
- **每级属性增长**：+1点/级（可分配至任意属性）

### 暴击阈值（WIS影响攻击）
```
暴击阈值 = max(15, 20 - floor(sqrt(WIS / 2)))
```
- 默认（WIS 10）：20（仅自然20暴击）
- WIS 18：20 - 3 = 17 → 17-20 均可暴击
- WIS 32：20 - 4 = 16 → 16-20 可暴击
- WIS 50：20 - 5 = 15（下限15）

### 受到暴击伤害减免（WIS影响防御）
```
被暴击伤害倍率 = max(0.2, 1.0 - floor(sqrt(WIS / 2)) × 0.1)
```
- WIS 10 → 1.0（全额暴击伤害）
- WIS 18 → 0.7（暴击伤害减30%）
- WIS 32 → 0.6
- WIS 50 → 0.5（暴击伤害减半）
- WIS 100 → 0.3

> WIS是双向属性：高WIS既扩大自身暴击范围，又降低被暴击伤害。

### 擦伤机制（Graze）
当攻击未命中但差距≤2时，触发擦伤：
```
if total_attack < target_ac and (target_ac - total_attack) <= 2:
    擦伤命中，伤害 = max(1, 基础伤害 / 2)
```
- 擦伤不触发暴击
- 擦伤仍可被装甲DR减免

### 伤势惩罚
根据当前HP百分比施加检定惩罚：

| HP百分比 | 状态 | 全检定修正 |
|----------|------|-----------|
| ≥50% | 健康 | 0 |
| 25%-49% | 轻伤 | -1 |
| 1%-24% | 重伤 | -2 |
| 0% | 濒死 | 0（濒死状态） |
