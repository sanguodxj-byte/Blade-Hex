# RPG系统

## 命运检定系统（内部机制，不向玩家暴露骰子细节）

> **设计原则**：所有骰子计算在引擎后台完成。玩家在UI上只看到：
> - 命中概率百分比
> - 结果反馈（命中/未命中/暴击/失误）
> - 伤害数值
>
> 骰子公式仅供内部设计参考，不在UI中显示"d20"、"1d8"等术语。

## 核心属性（6项）

| 属性 | 代号 | 检定用途 | 豁免 |
|------|------|----------|------|
| 力量 | STR | 近战攻击/伤害/攀爬/破门 | 强韧豁免 |
| 敏捷 | DEX | 远程攻击/AC/先攻/潜行 | 反射豁免 |
| 体质 | CON | HP/耐力/抵抗毒素 | 强韧豁免 |
| 智力 | INT | 法术强度/知识检定/法术位 | — |
| 感知 | WIS | 侦察/追踪/意志 | 意志豁免 |
| 魅力 | CHA | 士气/交涉/领导 | 意志豁免 |

平方根属性修正公式为：
$$\text{Modifier} = \lfloor\sqrt{\text{属性值} / 2}\rfloor$$

对应表格如下：

| 属性值范围 | 修正值 | 属性值范围 | 修正值 |
|------------|--------|------------|--------|
| 1          | +0     | 18-31      | +3     |
| 2-7        | +1     | 32-49      | +4     |
| 8-17       | +2     | 50-71      | +5     |

## 检定规则（内部计算）

### 内部公式（引擎用）
```
检定结果 = 随机数(1-20) + 属性修正 + 其他加值
```

### 难度等级(DC)
| 难度 | DC | 玩家感知 |
|------|-----|----------|
| 极易 | 5 | "轻而易举" |
| 简单 | 10 | "应该没问题" |
| 中等 | 15 | "有把握" |
| 困难 | 20 | "有些冒险" |
| 极难 | 25 | "希望渺茫" |
| 传奇 | 30 | "近乎不可能" |

## 优势/劣势
- **优势**：掷两次取较高（UI表现：命中率提升约25%）
- **劣势**：掷两次取较低（UI表现：命中率降低约25%）
- 优势劣势互相抵消

## 核心检定类型（内部计算）

### 命中检定
```
命中检定 = 随机数(1-20) + 属性修正(STR或DEX)
vs 目标AC

灵巧武器(finesse)且DEX修正>STR修正时，使用DEX修正代替STR。
远程武器始终使用DEX修正。
```

### 伤害计算
```
物理伤害 = 武器骰(XdY) * (1.0 + STR修正% + 武器精通%) + Buff加成
```
其中：
- **STR加成**：`floor(sqrt(STR)) × 10%`
- **武器精通加成**：精通等级 × 10%
- 精灵和半兽人等种族特性会额外乘以相应的伤害倍率（如精灵 x1.1，半兽人 x1.2）。
- 武器伤害骰由武器面板（含 Tier 缩放）决定，不加等级额外骰。

### 豁免检定
```
豁免检定 = 随机数(1-20) + 属性修正
vs 法术DC = 8 + 施法属性修正
```

## 生命值(HP)系统
```
最大HP = 基础生命(10) + CON生命加成 × 等级 + 装备HP + 饰品HP + 技能盘加成
```
其中：
- 基础生命值固定为 10。
- CON生命加成：`floor(sqrt(CON / 4))`，即 CON 的增长会在每次升级时为角色提供更高的生命增量。
- 矮人具有独特的种族特性“韧性”，使最大 HP 最终获得 20% 的增幅。
- 最低HP为 1。

## 护甲等级(AC)系统

### AC计算（闪避与装甲物理偏折）

```
闪避AC = 基础AC(8) + DEX修正(受限) + 护甲偏折AC + 盾牌偏折AC + 护甲AcBonus + 盾牌AcBonus + 技能盘/饰品 + Buff

有效AC = 闪避AC + 被动技能AC加成 + 防御模式(+2) + 士气修正
```

> 为了模拟坚硬防具偏折武器的效果，装甲本身的物理厚度（DR阈值）会折算为偏折 AC：`floor(sqrt(ArmorDR))`。
> 护甲和盾牌的 `AcBonus`（装备自带 AC 词缀，含随机词缀加成）在战斗中也全额参与加算，以与 UI 描述及实际数值对齐。

DEX贡献 = `floor(sqrt(DEX / 2))`：
- DEX 10 → +2, DEX 18 → +3, DEX 30 → +3, DEX 40 → +4

DEX上限（由护甲的 `max_dex_bonus` 控制）：
- 轻甲：无限制
- 中甲：上限1-2
- 重甲：上限0（完全无法闪避，纯靠装甲）

盾牌AC加成是**唯一从装备直接加入AC的来源**——盾牌是主动格挡，属于闪避行为。

护甲提供的防御通过**装甲耐久(DR)**系统实现，详见 06-装备与物品.md。

## 行动点(AP)系统
```
最大AP = 基础AP(12) + DEX修正 + floor(CON修正 / 2) - 护甲与盾牌AP惩罚
```
其中：
- **基础 AP**：固定为 12。
- **属性修正**：DEX和CON修正均根据平方根修正公式 $\lfloor\sqrt{\text{属性值}/2}\rfloor$ 计算得出。
- **AP 惩罚**：穿戴重型护甲或盾牌会根据装备配置扣减一定的 AP 容积（头盔不计入惩罚）。

> 1级角色（属性值在 10 左右，DEX修正 +2，CON修正 +2，无防具惩罚时）的最大 AP 通常为 15 点。
> 中后期角色（属性值在 30 左右，DEX修正 +3，CON修正 +3，无防具惩罚时）的最大 AP 通常为 16 点。

### 属性倾向与技能盘
角色没有固定职业。属性分布自然决定角色的**倾向方向**：
- 高STR → 近战伤害高 → 技能盘上近战节点效果更强
- 高DEX → 远程命中高/AC高 → 射击和闪避分支更有效
- 高CON → HP多 → 防御分支收益更大
- 高INT → 法术DC高 → 法术分支伤害/效果更强
- 高WIS → 治疗量高/侦察强 → 辅助分支更有效
- 高CHA → 士气高/招募便宜 → 指挥分支效果更好

详见 05-角色与技能系统.md

## 先攻系统（个体先攻制）

改用**个体先攻队列**（`InitiativeQueue`），不再采用阵营分组制。

```
先攻检定 = 随机数(1-20) + DEX修正 + 基础先攻(BaseInitiative) + 技能盘先攻加成 + 饰品先攻加成
```

- 每轮开始时为所有存活单位掷先攻
- 按先攻值**降序**逐个行动
- 一个单位行动完毕后推进到下一个
- 所有单位行动完一轮后自动开始新一轮
- 实现见 `TurnManager` / `InitiativeQueue`

## 等级与成长（120级系统）

### 核心参数
- **等级上限**：120
- **初始属性点**：30点（1级时分配至6项属性，标准模板为全属性5点，根据种族/模板上下浮动）
- **单项属性下限**：3
- **每级属性增长**：+1点/级（可分配至任意属性）
- **每次升级获得** **1个技能点**（用于技能盘）
- **每5级获得1次** **跳跃机会**（技能盘上可跳过相邻规则）
- 法术位通过技能盘节点获取（非自动获得）

### 暴击阈值（WIS影响攻击）
```
暴击阈值 = max(15, 20 - floor(sqrt(WIS / 2)))
```
- 默认（WIS 10）：20（仅自然20暴击）
- WIS 18：20 - 3 = 17 → 17-20 均可暴击
- WIS 32：20 - 4 = 16 → 16-20 可暴击
- WIS 50：20 - 5 = 15（下限15）

### 受到暴击伤害减免（WIS影响防御）
```
被暴击伤害倍率 = max(0.2, 1.0 - floor(sqrt(WIS / 2)) × 0.1)
```
- WIS 10 → 1.0（全额暴击伤害）
- WIS 18 → 0.7（暴击伤害减30%）
- WIS 32 → 0.6
- WIS 50 → 0.5（暴击伤害减半）
- WIS 100 → 0.3

> WIS是双向属性：高WIS既扩大自身暴击范围，又降低被暴击伤害。

### 擦伤机制（Graze）
当攻击未命中但差距≤2时，触发擦伤：
```
if total_attack < target_ac and (target_ac - total_attack) <= 2:
    擦伤命中，伤害 = max(1, 基础伤害 / 2)
```
- 擦伤不触发暴击
- 擦伤仍可被装甲DR减免

### 伤势惩罚
根据当前HP百分比施加检定惩罚：

| HP百分比 | 状态 | 全检定修正 |
|----------|------|-----------|
| ≥50% | 健康 | 0 |
| 25%-49% | 轻伤 | -1 |
| 1%-24% | 重伤 | -2 |
| 0% | 濒死 | 0（濒死状态） |
