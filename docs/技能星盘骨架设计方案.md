# 技能星盘骨架设计方案（已落地版）

> 目标：把技能星盘从"程序随意生成布局"改为"规范的、数据驱动的固定骨架 + 内容填空"。
> 本文记录已落地的数据驱动骨架；几何密铺与配额以 `技能星盘节点重排方案.md` 为权威，节点语义以 `技能星盘节点内容设计.md` 为权威。

---

## 一、已确认的设计决策（来自评审）

| 维度 | 决策 | 理由 |
|------|------|------|
| 布局来源 | **外部配置文件**驱动，彻底替换 `AutoLayoutAndConnect` | 现状程序乱生成、不可控 |
| 节点位置表达 | **显式存每节点占用的所有瓦片坐标** | 镶嵌宝石需精确密铺，BFS 自动展开做不到 |
| 连通机制 | **统一为几何瓦片共边**，废弃 `Neighbors` 列表 | 单一真相源，布局即连通 |
| 美学原则 | **镶嵌宝石（单元自美）** | 玩家跨扇区零散点亮，整体造型路线（像素画）会"点不满即残缺"，与"鼓励多属性"冲突 |
| 连通结构 | **六个等边三角扇区从中心放射密铺**；不再使用 `trans_*` 过渡桥 | 多属性横跨依靠中心连通与几何共边 |
| 半径/密度 | 固定半径 `R=20`；六扇区各 399 内容瓦片 + 中心 6 瓦片 | 与重排方案一致 |
| 大节点成本 | 主动 4 瓦片；被动 3 瓦片；keystone 6；巨型 12 | 与节点内容设计一致 |

### 关键设计洞察（否决像素画的原因）
游戏**鼓励多属性/复合职业** → 玩家跨多个扇区各点一部分 → 若每扇区是一幅完整造型（剑/弓/法杖），多属性玩家看到的是六个残缺半成品。像素画只奖励"专精一路"，与机制矛盾。
**镶嵌宝石**则相反：美感来自每个节点自身是一块完整宝石，点亮任意子集都是"一片镶嵌的宝石"，无论跨几个扇区都自洽。

## 二、几何模型

### 2.1 三角瓦片坐标系（沿用现有 `SkillTreeCoord`）
- 每个 hex 格点 `(q,r)` 关联 2 个三角瓦片：`t=0` ▽、`t=1` ▲
- 瓦片编码 `EncodeTile(q,r,t) = Vector2I(q*2+t, r)`
- 共边邻接：每个三角瓦片恰好 3 个共边邻居（`GetTileNeighbors`）
- C# 端与 `tools/skill_editor.html` 坐标系**已完全一致**

### 2.2 宝石单元（节点形状）
每个节点 = 一组**连续共边**的三角瓦片，构成一块"宝石"：

| 节点类型 | 瓦片数 | 形状 | 几何 |
|---------|-------|------|------|
| pip | 1 | 单三角 | 只作收缝与微属性 |
| 小节点 | 2 | 菱形 | 1 个 ▽ + 1 个相邻 ▲ 拼成菱形 |
| 主动节点 | 4 | active_kite | 4 个连续三角组成紧凑宝石 |
| 被动节点 | 3 | passive_triad | 3 个连续三角组成紧凑三连 |
| Keystone | 6 | 六角冠/王冠 | 6 个三角组成紧凑冠形 |
| 巨型 | 12 | 六种对称几何模板 | 每扇区一种：sunburst / arrowhead / bastion / crystal / hourglass / crown |
| start | 6 | 中心六边形 | 同 Keystone 形态，居原点 |

**密铺约束**：相邻宝石必须**恰好共享三角边、互不重叠**。这要求配置里每个节点显式列出其占用瓦片，且全局保证：
1. 无两节点占用同一瓦片（不重叠）
2. 想连通的两节点，至少有一对瓦片共边（连通）

## 三、骨架结构（六扇区放射密铺）

```
                    INT区
          WIS区  ╲   │   ╱  CON区
                  ╲  │  ╱
                   ╲ │ ╱
        ───────────start───────────
                   ╱ │ ╲
                  ╱  │  ╲
          CHA区  ╱   │   ╲  STR区
                    DEX区
```

- **start** 居中，占 6 个中心三角瓦片，六个属性扇区各共享一瓣入口。
- **6 扇区**：每个属性扇区是由中心到六边形相邻顶点连线形成的等边三角，边长 20。
- **内容区**：每扇区 `r2-r20` 全部 399 瓦片密铺，不留空格。
- **无过渡桥**：最终重排方案不再生成 `trans_*` 节点；跨属性路线通过中心 start 和扇区间共边密铺自然连通。

### 每扇区固定槽位
| 类型 | 数量 | 瓦片/节点 | 小计瓦片 |
|----|----:|----:|----:|
| 巨型 | 1 | 12 | 12 |
| keystone | 3 | 6 | 18 |
| 主动 | 6 | 4 | 24 |
| 被动 | 8 | 3 | 24 |
| 小节点 | 156 | 2 | 312 |
| pip | 9 | 1 | 9 |
| **合计** | **183** |  | **399** |

整盘为 `6 * 183 + start = 1099` 个节点，`6 * 399 + start 6 = 2400` 个三角瓦片。

## 四、配置文件格式

载体已拆成两份 JSON：
- `BladeHexCore/src/SkillTree/skill_tree_layout.json`：几何骨架，只含 `id / region / type / depth / tiles`。
- `BladeHexCore/src/SkillTree/skill_tree_content.json`：节点内容，只含 `id / name / effect / statBonuses / figureTemplate / contentMode` 等内容字段。

### 4.1 布局格式
```json
{
  "version": 1,
  "hexRadius": 20,
  "nodes": [
    {
      "id": "str_s01",
      "region": "str",
      "type": "small",
      "depth": 1,
      "tiles": [[2,0,0],[2,0,1]]
    }
  ]
}
```

**连通性不在 JSON 里显式写** —— 由加载后对 `tiles` 做几何共边计算自动得出（呼应"统一几何共边"）。

### 4.2 内容格式
```json
{
  "version": 1,
  "randomPools": {
    "str": [{ "stat": "melee_damage", "min": 1, "max": 2, "weight": 30 }]
  },
  "nodes": [
    {
      "id": "str_s01",
      "name": "属性星纹",
      "description": "按角色种子从本扇区风格池生成纯属性。",
      "contentMode": "random_attribute",
      "figureTemplate": "attribute_pair_2"
    }
  ]
}
```

## 五、已落地实现点

1. **SkillNodeData**：新增 `ExplicitTiles: Vector2I[]` 字段；`GetTiles` 优先返回它。
2. **SkillNodeShape.GetTiles**：`if (node.ExplicitTiles?.Length > 0) return node.ExplicitTiles;` 否则回退现有模板展开。
3. **SkillTreeLayoutLoader**：从 layout/content 两份 JSON 读节点，重复 ID 直接报错。
4. **SkillTreeData 构造**：加载固定 JSON 布局；旧 `ConstellationBuilder` 已移除。
5. **连通**：`NodeFiller` 基于几何共边，布局即连通；`Neighbors` 不再作为真相源。
6. **瓦片成本**：点亮成本等于显式瓦片数；start 成本为 0。
7. **前端**：`SkillTreeUI` 读取 `GridPosition` + `SkillNodeShape.GetTiles()`，支持显式多瓦片节点、扇区三角填色和节点边界命中。

### 校验覆盖
- `SkillTreeLayoutTests` 覆盖节点内连通、全局不重叠、从 start 可达、R=20 界内、layout/content ID 唯一且完整、每扇区 399 内容瓦片、配额、形状模板、随机属性合法键。
- `SkillTreeLayoutLoader.Validate` 覆盖运行时加载后的基本几何安全。

## 六、后续风险

1. 巨型/部分 keystone 的规则改写效果仍依赖战斗管线钩子持续补齐。
2. UI 仍需截图级验收：确认扇区分割线、巨型模板、选中/悬停边界和文本面板在实际 Godot 画面中符合预期。
3. 若后续改生成器，必须保持 layout/content ID 完全一致、六扇区各 399 内容瓦片、无 transition 节点。
