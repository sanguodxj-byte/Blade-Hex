## Blade&Hex 战斗伤害算法 — 伪随机保底机制评估

### 一、当前随机要素盘点

Blade&Hex 的战斗系统基于 d20 体系，经过深入审阅 `CombatRuleEngine.cs`、`RPGRuleEngine.cs`、`BattleUnitModel.cs` 和 `CombatRandom.cs` 的实现，当前系统中存在以下独立随机环节：

**R1 — 攻击检定（d20）**：`RollAttack()` 中的核心命中判定。自然 20 自动暴击命中，自然 1 自动失败，差值 ≤2 触发擦伤半伤。支持优势/劣势（2d20 取高/低）。

**R2 — 伤害骰（武器骰子）**：`RollDamage()` 中按武器类型投掷若干 d6/d8/d10 等。伤害值经过多层乘算管道（`DamageCalcPipeline` 的 Base → Increased → More → FinalMult 四阶段）。

**R3 — 穿甲检定（独立 d20）**：`ApplyDamage()` 中独立的 d20 + STR 穿透加值 vs 护甲 DR 阈值。自然 20 强制穿透。

**R4 — 独立暴击概率（BonusCritChance）**：技能树 `critical_rate` 节点提供的额外暴击概率，在命中后独立掷骰判定（`CombatRandom.RandRange(0, 999)` 对照阈值）。

**R5 — 豁免检定（d20）**：`MakeSave()` 中的强韧/反射/意志豁免，用于控制减益效果的命中。

当前的随机源抽象设计良好（`IRandomSource` 接口 + `CombatRandom` 静态包装 + `SeededRandomSource` 确定性重放），这为后续引入 PRD 提供了非常干净的实施基础。


### 二、伪随机保底机制（PRD）概述

伪随机分布（Pseudo-Random Distribution）最早由 Warcraft III 推广，后被 Dota2、Fire Emblem 等广泛采用。其核心思想是：当一个概率事件（如暴击、命中）连续多次未触发时，逐步提升下一次触发的实际概率，使得"连续不中"的极端情况被压缩，同时在长期统计上仍保持与标注概率一致的平均触发率。

Dota2 的经典 PRD 公式为 `P(N) = C × N`，其中 N 是累计未触发次数，C 是一个由目标概率 p 决定的常数。当 `C × N ≥ 1` 时强制触发，然后计数器归零。

PRD 的效果是：把真实概率分布从几何分布（独立事件，长尾方差大）转变为截断递增分布（方差显著收窄），让玩家感知更加"公平"和"可预测"。


### 三、当前系统已有的方差抑制手段

在评估是否引入 PRD 之前，必须认识到 Blade&Hex 已经有多层设计在抑制极端方差：

**擦伤机制（Graze）**：miss 差值 ≤2 时转为半伤命中。这意味着在高 AC 对局中，完全空手的概率被大幅压缩。以 75% 命中率为例，加上擦伤后实际有效输出率约 85%+。这是一个非常强力的保底机制。

**优势/劣势双骰**：2d20 取高/低的机制本质上就是一种概率曲线修正。优势的等效命中率提升相当于 `1-(1-p)²`，已经大幅降低了连续低骰的可能。

**最低伤害保证**：整个伤害管道中到处可见 `Math.Max(1, ...)` 的下限保护，确保任何命中攻击至少造成 1 点伤害。

**士气系统间接调节**：低士气降低命中、增加失误率，但暴击阈值下限锁定在 15。这防止了士气崩溃时的螺旋恶化。

**永久死亡 + 无复活**：这个宏观设计决策意味着每次伤害的期望值稳定比"爆发上限"更重要——玩家对稳定输出的需求高于对极端爆发的追求。


### 四、逐环节评估：哪些值得加入 PRD

#### R1 — 攻击检定：**建议有限度引入**

当前攻击检定的擦伤机制已经处理了"差一点就中"的情况，但无法解决"连续骰出 2~5 极低值"的问题。在一个 6 人小队永久死亡的游戏中，连续 3 次 miss（即使概率只有约 1.5%~3.5%）可能直接导致一个关键角色阵亡，玩家体验会极其挫败。

然而，引入经典 PRD 需要谨慎，原因有三：

第一，d20 体系的命中判定不是简单的"命中/未命中"二元结果——还有暴击、大失败、擦伤三个特殊结果。经典 PRD 只处理二元概率，需要扩展。

第二，攻击检定频率不像 MOBA 中的普攻那么密集。在回合制战术游戏中，一次战斗可能只有 20~40 次攻击检定，样本量小意味着方差的感知更明显，但也意味着 PRD 的"平均概率收敛"优势发挥不充分。

第三，玩家看到的命中率是百分比（代码中 `HitChancePercent`），如果标注 75% 命中率但实际 PRD 调节后短期体感偏差，可能引发"数值不透明"的不信任。

**推荐方案**：引入一个轻量级"连续 miss 保底"而非完整 PRD。具体而言，当同一单位连续 2 次普通攻击未命中（不含擦伤、不含大失败）时，第 3 次攻击获得优势（2d20 取高）。这不是改变概率分布，而是在极端情况下给予一次补救。计数器在命中或战斗结束时归零。

这种方案的优势在于：实现极其简单（只需在 `AttackInput` 上增加一个 `consecutiveMisses` 字段，在 `RollAttack` 开头判断是否强制优势）；不改变标注命中率的语义（玩家看到的是 75%，单次仍然是 75%）；只在极端序列（连 miss 3 次概率约 0.02%~3%）时介入；与现有优势机制完全兼容，不引入新概念。

#### R2 — 伤害骰：**不建议引入 PRD**

武器伤害骰（如 2d6、3d8）本身已经是多骰求和，根据中心极限定理，骰子数量越多分布越趋近正态分布，极端值的概率天然很低。以 2d6 为例，最低值 2 的概率仅 2.8%，最高值 12 也是 2.8%，中间值 7 概率 16.7%。3d6 的方差更小。

伤害骰是"量大管饱"型的随机——一场战斗中同一武器可能掷骰十几次，单次低伤害会被后续正常伤害平均掉。引入 PRD 不仅收益极低，而且实现复杂（需要为每个单位每种武器维护独立的骰面计数器），还会破坏"重击感"——玩家掷出全 6 时的爽快感是战术游戏的重要情感节奏。

此外，伤害骰的方差已经被伤害管道中的多个固定加成项（STR 加成%、武器精通%、被动近战加成等）进一步稀释。在高精通等级下，掷骰部分占总伤害的比例可能只有 30~40%。

#### R3 — 穿甲检定：**建议保持现状**

穿甲检定是独立 d20 + STR 穿透加值 vs DR 阈值的二元判定。表面上看它是 PRD 的理想候选（二元结果，单一概率），但有几个特殊因素使其不适合：

穿甲失败并不意味着"无输出"——根据 `DamagePenetrationTable`，未穿透时仍然有装甲伤害（Slash 40%、Crush 10% HP 泄漏），这不是一个 0/1 的结果。引入 PRD 会让"连续穿甲失败后强制穿甲"变得可预期，可能导致重甲单位在面对高 STR 敌人时完全无法起到预期的减伤作用。

更重要的是，穿甲系统已经有自然 20 强制穿透作为保底。如果再加上 PRD，双重保底会使得重甲的 DR 机制在高 STR 对手面前过于脆弱，削弱了装甲体系的设计意义。

#### R4 — 独立暴击概率：**建议引入 PRD**

技能树 `critical_rate` 节点的独立暴击概率是纯二元独立掷骰（`CombatRandom.RandRange(0, 999) < threshold`），且概率通常较低（5%~15%）。这正是 PRD 的最佳应用场景。

低概率二元事件的几何分布特性意味着：10% 暴击率下，连续 20 次不暴击的概率仍有约 12.2%。玩家点了天赋却长时间看不到效果会产生"天赋没用"的错觉。

Dota2 的 PRD 公式可以直接应用：对于 10% 目标暴击率，C ≈ 0.01487，首次暴击实际概率约 1.5%，之后每次递增 1.5%，到第 67 次时强制暴击。长期平均暴击率仍然是 10%。

实现上，只需在 `CombatRandom` 或 `AttackInput` 中维护一个 `bonusCritCounter`，每次命中未触发 BonusCrit 时 +1，触发时归零。`CombatRuleEngine.RollAttack` 中替换当前的简单阈值比较为 PRD 公式计算。

#### R5 — 豁免检定：**不建议引入 PRD**

豁免检定是针对减益效果（中毒、冰冻等）的抵抗力判定，频率很低（一场战斗可能只有几次），而且豁免失败的结果是"被施加减益"而非直接致命。减益效果本身有持续时间作为自然限制，引入 PRD 收益不大。同时，如果怪物的豁免被 PRD 保护，玩家的控制技能会显得不稳定，反而降低体验。


### 五、实现方案建议

如果决定引入 PRD，以下是推荐的实施策略：

**架构层面**：在 `IRandomSource` 接口上增加 `bool RollPRD(float targetChance, ref int counter)` 方法。`GodotRandomSource` 和 `SeededRandomSource` 各自实现。`CombatRandom` 增加对应的静态包装。这样保持了现有的随机源抽象设计和确定性重放能力。

**数据层面**：在 `UnitRuntimeState` 中添加 PRD 计数器字段（`int BonusCritPrdCounter`、`int AttackMissPrdCounter`），确保战斗结束时重置。

**测试层面**：利用现有的 `HeadlessCombatLoop` 和 `SeededRandomSource` 进行蒙特卡洛模拟。跑 10000 场战斗，对比 PRD 前后的以下指标：平均每次战斗伤害、伤害方差、连续 miss ≥3 次的频率、BonusCrit 实际触发率与期望的偏差。

**调参层面**：PRD 常数 C 可以从查表开始（Dota2 有完整的 C-p 对照表），也可以根据游戏节奏自行调整。建议从较弱的 PRD（C 值偏小，强制触发阈值偏大）开始，通过模拟数据逐步微调。


### 六、蒙特卡洛模拟验证

为量化上述定性分析的结论，编写了 `tools/prd_simulation.py` 模拟脚本，对 5000 场战斗（每场 30 次攻击）进行蒙特卡洛模拟。以下是核心数据。

#### 6.1 攻击检定轻量保底（连续 2 miss → 第 3 次优势）

| 标注命中率 | 无保底: 平均命中 | 有保底: 平均命中 | 命中提升 | 无保底: ≥3连miss战斗占比 | 有保底: ≥3连miss战斗占比 | 降幅 |
|-----------|----------------|----------------|---------|----------------------|----------------------|-----|
| 55%       | 16.4/30        | 17.3/30        | +0.96   | 83.9%                | 58.7%                | -25.2pp |
| 70%       | 20.4/30        | 20.9/30        | +0.42   | 48.7%                | 23.5%                | -25.2pp |
| 85%       | 24.4/30        | 24.6/30        | +0.17   | 15.0%                | 5.1%                 | -9.9pp  |

数据表明：轻量保底对平均命中率的影响极小（<1 次/场），但显著压缩了极端 miss 序列。在 70% 命中率下，出现 3 次以上连续 miss 的战斗占比从 48.7% 降至 23.5%，最大连 miss 从 10 降至 8。这正是设计意图——保底不改变期望，只消除极端坏序列。

55% 命中率下最大连 miss 从 14 降至 10，说明该保底在低命中率环境下效果更显著，恰好是玩家最需要安慰的场景。

#### 6.2 独立暴击概率 PRD（Dota2 式）

| 标注暴击率 | C 值 | 无PRD实际率 | PRD实际率 | 无PRD: 整场0暴击 | PRD: 整场0暴击 | 最大间隔差 |
|-----------|------|-----------|----------|----------------|---------------|----------|
| 5%        | 0.003818 | 5.04% | 3.34% | 33.1% | 39.6% | 0 |
| 10%       | 0.014708 | 9.91% | 8.48% | 11.4% | 2.1%  | 0 |
| 15%       | 0.032378 | 15.11%| 13.72%| 3.3%  | 0.0%  | -1 |

需要注意：由于单场战斗仅有约 21 次命中机会（30 次攻击 × 70% 命中率），PRD 的强制触发机制在低概率下几乎无法在单场内触发（5% 暴击的强制触发需要约 262 次命中）。因此 PRD 在短期内的实际暴击率会略低于标注值。

但 PRD 的核心收益体现在 10% 和 15% 区间：整场 0 暴击的战斗占比从 11.4% 降至 2.1%（10% 暴击率），从 3.3% 降至 0（15% 暴击率）。这意味着玩家点了技能树暴击节点后，几乎每场战斗都能至少看到一次暴击触发，体验上远优于"点了天赋整场白板"。

5% 的极端低概率下 PRD 效果有限，这符合预期——概率太低时 PRD 的递增斜率太平缓，在短战斗中无法发挥作用。如果 5% 暴击率的体验确实需要保底，可以考虑将保底机制改为"每场战斗至少触发 1 次"的硬保底，但这已超出 PRD 的设计范畴。

#### 6.3 穿甲检定（保持现状的验证）

| 场景 | 总穿透率 | 自然20贡献 |
|------|---------|----------|
| 轻甲 (STR+3 vs DR15) | 45.1% | 5.1% |
| 板甲 (STR+5 vs DR18) | 40.2% | 4.9% |
| 高力量vs板甲 (STR+8 vs DR18) | 54.7% | 5.0% |

自然 20 强制穿透贡献约 5% 的穿透率，在高 STR 场景下（总穿透 54.7%）这一保底的边际价值较低，但在低 STR 场景下（总穿透 40.2%）它占了穿透总量的约 12%。这说明自然 20 保底已经在发挥作用，无需额外 PRD。

#### 6.4 武器伤害骰方差（不需要 PRD 的论据）

| 骰子 | 均值 | 最低值概率 | 最高值概率 |
|------|-----|----------|----------|
| 1d6  | 3.5 | 16.60%   | 16.47%   |
| 2d6  | 7.0 | 2.75%    | 2.78%    |
| 3d6  | 10.5| 0.47%    | 0.45%    |
| 2d8  | 9.0 | 1.63%    | 1.65%    |
| 3d8  | 13.5| 0.20%    | 0.19%    |

2d6 以上的多骰系统中，极端值概率已低于 3%，3d6 更是低至 0.47%。加上 STR 和精通的固定加成稀释，实际伤害方差不需要 PRD 干预。


### 七、风险评估

引入 PRD 的主要风险有以下几点：

**复杂度膨胀**：战斗系统已经相当复杂（多层乘算管道、穿甲分裂、方向倍率、士气修正等）。每增加一层机制都会增加调试和平衡的难度。建议只在最影响玩家体验的环节（R1 的轻量保底和 R4 的完整 PRD）引入。

**竞技公平性**：如果未来有多人对战模式，PRD 的计数器状态可能成为同步和作弊的隐患。需要确保 PRD 计数器在 `UnitRuntimeState` 中正确序列化，且不能被客户端篡改。

**"伪随机感"反噬**：资深玩家可能识破 PRD 模式（"我已经 miss 两次了，下次一定中"），并利用这个规律做决策。在单人游戏中这不是问题（甚至是奖励学习能力的体现），但在 PvP 中可能导致策略扭曲。

**与永久死亡的交互**：Blade&Hex 没有复活机制。PRD 保底虽然降低了"连续倒霉致死"的概率，但也降低了"对手连续倒霉被我翻盘"的戏剧性。需要权衡"公平感"和"紧张感"之间的平衡。


### 八、结论

Blade&Hex 的战斗伤害算法整体设计成熟，现有的擦伤机制、优势/劣势系统、最低伤害保证已经构成了较好的方差抑制基础。不建议全面铺开 PRD，但推荐在两个点做针对性优化：

对攻击检定（R1），引入"连续 2 次 miss 后第 3 次获得优势"的轻量保底。实现成本极低，只在极端序列时介入，不改变标注概率的语义。

对独立暴击概率（R4），引入标准 Dota2 式 PRD。这是低概率二元事件的经典解法，能有效消除"点了天赋没效果"的负面体验。

其他随机环节（伤害骰、穿甲检定、豁免检定）保持现状即可。它们的方差特性已经被现有设计充分覆盖，引入 PRD 的边际收益不足以抵消复杂度成本。
