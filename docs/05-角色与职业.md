# 角色与技能系统

## 核心理念
**没有固定职业，也没有固定单位类型。** 每个角色出生时都是白纸，只有中心节点"启程"。角色由**属性倾向 + 随机特质 + 技能盘选择 + 当前装备**四要素共同塑造。

- **技能盘**决定角色"会做什么"（能力）
- **装备**决定角色"是什么"（近战/远程/骑兵/法师...）
- 一个点了"连击"的角色拿长弓时仍然是远程单位，连击对远程无效
- 同一个角色换一套装备就是完全不同的战场角色（详见06-装备与物品.md）

技能盘上的节点**不锁定单位类型**——近战分支的节点在拿弓时仍然存在，只是不生效。玩家可以随时换装备改变角色定位。

## 角色生成

### 种族
种族提供基础修正，详见 12-种族与招募.md。

| 种族 | 属性修正 | 种族特性 |
|------|----------|----------|
| 人类 | 无 | 适应（每10级额外获取一个技能点） |
| 精灵 | 无 | 技艺（剑类和弓类命中+1，伤害修正x1.1） |
| 矮人 | 无 | 韧性（AC+1，生命值+20%） |
| 半兽人 | 无 | 狂暴（血量低于50%时伤害加20%），威胁直觉（先攻+2） |
| 半精灵 | 无 | 双重血统，社交天赋 |

### 属性生成
- 所有角色1级时使用 **3d6×6**（完全随机），再加种族修正
- 主属性在此时自然浮现——STR最高就是力量型，INT最高就是智力型
- 玩家主角可选用标准阵列(15,14,13,12,10,8)替代

### 属性倾向
根据最高两项属性，自动标注角色的"倾向标签"：

| 最高属性 | 倾向标签 | 技能盘方向 |
|----------|----------|-----------|
| STR | 力量倾向 | 近战、重甲、物理爆发 |
| DEX | 灵巧倾向 | 远程、轻甲、暴击、闪避 |
| CON | 体魄倾向 | 坦克、耐力、抗性 |
| INT | 智力倾向 | 法术、范围伤害、法术位 |
| WIS | 感知倾向 | 治疗、侦察、意志、弓术 |
| CHA | 魅力倾向 | 士气、指挥、圣法术、社交 |

倾向标签**不限制**技能盘选择，但影响AI自动加点方向和推荐路径。

### 随机特质
每个角色生成时获得 **2-4个随机特质**。特质**直接增减基础属性值**或提供功能性效果，是属性生成的第二层随机。

#### 特质设计原则
- 特质不是"好/坏"的二元判断，而是**引导角色走向不同方向**
- 一个减INT加STR的特质对法师是坏事，对战士是好事
- 玩家看到特质后决定如何利用——这才是有趣的选择
- 功能性特质（非属性类）提供独特体验但不直接决定强弱

#### 属性类特质（最常见）
每个角色2-3个属性特质，随机抽取后应用到基础属性上：

| 特质 | 效果 | 意味着 |
|------|------|--------|
| 蛮力 | STR+2 | 天生力气大，适合近战 |
| 长臂 | 近战射程+1（可攻击2格） | 适合前排控制 |
| 快手 | DEX+2 | 灵活，适合远程/闪避 |
| 猫眼 | 远程命中+1 | 神射手潜质 |
| 硬骨头 | CON+2 | 抗打，HP多 |
| 壮硕 | CON+1，速度-1 | 耐打但笨重 |
| 过人慧心 | INT+2 | 聪明，适合法术 |
| 博闻强记 | 法术位+1（每阶） | 法术容量大 |
| 敏锐直觉 | WIS+2 | 洞察力强，适合治疗/侦察 |
| 警觉 | 先攻+3，被动侦察范围+1 | 难以被偷袭 |
| 天生统帅 | CHA+2 | 领袖气质，适合指挥 |
| 亲和力 | 商店价格-15%，招募价格-10% | 省钱 |
| 蛮勇 | STR+1，CON+1，WIS-2 | 能打但鲁莽 |
| 术士血脉 | CHA+2，法术DC+1 | 天生施法者 |
| 瘦弱 | STR-1，DEX+1 | 力量不足但灵活 |
| 迟缓 | DEX-2，CON+1 | 慢但结实 |
| 脆弱 | CON-2 | 容易受伤 |
| 愚钝 | INT-2 | 不适合法术 |
| 笨拙 | DEX-1，远程命中-1 | 不适合远程 |
| 笨嘴拙舌 | CHA-2 | 社交困难 |

#### 功能性特质（较少见，0-1个）
| 特质 | 效果 | 说明 |
|------|------|------|
| 夜视 | 获得黑暗视觉 | 洞穴/夜间无惩罚 |
| 铁胃 | 免疫食物中毒 | 长途行军安全 |
| 适应力 | 疲劳惩罚减半 | 长途行军优势 |
| 厚皮 | 受到物理伤害-1 | 全局减伤 |
| 不屈 | HP归零时50%概率保持1HP（每战1次） | 绝境翻盘 |
| 以太共鸣 | 施法时恢复1d4 HP | 法术/近战双修有利 |
| 预感 | 被伏击时自动获得一轮准备 | 野外安全 |
| 旧伤 | 战斗开始时HP-10% | 永久debuff |
| 贪吃 | 补给消耗×1.5 | 养不起 |
| 胆小 | HP<50%时攻击-1 | 关键时刻掉链子 |
| 仇外 | 与外族队友在一起时忠诚度-10 | 管理成本 |

#### 特质如何塑造角色
特质和属性一起**决定角色的起点**，但不决定终点：

```
例子A：STR 16 + "蛮力"(STR+2) + "愚钝"(INT-2) + "厚皮"(-1物理伤害)
  → 极端力量型，技能盘点近战是顺水推舟
  → 但玩家可以用跳跃点法术分支，做"笨但能打的魔剑士"

例子B：STR 14 + DEX 14 + "瘦弱"(STR-1,DEX+1) + "快手"(DEX+2) + "旧伤"(HP-10%)
  → 属性被特质推向了DEX方向
  → 低HP但有高闪避，技能盘走远程/闪避是对的
  → 但玩家也可以硬走近战——高DEX意味着高AC，做个闪避坦克

例子C：全属性平均12，无极端特质
  → 没有明显方向的万金油
  → 技能盘怎么点都行，完全由玩家定义
```

**玩家手下的"错配"不一定错** — 高STR点远程？那是个能拉硬弓的重甲射手。低INT点法术？那是个靠跳跃强行补位的野路子法师。随机生成角色做不到这种创意，但玩家可以。

### 整体结构

```
                    ┌─────────┐
                    │ 远程专精 │ ← 大节点（技能）
                    └────┬────┘
                         │
              ┌──── 小节点(副属性) ────┐
              │     (暴击率+3%)         │
              │                        │
    ┌─────────┴─────┐         ┌────────┴──────┐
    │   双重射击    │         │   穿透射击    │
    └────────┬──────┘         └───────┬───────┘
             │                        │
          小节点                    小节点
       (射程+1)                 (移动+1)
             │                        │
             └────────┬───────────────┘
                      │
              ┌───────┴───────┐
              │    启 程      │  ← 中心节点（所有角色1级解锁）
              └───────┬───────┘
                      │
             ┌────────┴───────────────┐
             │                        │
          小节点                    小节点
       (HP+5)                  (AC+1)
             │                        │
```

### 节点类型

#### 大节点（技能节点）
- 提供**主动技能**或**核心被动技能**
- 占1个技能点
- 命中后获得显著能力提升
- **节点效果仅在对应条件下生效**：近战节点在拿远程武器时不生效，法术节点在没有法术位时不生效——但节点不会被移除，换回对应装备即可再次生效

### 点亮规则

#### 基本规则
- 每次升级获得 **1个技能点**
- 只能点亮**与已点亮节点相邻**的节点
- 从"启程"开始向外扩展
- 小节点通常连接两个大节点，作为路径上的"过路点"

#### 跳跃规则（每5级1次）
- **5级、10级、15级、20级**各获得1次"跳跃"机会
- 跳跃可以**无视相邻规则**，直接点亮技能盘上任意一个节点
- 跳跃到远处的大节点不需要点亮路径上的小节点
- 跳跃使角色可以跨分支发展（如近战角色跳到法术分支）

#### 路径解锁
- 从启程向外辐射出**6条主路径**（对应6个属性方向）
- 每条路径分为3层：入门→进阶→大师
- 不需要专走一条路，可以混合发展

```
                    STR方向（近战/力量）
                        ↑
        WIS方向 ← 启程 → DEX方向
        （治疗/感知）  ↓   （远程/灵巧）
                    INT方向（法术/智力）

        CHA方向（指挥/魅力）← 启程 → CON方向（防御/体魄）
```

### AI自动加点（随机生成角色）

酒馆招募和NPC角色的技能点**根据属性和特质量身分配**，但准确度随等级递进——低级角色像新手一样会"犯傻"，高级角色像老手一样精准。

#### 加点逻辑
```
1. 读取最终属性（基础3d6 + 种族修正 + 特质修正）
2. 读取功能性特质，标记特殊倾向
3. 确定主方向（最高属性）和副方向（次高属性）
4. 按准确度分配技能点
```

#### 准确度随等级变化

| 等级 | 加点风格 | 说明 |
|------|----------|------|
| 1-3级 | **摸索期** | 有40%概率点非主方向的小节点，可能出现高STR去点DEX路线的情况。像新手一样在探索自己的可能性 |
| 4-6级 | **渐入佳境** | 80%概率沿主方向走，20%偏移。开始发现自己的长处 |
| 7-9级 | **成熟期** | 90%概率沿主方向走，跳跃机会用于加速主方向。基本形成了清晰的build |
| 10级+ | **老练期** | 95%沿主方向，跳跃机会30%用于补短板（跨分支），70%用于深化主方向。精确且高效 |

#### 准确度影响什么
```
低级角色(1-3级)：
  高STR角色 → 可能点了"基础射击"（DEX路线）→ "这个人力气大但非要去当弓箭手"
  → 在玩家手里不一定是错的！可能变成一个能拉硬弓的力量型射手

高级角色(7级+)：
  高STR角色 → 几乎一定点了"基础剑术"→"连击"→"剑术精通"→ 清晰的近战build
  → 可预测，可信赖，但缺少惊喜
```

#### 为什么低级角色不精准是好事
- 酒馆里看到3级角色"属性很好但技能盘歪了" → 便宜捡漏的机会
- 玩家接手后可以用跳跃纠正方向 → 变废为宝的成就感
- 高级角色技能盘合理但贵 → 买现成 vs 淘宝养成 的策略选择

#### 特质对AI加点的影响
功能性特质会**微调**AI的方向偏好：
| 特质 | AI方向微调 |
|------|-----------|
| 长臂(近战射程+1) | STR方向+20%权重 |
| 猫眼(远程命中+1) | DEX方向+20%权重 |
| 博闻强记(法术位+1) | INT方向+20%权重 |
| 术士血脉(法术DC+1) | INT方向+30%权重 |
| 亲和力(社交) | CHA方向+10%权重 |
| 旧伤(HP-10%) | CON方向+10%权重（AI倾向于补HP） |

## 职业称号系统

> 角色没有固定职业，但技能盘上的投资方向会自然形成**职业称号**。称号是纯粹的标签——不限制任何机制，仅用于 UI 展示、角色卡识别和 AI 推荐路径。

### 判定规则

1. **涉足定义**：在某个属性区域（STR/DEX/CON/INT/WIS/CHA）点亮 ≥1 个大节点 = 涉足该属性
2. **职业名**：由已涉足的属性**组合**决定（与投资比例无关，只看是否涉足）
3. **动态变化**：每次点亮新的大节点后重新判定。一个「战士」点亮「以太感知」后立刻变为「魔剑士」
4. **主属性标记**：已涉足属性中大节点数最多的那个为「主属性」（并列取角色属性值最高的），在 UI 中排在组合首位
5. **无涉足**：仅点亮「启程」和路径上的小节点（未点亮任何大节点）时，称号为**「无名者」**

### 与现有系统的关系

- **不锁定机制**：职业名不影响技能盘选择、不限制可点亮节点、不修改任何数值
- **与属性倾向共存**：属性倾向（生成时由属性分布决定）是先天底色，职业称号（技能盘投资决定）是后天选择
- **AI加点参考**：AI自动加点时，倾向将属性倾向对应区域的节点优先级提高（已有逻辑不变）
- **UI 展示**：角色卡、技能盘界面、战斗单位信息栏均可显示当前职业称号

## 角色状态
| 状态 | 说明 |
|------|------|
| 健康 | 正常 |
| 轻伤 | HP < 50%, -1全检定 |
| 重伤 | HP < 25%, -2全检定 |
| 濒死 | HP = 0, 每回合需强韧豁免 |
| 死亡 | HP ≤ -HP上限, 永久死亡 |

## 永久死亡
- 角色死亡不可复活（除非高阶复活法术）
- 装备可由队友拾取
- 每个角色的技能盘配置和特质都是独一无二的，死亡=不可替代的损失

## 原型阶段
- 技能盘先实现3条分支（近战/远程/防御）
- 每条分支2-3个大节点
- 小节点5-6种
- 跳跃规则完整实现
- 属性类特质20个 + 功能性特质5个
- AI加点实现摸索期和成熟期两档
