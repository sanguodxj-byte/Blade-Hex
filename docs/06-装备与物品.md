# 装备与物品

## 核心设计原则：单位类型 = 当前装备

**角色没有任何固定的类型标签。** 一个角色是近战单位还是远程单位，是骑兵还是步兵，是长枪兵还是巨剑士——完全由他当前持有的装备决定。

| 身份判定 | 取决于 |
|----------|--------|
| 近战单位 | 手持近战武器（长剑/巨斧/长枪...） |
| 远程单位 | 手持远程武器（长弓/十字弩/手弩...） |
| 法术单位 | 手持法杖/魔导书且有点出法术节点 |
| 骑兵 | 骑乘状态（有坐骑） |
| 长枪兵 | 手持长枪（武器属性决定） |
| 重甲兵 | 身穿重甲 |
| 双持单位 | 主副手各持一把轻巧武器 |
| 盾战士 | 手持盾牌 |

**换装备 = 换身份。** 一个穿板甲持长剑骑马的骑士，下马换成长弓就是弓骑兵，再下马换上巨斧就是重装步兵。同一个角色，同一套属性和技能盘，完全不同的战场角色。

### 动态身份的影响
- 技能盘上的节点**不锁定单位类型**。点了"连击"的人拿弓一样能射，只是连击只对近战生效
- 坐骑可以上/下马，坐骑死亡则失去骑兵身份
- AI自动根据当前装备选择最优行动（有弓就射，被贴身就切近战）
- 战场上的角色图标/外观根据当前装备动态变化

## 主副武器系统

### 核心规则
每个角色有 **主武器槽** 和 **副武器槽**，可以同时装备两把武器并在战斗中随时切换。

- **切换武器** = 消耗1个**次要行动**（不是主行动）
- 切换后本回合仍可用主行动攻击
- 两把武器的属性、射程、伤害完全独立
- 角色面板同时显示两把武器，当前持有的高亮

### 武器槽配置示例
| 主武器 | 副武器 | 战术用途 |
|--------|--------|----------|
| 长剑（近战） | 长弓（远程） | 标准战弓组合：近可砍远可射 |
| 巨剑（近战） | 十字弩（远程） | 重武+高伤远程 |
| 长剑 + 盾牌 | 匕首（副手） | 防御模式切换攻击模式 |
| 法杖（施法） | 长剑（近战） | 魔剑士：远施法近砍人 |
| 长弓（远程） | 匕首（近战） | 射手被贴身时切近战自卫 |
| 巨斧（近战） | 火油瓶×3（消耗品） | 蛮族：砍完丢火瓶 |
| 长枪（近战） | 长剑（近战） | 反骑+通用近战 |

### 双手武器规则
- 双手武器（巨剑/巨斧/长弓/十字弩）装备时占用主副两槽
- 装备双手武器 → 副武器槽不可用
- 想带副武器 → 必须选单手武器
- 这意味着选择双手武器 = 放弃切换灵活性，换更高伤害

### 盾牌与副武器
- 盾牌不占副武器槽，而是独立的盾牌槽
- 所以可以同时装备：主武器(长剑) + 副武器(长弓) + 盾牌
- 但持盾时只能使用单手主武器
- 切到双手副武器时盾牌自动收起（失去AC加成）

## 武器系统

### 武器分类（3×3×3 矩阵体系）

武器按 **伤害类型 × 重量级** 分为 3×3×3 矩阵 + 远程 + 施法媒介，完整子类型见 `weapon_subtypes.json`。

#### 近战武器（砍伤/刺伤/钝伤 × 轻/中/重 × 3档）

| 伤害类型 | 轻型(L) | 中型(M) | 重型(H) |
|---------|---------|---------|---------|
| **砍伤(Slash)** | 匕首/萨克斯短刀/廓尔喀刀 | 武装剑/战斧/游牧弯刀 | 巨剑/巨斧/长柄刀 |
| **刺伤(Pierce)** | 细剑/刺剑 | 步兵矛/宽刃矛/锥枪 | 骑枪/钩镰/三叉戟 |
| **钝伤(Crush)** | 棍棒/轻锤/拳套 | 翼锤/军锤/链枷 | 大锤/巨棒/长柄锤 |
| 伤害骰 | 1d4~1d6 | 1d8~1d12 | 2d6~2d8 |
| 基础AP | 2~3 | 4 | 5~6 |
| 攻击范围 | 1格 | 1格 | 1~2格 |

武器具体属性由 `weapon_subtypes.json` 中的 `subtype` 配置驱动（伤害骰、AP消耗、命中修正、穿透修正、射程、装填AP等）。

#### 远程武器（投掷/弓/弩 × 轻/中/重 × 3档）

| 类型 | 轻(L) | 中(M) | 重(H) |
|------|-------|-------|-------|
| **投掷(Thrown)** | 飞刀/飞镖/掷斧 | 标枪/重标枪/渔叉 | 掷石/重掷矛/掷锤 |
| **弓(Bows)** | 短弓/猎弓/游牧弓 | 强弓/反曲弓/战弓 | 长弓/复合弓/巨弓 |
| **弩(Crossbows)** | 轻弩/猎弩/手弩 | 标准弩/强弩/狙弩 | 重弩/攻城弩/弩炮 |

> 伤害骰为内部计算用，玩家UI显示伤害范围。远程武器具体射程、装填AP等由 `weapon_subtypes.json` 定义。

#### 施法媒介（Catalyst）

| 类型 | 说明 |
|------|------|
| 法杖(Staff) | 近战可用（1d6），施法触媒 |
| 法珠(Orb) | 纯施法触媒 |
| 魔杖(Wand) | 纯施法触媒 |

`CanCastSpells` 检查条件：主手持 `IsCatalyst=true` 的武器，不可装备盾牌，只能穿 DR≤3 的轻甲。

### 武器属性
| 属性 | 说明 |
|------|------|
| 轻巧 | 可双持，可作副手武器 |
| 投掷 | 可远程投掷（标明射程） |
| 双手 | 占用主副两槽，不可同时带副武器 |
| 装填 | 射击后需1次要行动装填才能再次射击 |
| 钝击 | 钝击伤害类型，对骷髅等亡灵全额伤害 |
| 破甲 | 计算命中时目标AC-2 |
| 长柄 | 近战攻击范围2格（不需要贴身） |
| 反骑兵 | 对冲锋中的目标伤害翻倍 |

### 武器切换与战术

| 战术场景 | 操作 | 消耗 |
|----------|------|------|
| 远程被贴身 | 切换副武器(近战) → 攻击 | 次要行动 + 主行动 |
| 近战需要远程 | 切换副武器(弓) → 射击 | 次要行动 + 主行动 |
| 双持攻击 | 主手攻击 → 副手攻击 | 主行动 + 次要行动 |
| 持盾防御 → 双手猛攻 | 收盾(免费) → 切副武器(巨剑) | 次要行动 |
| 法师被贴身 | 法杖近战反击 | 主行动（无需切换） |

### 切换限制
- 切换消耗次要行动，每回合只有1个次要行动 → 每回合最多切换1次
- 切换后**不能在同一回合再次切换回原武器**
- 被眩晕/冰冻时不可切换
- 切换不触发借机攻击（视为原地动作）

## 防具系统

### 闪避(AC) + 装甲(DR) 分离与偏折系统

防御分为两个独立的层级，虽然遵循“闪避AC与装甲DR分离”原则，但为了模拟护甲对攻击的物理偏折作用，铠甲的装甲强度（DR阈值）会部分折算为偏折AC，以补偿因重甲限制敏捷带来的防御损失：

| 防御层 | 对抗什么 | 由什么决定 | 说明 |
|--------|---------|-----------|------|
| **闪避AC** | 命中检定 | BaseAc + DEX修正(受限) + 护甲/盾牌偏折AC + 装备AcBonus词缀 + 技能/饰品 | 能不能躲开或偏折弹开攻击 |
| **装甲DR** | 命中后的伤害分配 | 护甲耐久 + 穿透阈值 | 打中了疼不疼 |

**偏折设计**：重甲极大地限制了角色的敏捷闪避（DEX上限为0或1），但是其厚重的材质和弧形表面可以将武器的斩击或刺击偏折开（模拟为 AC 增加）。代码将装甲强度折算为 AC：`floor(sqrt(ArmorDR))`。

### 护甲表

| 类型 | 护甲分类 | DEX上限 | DR阈值 | AP惩罚 | STR需求 | 说明 |
|------|---------|--------|--------|--------|---------|------|
| 布衣(无甲) | Light | 无限制 | 0 | 0 | — | 纯DEX闪避 |
| 法师长袍 | Light | 无限制 | 3 | 0 | — | 法师限定，DR≤3 |
| 皮甲 | Light | 无限制 | 6 | 0 | — | 轻甲基础 |
| 镶钉皮甲 | Medium | ≤4 | 8 | 1 | — | 中甲入门 |
| 链甲 | Medium | ≤3 | 11 | 2 | — | 标准中甲 |
| 半板甲 | Heavy | ≤1 | 15 | 4 | STR 13 | 重甲基础 |
| 全板甲 | Heavy | 0 | 18 | 5 | STR 17 | 最重重甲 |

> **DEX上限**：限制DEX对闪避AC的贡献上限。全板甲DEX上限=0意味着完全不靠闪避。
> **AP惩罚**：护甲对行动点的惩罚（中甲-1~2，重甲-4~5）。
> **DR阈值**：穿透检定目标值，同时作为装甲耐久基数（MaxArmorPoints = DrThreshold × 10）。
> 头盔/靴/手套的详细数据见 `armors.json`。

> **DEX上限**：限制DEX对闪避AC的贡献上限。板甲DEX上限=0意味着穿板甲后完全不靠闪避。

### 盾牌

盾牌是**唯一同时影响AC和DR的装备**——举盾是主动格挡（影响是否命中），盾体也有物理厚度（吸收伤害）：

| 类型 | DR阈值 | AC来源 | AP惩罚 | 远程伤害乘数 | 特殊 |
|------|--------|--------|--------|-------------|------|
| 轻木盾 | 3 | `floor(sqrt(DR))=+1` | 0 | 0.50 | 可被斧类击碎 |
| 步兵圆盾 | 4 | `floor(sqrt(DR))=+2` | 0 | 0.45 | — |
| 步兵重盾 | 5 | `floor(sqrt(DR))=+2` | 1 | 0.35 | — |
| 骑士盾 | 6 | `floor(sqrt(DR))=+2` | 1 | 0.30 | — |
| 军团塔盾 | 8 | `floor(sqrt(DR))=+2` | 2 | 0.25 | 不可同时攻击 |

> 盾牌AC来自 `floor(sqrt(DR))`，不是固定值。盾牌还提供 `RangedDamageMultiplier` 减免远程伤害。

### AC计算（代码实际实现）
```
闪避AC = 基础AC(8) + DEX修正(受限) + sqrt(ArmorDR) + sqrt(ShieldDR) + 护甲AcBonus + 盾牌AcBonus + Buff
有效AC = 闪避AC + 被动技能AC加成 + 士气AC修正
```
其中：
- **基础 AC**：由单位 BaseAc 属性决定，默认为 10。
- **DEX修正**：$\lfloor\sqrt{\text{DEX}/2}\rfloor$，受当前穿戴护甲的 `max_dex_bonus` 限制贡献上限。
- **护甲偏折 AC**：$\lfloor\sqrt{\text{护甲DR阈值}}\rfloor$（当装甲值大于 0 且未损坏时生效）。
- **盾牌偏折 AC**：$\lfloor\sqrt{\text{盾牌DR阈值}}\rfloor$（当盾牌未损坏时生效）。
- **装备 AcBonus**：护甲或盾牌上附带的额外 AC 属性（包含词缀加成）。

DEX贡献 = floor(sqrt(DEX / 2))：
- DEX 10 → +2, DEX 18 → +3, DEX 30 → +3, DEX 40 → +4

DEX上限（由护甲的 `max_dex_bonus` 控制）：
- 轻甲：无限制
- 中甲：上限1-2
- 重甲：上限0（完全不依赖DEX闪避）

### 防御体系效果对比

| 配置 | 闪避AC | DR耐久 | 被剑砍(砍伤) | 被弩射(刺伤) | 被锤砸(钝伤) |
|------|--------|--------|-------------|-------------|-------------|
| 皮甲无盾 DEX18 | **13** | 5 | 容易打中，轻甲挡一点 | 容易打中，穿透100%HP | 容易打中，磨甲 |
| 锁甲+盾 DEX14 | **14** | 17 | 较难打中，中甲较硬 | 较难打中，穿透100%HP | 较难打中，磨甲 |
| 板甲+盾 DEX10 | **12** | 33 | 容易打中(铁罐子)，极硬甲 | 容易打中，穿透100%HP | 容易打中，磨甲极慢 |
| 无甲无盾 DEX30 | **13** | 0 | 较难打中，打中就真伤 | 较难打中，直接满伤 | 较难打中，直接满伤 |

> **刺伤克制无盾目标**：穿透后100%HP伤害，DR无效。
> **盾牌克制远程**：提高AC让箭更难命中。
> **钝伤克制重甲**：90%伤害打DR，持续磨甲。

### 装甲耐久(DR)系统

防具除了AC，还有独立的DR装甲耐久：

```
战斗开始：初始化 DR = 护甲.max_dr + 盾牌.max_dr + 天然护甲

被命中后 → 穿透判定：
  d20自然骰 >= DR穿透阈值 → 穿透成功
    ├── 砍伤穿透：70%HP + 30%DR伤害
    ├── 刺伤穿透：100%HP，0%DR（无视装甲）
    └── 钝伤穿透：30%HP + 70%DR（装甲损毁后伤害×1.5）
  
  d20 < DR穿透阈值 → 未穿透
    ├── 砍伤未穿：0%HP + 40%DR伤害（削甲）
    ├── 刺伤未穿：0%HP + 10%DR伤害（几乎不削甲）
    └── 钝伤未穿：10%HP + 90%DR伤害（始终漏一点）

DR ≤ 0 → 装甲损毁，DR阈值降为0，所有伤害直接打HP
```

**战术意义**：
- 刺伤（弩/枪）→ 对付重甲最佳（穿透后100%HP伤害）
- 钝伤（锤）→ 削甲利器（90%DR伤害），适合持久战磨掉装甲
- 砍伤（剑/斧）→ 平衡型，穿透后70%HP

## 坐骑系统

### 核心规则
坐骑是独立的装备位，不是角色永久属性。骑上马就是骑兵，下马就是步兵。

### 坐骑装备槽
- 每个角色有一个**坐骑槽**（可为空）
- 坐骑槽有坐骑 → 骑乘状态 → 获得骑兵加成
- 坐骑槽为空 → 步兵 → 无骑兵加成

### 坐骑类型
| ID | 坐骑 | 速度 | HP | 负重 | 冲锋加成 | 特殊 | 价格 |
|----|------|------|----|------|---------|------|------|
| `pack_horse` | 驮马 | +1 | 15 | 高 | 0% | 纯运输 | 20 |
| `war_horse` | 军马 | +2 | 20 | 中 | +25% | 可骑射(短弓/手弩) | 80 |
| `elite_war_horse` | 战马 | +3 | 25 | 中 | +50% | 免疫恐惧，可骑射 | 200 |
| `elf_stag` | 精灵角鹿 | +2 | 18 | 低 | +25% | 森林穿行，潜行不中断，可骑射 | 150 |
| `dwarf_war_bear` | 矮人战熊 | +1 | 30 | 高 | +25% | 1d4额外伤害，免疫恐惧 | 250 |
| `wolf` | 狼 | +2 | 12 | 低 | +25% | 包夹命中+1，可骑射 | 60 |

> 坐骑数据通过 `MountData.GetAllMounts()` 工厂方法创建。`AllowedMountedWeapons` 控制骑射可用武器。

### 骑乘状态效果
| 效果 | 说明 |
|------|------|
| 速度+坐骑加成 | 移动格数增加 |
| 冲锋可用 | 移动3+格后攻击获得冲锋加成 |
| 高度+1 | 骑在马上视为比步兵高1级（对步兵获得高地优势） |
| 不可进入密林/洞穴 | 骑乘状态不可进入某些地形 |
| 被长枪克制 | 被长枪武器攻击时伤害×2 |
| 骑射 | 骑乘时可用短弓/手弩射击（长弓/十字弩不可） |

### 上/下马
- **上马**：消耗1个次要行动（马在相邻格时）
- **下马**：消耗1个次要行动（自由选择时机）
- **坐骑被打死**：骑手被摔下，受到1d6伤害，下回合速度-2

### 坐骑独立HP
- 坐骑有独立HP（军马HP 15，战马HP 25）
- 敌方可以选择**攻击骑手**或**攻击坐骑**
- 坐骑HP归零 → 坐骑死亡，骑手落地
- 坐骑可以被治疗（治疗法术选择目标时包含坐骑）

## 动态单位身份速查

### 身份如何随装备变化

```
同一角色 — 穿锁甲，属性不变，技能盘不变：

装备A：长剑 + 长弓 + 无坐骑
  → 步兵剑弓手（可近可远，标准万金油）

装备B：长剑 + 盾牌 + 军马
  → 骑兵盾战士（冲锋+防御，怕长枪）

装备C：长枪 + 无副武器 + 军马
  → 骑兵枪骑兵（冲锋伤害极高，反骑也强）

装备D：巨剑 + 无副武器 + 无坐骑
  → 重装步兵（高伤害近战，无远程手段）

装备E：长弓 + 匕首 + 精灵角鹿
  → 骑射手（骑射+森林穿行，被贴身切匕首）

装备F：法杖 + 长剑 + 无坐骑
  → 战斗法师（远程法术+近战自卫）
```

**一个角色通过换装就能胜任完全不同的战场角色。** 招募一个高STR高DEX的角色，给他配不同装备就是不同兵种。不需要分别招募"骑兵""弓手""长枪兵"。

## 物品与消耗品

| 物品 | 效果 | 使用时机 |
|------|------|----------|
| 治疗药水 | 恢复2d4+2 HP | 主行动 |
| 强效治疗药水 | 恢复4d4+4 HP | 主行动 |
| 解毒剂 | 解除中毒 | 主行动 |
| 火油瓶 | 投掷至目标格，范围内1d4火伤×3轮 | 主行动 |
| 圣水 | 投掷至目标格，亡灵2d6伤害 | 主行动 |
| 法术卷轴 | 施放一次法术 | 主行动 |
| 磨刀石 | 本场战斗近战伤害+1 | 战斗外使用 |

### 消耗品装备化
- 火油瓶/圣水/投掷武器 可放入副武器槽
- 从副武器槽使用 = 主行动直接投掷
- 用完后槽位清空，自动切回主武器

```
┌─────────────────────────────────────┐
│  主武器槽   [长剑]                  │
│  副武器槽   [长弓]    ← 可切换      │
│  盾牌槽     [铁盾]    ← 收起/装备   │
│  坐骑槽     [军马]    ← 上/下马     │
│  头部       [铁盔]                  │
│  身体       [锁甲]                  │
│  饰品×2     [力量戒指][抗毒项链]     │
└─────────────────────────────────────┘
```

- **主武器 + 副武器**：随时切换，消耗次要行动
- **盾牌**：独立槽，持盾时主武器必须单手
- **双手武器**：占用主副两槽，无法带副武器
- **坐骑**：独立槽，有坐骑=骑兵，无=步兵

## 原型阶段
- 主武器4种（长剑/巨剑/长枪/法杖）+ 副武器3种（长弓/十字弩/匕首）
- 防具2种（皮甲/锁甲）+ 盾牌1种（木盾）
- 坐骑1种（军马）
- 消耗品1种（治疗药水）
- 武器切换 + 上下马机制完整实现
- **身份动态变化**：验证同角色换装备后行为正确变化

## 饰品系统

### 饰品类型
| 类型 | 槽位 | 效果范围 |
|------|------|----------|
| 戒指 | 饰品槽 | 属性加成、特殊效果 |
| 项链 | 饰品槽 | 抗性、豁免加成 |
| 护符 | 饰品槽 | 被动技能触发 |

每个角色有 **2个饰品槽**，可同时装备两件饰品。

### 饰品示例
| 饰品 | 效果 |
|------|------|
| 力量戒指 | STR +2 |
| 抗毒项链 | 免疫中毒 |
| 火焰护符 | 火焰抗性 +50% |
| 敏捷指环 | DEX +2 |
| 生命之心 | 每回合恢复 1 HP |

## 装备词缀系统

详见 **20-装备稀有度与词缀系统.md**，此处概述：

### 词缀来源
装备可通过词缀获得额外属性。词缀由 `EquipmentAffix`（Resource）定义。

| 词缀类型 | 前缀/后缀 | 效果 |
|----------|-----------|------|
| 属性加成 | 前缀（如"强力的"） | +1~+3 某项属性 |
| 伤害加成 | 前缀（如"锋利的"） | +1~+3 武器伤害 |
| 抗性加成 | 后缀（如"防火的"） | +20%~50% 特定抗性 |
| 特殊效果 | 后缀（如"吸血的"） | 特殊被动效果 |

### 稀有度
| 稀有度 | 词缀数 | 颜色 |
|--------|--------|------|
| 普通 | 0 | 白色 |
| 优秀 | 1 | 绿色 |
| 精良 | 2 | 蓝色 |
| 史诗 | 3 | 紫色 |
| 传说 | 4+ | 橙色 |

## 装备方案系统（军队用）

军队中的士兵不逐个装备，而是使用**装备方案**：

| 概念 | 说明 |
|------|------|
| 装备方案 | 预定义的装备组合（如"长枪兵方案"=长枪+皮甲+无盾） |
| 应用对象 | 一个兵种堆叠的所有士兵 |
| 切换方式 | 在军队管理界面切换方案 |
| 成本 | 切换方案消耗金币和物资 |

这确保了"单位身份由装备决定"的设计原则同样适用于军队士兵。

## 数据模型参考

装备系统的核心数据类（均为 `Resource`）：

| 类 | 基类 | 用途 |
|----|------|------|
| `ItemData` | Resource | 基础物品（名称、描述、图标、重量、价值） |
| `WeaponData` | ItemData | 武器（伤害骰、射程、属性、武器标签） |
| `ArmorData` | ItemData | 防具（AC、DEX上限、STR需求、护甲类型） |
| `ConsumableData` | ItemData | 消耗品（使用效果、使用时机） |
| `MountData` | ItemData | 坐骑（速度加成、HP、特殊效果） |
| `AccessoryData` | ItemData | 饰品（属性加成、特殊效果） |
| `EquipmentAffix` | Resource | 词缀（修改项、数值范围、适用装备类型） |

### UnitData 装备槽结构

`UnitData` 中的装备相关字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `PrimaryMainHand` | ItemData | 主武器组主手（当前） |
| `PrimaryOffHand` | ItemData | 主武器组副手 |
| `SecondaryMainHand` | ItemData | 副武器组主手（可切换） |
| `SecondaryOffHand` | ItemData | 副武器组副手 |
| `Armor` | ArmorData | 身体护甲 |
| `Shield` | ArmorData | 盾牌（独立槽位） |
| `Helmet` | ArmorData | 头盔 |
| `Boots` | ArmorData | 靴子 |
| `Gloves` | ArmorData | 手套 |
| `Accessories` | AccessoryData[] | 饰品（最多2个） |
| `Mount` | MountData | 坐骑 |

> 主/副武器组切换消耗1次要行动。双手武器占用主副两槽。`CombatStats.GetMainHand/GetOffHand` 通过 `usingPrimaryWeapon` 参数选择当前激活的武器组。
