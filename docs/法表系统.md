# 法表系统设计 (Spell Book System)

> 数据版本：2026-05-17-v1.3
> 替代：v0.6 §10.3 / §10.4 的"散件 spell 列表"
> 关联文档：`Blade_Hex_战斗数值系统_修订版_v0.6_含节点系统.md`、`技能盘系统.md`
>
> ⚠ **核心约束（全游戏）**：本游戏**完全没有复活机制**。HP 归零即永久阵亡（对玩家与敌人均成立）。任何 spell / 技能 / 道具 / 节点都不能让阵亡单位复活。这是法表系统、技能盘、道具系统、剧情系统都必须遵循的硬规则。

## 1. 设计目标

旧 v0.6 §10.3/§10.4 把 spell 散落在技能盘节点上，每个节点对应 1 个固定 spell。问题：

- 法师可选 spell 数量极少（只有节点解锁的几个）
- spell 单点能量过低，与"法师"该有的爆发感不符
- sim 实测：法师即便能施法仍 ~10% 胜率，因为 spell 吃 AC、吃 DR、吃护甲耐久

新法表系统改为：

- **5 个学派 × 5 个环位 = 25 个 spell**
- 法师通过 5 个 INT 大节点解锁"1-5 环法术槽"，**激活时玩家选择学派**，固定获得该系该环位的 spell
- 战斗中的法术书掉落、商店购买、剧情奖励可补充更多法术
- 每个法师后期能学到 5-10 个 spell（5 槽 + 法术书）

## 2. 核心规则（最重要的设计取舍）

### 2.1 法术 = 必定命中、真伤

**Spell 不掷命中检定，不查目标 AC**。法术施放后伤害必定结算到目标身上，**完全无视护甲 DR、护甲耐久、盾牌耐久**。这与武器攻击形成本质对比：

* **武器攻击**：d20 攻击检定 vs AC → 命中后 d20 穿透检定 vs DR → 装甲耐久吸收
* **法术攻击**：直接造成 baseDice + Mod(INT) [+NodeSpellDamage] 的真伤到 HP

理由：
1. INT 主属性除了 Mana 外没有 AC / HP 收益，必须给法术爆发位置才能与 STR/CON 持平
2. 装甲不应抵御能量攻击（火、闪电、心灵）
3. 简化战斗结算管线（无穿透判定 / 无装甲耐久扣减 / 无盾牌远程减免）

### 2.2 AOE 全额真伤

AOE 法术对**所有目标全额结算**伤害，**不再有"主目标 100% / 副目标 50%"的递减规则**。

理由：法师本就靠 AOE 站身位，强制"主+副"结算等于强迫法师"专心打谁"，与 nuker 定位矛盾。

例外：连锁闪电（毁灭系 4 环）按"跳跃距离逐次衰减"是 spell 自身的机制，与 AOE 不同。

### 2.3 豁免 = 只对 debuff，不减伤害

法术的 baseDice + Mod(INT) 伤害**永远全额结算**。豁免检定只决定：

* 控制效果（晕眩 / 麻痹 / 减速 / 恐惧 / 混乱）是否成功施加
* 状态效果的持续时间是否减半（按效果定义）

**不存在"豁免减半伤害"机制**。

### 2.4 学派无元素抗性

**单位 Resistances/Weaknesses/Immunities 字段对法术伤害无效**。火 / 冰 / 闪电都视为同一种"魔法真伤"，仅作为风味区分（VFX、关卡剧情、法术书学习）。

理由：
1. 避免"亡灵免疫心灵"等 spell 在不同关卡间烈度起伏
2. 法师面对所有敌人输出曲线一致，平衡好做
3. 元素互克是动作游戏的特色，不适合本系统的 build 多样性目标

### 2.5 不能暴击

法术不掷命中，自然没有自然 20 暴击。所有 spell 默认**不能暴击**。

## 3. 资源系统

### 3.1 Mana
保留 v0.6 §10.0 公式：

```
MaxMana = 10 + INT + floor(Level/2) + NodeManaMax
```

每场战斗开始 Mana 满。**法师战斗外可饮用 Mana 药水**回复但战斗内不能。

### 3.2 法术 DC（控制效果豁免阈值）

```
SpellDC = 8 + ProfBonus(Level) + Mod(INT) + NodeSpellHit
```

* `ProfBonus(Level)` 复用 RPGRuleEngine 的熟练加值表
* Mod(INT) 同 RPG 系统 floor(sqrt(INT/2))
* 受目标豁免对抗（具体豁免类型见每个 spell）

### 3.3 法术伤害（继承 v0.6 11.4 节点法伤）

```
EffectiveSpellDamage = baseDice + Mod(INT) + NodeSpellDamage
```

* 所有目标全额结算（§2.2）
* `NodeSpellDamage` 来自 SkillTree 的 spell_damage 累加
* int_b10 "知识就是力量" 节点：`+Mod(INT)`（叠加，所有目标全额）

### 3.4 装备限制（继承 v0.6 §10.0）

施法者必须满足：

* 主手装备 Catalyst（魔杖 / 法球 / 法杖），不可持非 Catalyst 武器
* 不可装备盾牌
* 仅可穿 Cloth 类护甲（DR ≤ 3）

不满足条件时，法师无法施放任何 spell。

## 5. 法术获取系统

### 5.1 技能盘大节点（5 个）— 主动选系

INT 区域的 5 个 BIG 节点改为"环位法术槽"，激活时**玩家从 5 个学派中选择 1 个**，获得对应学派该环位的 spell：

| 节点 ID | 名 | 解锁 | 选择 |
|---|---|---|---|
| `int_b02` 法术研习·1 环 | 1 环槽 | 任意激活路径 | 玩家选 1 系：火花 / 模糊术 / 祈福 / 法师护甲 / 治疗轻伤 |
| `int_b03` 法术研习·2 环 | 2 环槽 | 前置 int_b02 | 玩家选 1 系：冰锥术 / 镜像 / 迅捷术 / 反法术 / 治疗重伤 |
| `int_b05` 法术研习·3 环 | 3 环槽 | 前置 int_b03 | 玩家选 1 系：火球术 / 隐形术 / 群体祈福 / 魔法抗性 / 群体治疗 |
| `int_b09` 法术研习·4 环 | 4 环槽 | 前置 int_b05 | 玩家选 1 系：连锁闪电 / 时间停滞 / 英勇加持 / 力场屏障 / 再生术 |
| `int_b11` 法术研习·5 环 | 5 环槽 | 前置 int_b09 | 玩家选 1 系：雷暴 / 影武者分身 / 圣裁之力 / 神圣庇护 / 生命之歌 |

* 玩家可在 5 个槽中选不同系，最大化 build 多样性
* 选定后保存到 `CharacterSkillTree.LearnedSpells`，**不可改变**（除非通过法术书覆盖）
* 激活前置：必须按技能盘的相邻规则解锁；越高环位需要更高 INT（建议 1 环 INT≥10、5 环 INT≥30）

### 5.2 法术书（剧情 / 战利品 / 商店）

* 战斗 / 探索掉落"法术书 - 火球术"等具体物品
* 法师使用法术书后永久习得对应 spell（如已习得则碎为 mana 药水）
* 法术书价格：4 × Ring^2 × 100 金（1 环 400、5 环 10000）
* 高级 NPC 商店与城镇法师塔提供小批量轮换库存

### 5.3 已学法术列表

* 法师 INT 大节点全 5 个激活 + 4 本法术书 → 9 个 spell
* 期望中后期法师每场战斗能挑选 4-6 个 spell 释放（受 Mana 限制）

### 5.4 阵亡 = 永久（无复活机制）

**本游戏完全没有复活机制**。`wis_b03` 节点（旧"复活"）从设计中**彻底移除**，不会改写为非 Spell 技能、不会有"复活卷轴"消耗品、不会有"灵魂之石"等剧情道具。HP 归零的单位永久离场。

理由：
1. 战斗节奏需要"死亡有重量" — 队友倒下意味着真实损失，玩家会更谨慎调度阵型
2. 复活机制让治疗系无限期挂着"复活"作为终极保险，使生命系定位模糊
3. 与 v0.6 §11.8 中已有的"濒死保险机制"（`con_b09 最后阵地` HP < 25% 触发 AC+5 / 伤害+50%、`iron_will` 致命时 DC15 强韧豁免存活于 1HP）配合 — 这些都是**死亡前的反打窗口**，而不是死后的复活

旧 spell `resurrect`、节点 `wis_b03 复活`、文档中所有"复活"相关条目均废弃，由更强的"濒死防御"机制（如 4 环防护系"力场屏障"、5 环"神圣庇护"、4 环生命系"再生术"、5 环生命系"生命之歌"）填补预防性保险位置。

## 6. 战斗内决策与限制

### 6.1 行动限制（继承 v0.6 §3.3 / §10.0）

* Spell 不计入"每回合 1 次非 Spell 主动技能"
* 但 Spell 之间无频次硬限制（Mana / AP 是唯一限制）
* `Counterspell`（反法术 2 环）作为反应技，每回合最多触发 1 次

### 6.2 反法术触发顺序

`Counterspell` 抵消顺序：

1. 防御方先消耗 Mana 进入 stack
2. 攻击方可继续 `Counterspell` 抵消防御方的 `Counterspell`（嵌套 2 层为上限）
3. 最终未被抵消的 spell 正常结算

### 6.3 法术伤害堆叠

* 所有 spell（含 AOE）目标全额受 `NodeSpellDamage` + `int_b10` 知识就是力量加成
* `NodeSpellHit` 加在 SpellDC 上（提高豁免阈值），不影响 spell 伤害

## 7. 法术 vs 武器输出对比

为了让法师中后期不再过弱，调整法术爆发系数。lvl 30 法师 INT=22，Mod=4：

| 环 | spell 名（毁灭系）| 单目标真伤期望 | AOE 总真伤期望 | Mana 效率 |
|---|---|---|---|---|
| 1 | 火花 | 4.5 + 4 = 8.5 | - | 2.1 dmg/mana |
| 2 | 冰锥术 | 10.5 + 4 = 14.5（每目标全额）| ~43（3 目标）| 5.4 dmg/mana |
| 3 | 火球术 | 14 + 4 = 18 | ~72（4 目标全额）| 6.0 dmg/mana |
| 4 | 连锁闪电 | 17.5 + 4 = 21.5 | ~75（4 目标递减，特殊机制）| 4.7 dmg/mana |
| 5 | 雷暴 | 27 + 4 = 31 | ~186（6 目标全额）| 7.8 dmg/mana |

对比战士（巨剑 3d6 + STR Mod 4 + Lv5 精通 50%）：~21 / 命中, 命中率 ~50%, 50% 穿透概率, 然后还要被 DR 削减…实际单回合 ~15-20 dmg 单目标。

法师 4-5 环 spell 单 cast 即追上战士整回合输出，且 5 环 AOE 单回合横扫小队 → **重塑法师 nuker 定位**。

低环位 spell 仍弱于武器，但消耗低，可作为续航。3 环以上是关键打击。法师此时不再受 AC / 装甲 / 学派抗性影响 → 输出曲线极其稳定。

* `basic_heal` / `group_heal` / `life_circle` (并入生命系 1-3 环)
* `resurrect` / `wis_b03 复活` (**彻底移除** — 全游戏无复活机制，详见 §5.4。`wis_b03` 节点 ID 应替换为新的 WIS 系治疗 / 防护节点)
* `arcane_judgment` / `purifying_flame` / `wis_b05 神圣制裁` (取消，生命系不允许伤害；如需"对邪恶 +攻"应在毁灭系外处理)
* `blessing` (并入附魔系 1 环祈福)

## 10. 后续设计开放点

* **法师装备 Catalyst 的属性词缀**：法术伤害 +X / SpellDC +X / Mana 上限 +X（替代武器系的 PenBonus）
* **法术学派精通**：技能盘 INT 区域可设计 5 个新小节点 `int_s_destruction_master` 等，对应学派伤害 +20%
* **持续轮换**：每场战斗 Mana 不能完全回满（如 50% 起步），强迫法师选择 spell 优先级
* **法术升阶**：高环位法术槽中再点亮"法术增幅"小节点，可使低环位 spell 多消耗 Mana 但伤害提升 1 倍
* **混合法师**：5 个 INT 槽如果跨多系（如 1环毁灭 / 2环防护 / 3环附魔...），获得"通才"特性（额外 Mana / 法术书价格折扣）
* **专精法师**：5 个 INT 槽全选同一系，获得"专精"特性（该系 spell 伤害 +50% 或 Mana 减半）
