# 地图纹理与拼接规格 — 大地图 × 战斗地图

> 本文档是**美术资产 + 渲染架构的单一数据源 (SSOT)**。所有地形相关的贴图、着色器、立牌立牌资源都按本文档的路径和命名规则生产。
>
> **两套地图的画风是独立的**，不共享纹理资产，但共享 `HexOverworldTile.TerrainType` 枚举和 `TerrainVisualProfile` 数据层。

---

## 一、画风定位

| 维度 | 大地图 | 战斗地图 |
|---|---|---|
| 视角 | 俯视 2D 平面 | 斜 45° 3D 相机 |
| 几何 | 纯 2D 六边形精灵 | 3D 六棱柱（半径 96，高 48）|
| 贴图性质 | 整张六边形 PNG，带 alpha | tileable 2D 写实贴图 |
| 画风 | 羊皮纸、粗粝水彩、战场兄弟 | HD-2D 风格，写实但风格化（八方旅人 / Triangle Strategy） |
| 光照 | 无（绘画里已自带明暗）| unshaded + shader 内假光（非 PBR，无法线，无金属） |
| 角色 | 小纸片人图标 | Sprite3D billboard（`BillboardMode.FixedY`）|
| 树木/岩石/建筑 | 绘进地形贴图里 | 独立 Sprite3D billboard 立牌 |
| 过渡 | 边缘羽化 + 过渡 overlay | shader splatmap 邻边混合 |

## 三、大地图贴图绘制规范

### 3.1 几何与尺寸

- 外接圆半径对应 `HexOverworldTile.HexSize = 156 px`
- 贴图画布尺寸建议 **320 × 280 px**（两倍精度，超出部分用 alpha 溢出）
- **关键**：主色调必须在 hex 几何边界外**溢出 8–12 px**，alpha 从 100% 渐变到 0%。相邻格叠上去时羽化带互压，看起来连续。硬边会非常难看。

### 3.2 画风约束（摘自 `14-大地图设定.md`）

- 大地色系：赭石、焦褐、枯黄、苔绿、铁灰
- sRGB 饱和度压到 40% 以下
- 粗粝水彩质感，不用平滑渐变，用粗笔触块面
- 地面细节：手绘微缩感，避免 Q 版、日系、高光反射
- 参考作品：《战场兄弟》大地图、中世纪古地图（Mappa Mundi）、《暗黑地牢》地图美术

### 3.3 变体要求

每种主地形至少 **2 个 variant**（森林/密林/沼泽为 3 个），打破重复。`VariantHasher.Pick(coord)` 对世界坐标做确定性哈希，同一个坐标永远选到同一张 variant，战斗结束回到大地图也不会"换脸"。

### 3.4 方向位 overlay

道路和河流不能一格一格画成同一张图。命名约定：

```
road_{bitmask}.png
```

`bitmask = sum(1 << dir for dir in connected_directions)`，方向索引 0..5 对应 `HexOverworldTile.CubeDirections`（东 / 东北 / 西北 / 西 / 西南 / 东南，flat-top 六边形）。

**至少必须画的 12 个位掩码**（其它组合可通过旋转变体得到）：

| bitmask | 形态 | 描述 |
|---|---|---|
| `0b000000` (0) | 孤立点 | 断头，通常不会出现 |
| `0b000001` (1) | 东端点 | 进入地块的入口 |
| `0b001001` (9) | 东-西直线 | 横穿 |
| `0b010010` (18) | 东北-西南 | 对角线 |
| `0b100100` (36) | 西北-东南 | 对角线 |
| `0b000011` (3) | 东-东北 | 60° 弯 |
| `0b000111` (7) | 东-东北-西北 | T 字 |
| `0b001011` (11) | 东-东北-西 | T 字另向 |
| `0b001111` (15) | 东+东北+西北+西 | 四叉 |
| `0b011011` (27) | 三叉（方向旋转对称） | 三叉 |
| `0b111111` (63) | 六向全连 | 枢纽 |
| `0b010011` (19) | 东-东北-西南 | Z 形 |

河流命名同理：`river_{bitmask}.png`。

过渡贴图独立：

- `beach_arc_0.png` ~ `beach_arc_3.png`：陆-水过渡，带沙滩弧
- `forest_edge_0.png` ~ `forest_edge_5.png`：森林外缘的散落树冠

这些 overlay 渲染在基础地形 bucket 之上、实体 icon 之下。

## 四、战斗地图贴图绘制规范

### 4.1 顶面 tileable 贴图

- 尺寸：**512 × 512 px**（必要时 1024 × 1024，仅限高可见性地形如 grassland）
- **可水平和垂直无缝平铺**（tileable），边缘像素必须匹配
- 无法线贴图、无金属/粗糙度贴图，shader 是 `unshaded`
- 画风：写实草皮/泥土/石板/沙地/雪地顶视 + 斜 45° 视角参考
- 色调饱和度**比大地图高 20~30%**，与角色立牌颜色匹配
- 平均色调须落在 `TerrainVisualProfile.DominantColor ±10%` 范围内（文档首部色值表可用 minimap 时还会使用）

### 4.2 侧面 cliff 贴图

- 尺寸：**256 × 512 px**（高宽比匹配六棱柱侧面拉伸）
- 垂直可重复、水平可重复
- 同样 unshaded，底部由 shader 加暗 AO

### 4.3 视觉一致性要求

进入战斗的格子必须"看起来是刚才大地图上那个格子"。靠色调锚点保证：

```
平均色(OverworldKey 贴图) ≈ 平均色(BattleTopKey 贴图) ≈ DominantColor
```

美术交付前，可用 Photoshop 取色对比两张贴图的整体平均 Lab 值，差值 ΔE < 15 为合格。

### 4.4 立牌贴图

- 尺寸参考：
  - 树：**128 × 256 px**（窄高比，底部为根部）
  - 岩石/灌木：**128 × 128 px**
  - 建筑/废墟：**256 × 256 px** 或 **512 × 512 px**
- 底部 8 px 为"接地带"，带 alpha 渐变羽化，和地面纹理融合
- 整体边缘必须有 alpha，禁止硬边
- 画风：HD-2D 写实风，参考八方旅人树木、Triangle Strategy 建筑
- 光源方向：从相机左上方打光，阴影打在右下方（与 shader `fake_light_dir = (0.35, 0.9, 0.25)` 一致）

### 4.5 阴影贴花（可选，后续补充）

每个立牌底部放一个半透明椭圆阴影贴花：

```
res://src/assets/props/shadows/
  shadow_small.png    128×64，适合灌木/小石
  shadow_medium.png   256×128，适合树木
  shadow_large.png    512×256，适合建筑
```

当前版本 `BattlePropRenderer` 未挂阴影贴花，待后续抛光。

## 五、Shader 契约

### 5.1 顶面 shader (`battle_ground_top.gdshader`)

**uniform 清单**：

| 名称 | 类型 | 默认 | 说明 |
|---|---|---|---|
| `top_texture` | `sampler2D` | — | 主顶面贴图，tileable |
| `texture_tile_scale` | `float` | `0.12` | 贴图在世界 XZ 的缩放比 |
| `neighbor_tops` | `sampler2DArray` | 空 | 邻居地形贴图数组，用于边缘混合（暂未启用） |
| `neighbor_blend_enabled` | `bool` | `false` | 是否启用邻边混合 |
| `edge_blend_inner` | `float` | `0.55` | 边缘混合起始半径（归一化） |
| `edge_blend_outer` | `float` | `0.95` | 边缘混合完全半径 |
| `fake_light_dir` | `vec3` | `(0.35, 0.9, 0.25)` | 假光方向 |
| `fake_light_min/max` | `float` | `0.75 / 1.05` | 亮度范围 |

**per-instance data（由 `HexCellMultiMeshBatcher` 写入）**：

- `CUSTOM0`：`r = highlight_flag`, `gba = highlight_color_rgb`
- `CUSTOM1`（预留扩展）：`r = shroud_flag`, `g = edge_mix_strength`, `ba = neighbor_terrain_packed`

## 六、数据流架构

```
┌────────────────────────────────────────────────────────────┐
│ BladeHexCore (无渲染类型)                                    │
│                                                             │
│   HexOverworldTile.TerrainType          [枚举 SSOT]          │
│   TerrainVisualProfile                  [per-terrain 数据]   │
│   TerrainVisualRegistry                 [查表 SSOT]          │
│   VariantHasher                         [确定性变体选择器]    │
│   BattleTerrainBridge                   [战斗↔大地图 桥接]    │
│   BattlePropPlacement                   [立牌摆放数据]        │
│   BattlePropScatter                     [立牌散布算法]        │
└────────────────────────────────────────────────────────────┘
                         │ string IDs only
                         ▼
┌────────────────────────────────────────────────────────────┐
│ BladeHexFrontend/View (渲染)                                 │
│                                                             │
│  大地图侧:                                                    │
│   HexOverworldRenderer  → MultiMeshInstance2D               │
│                         → 按 profile.OverworldKey 加载贴图    │
│                                                             │
│  战斗地图侧:                                                  │
│   CombatMaterialManager → ShaderMaterial 缓存                │
│                         → 顶面 top_texture / 侧面 cliff       │
│   HexCellMultiMeshBatcher → MultiMeshInstance3D              │
│                         → MaterialOverride = 顶面材质         │
│   BattlePropRegistry    → Texture2D 按 prop_id 查表          │
│   BattlePropRenderer    → Sprite3D billboard 实例化          │
└────────────────────────────────────────────────────────────┘
```

**严格不变式**：

1. Core 层所有渲染相关字段都是 `string`（key / path），不持 `Texture2D` / `Material` / `Shader`
2. 两套 terrain→texture_name 的 switch 语句被 `TerrainVisualRegistry` 替代，后续新增地形只需改 registry
3. 大地图 / 战斗地图 共享 `TerrainType` 枚举，但**贴图路径和 key 独立**
4. Variant 选择由 `VariantHasher.Pick(worldCoord)` 决定，跨场景确定性一致

## 八、风险与注意事项

| 风险 | 缓解 |
|---|---|
| 美术交付周期长，阻塞架构联调 | 占位阶段用 `CreateSolidColorTexture` + `DominantColor`，渲染器已具备回退 |
| 变体数量在代码和贴图间不一致 | `TerrainVisualRegistry` 是 SSOT，代码永远按 `Count` 取模；美术交少了贴图会循环用现有的 |
| 大地图羽化做得不够会有格子感 | 绘制 template 统一：每张贴图四周 alpha 渐变带 ≥ 8 px |
| 战斗顶面色调跳变 | 章节 4.3 的 ΔE 校对流程；美术 QA 时逐 terrain 对比 |
| 立牌 z-fight | `Sprite3D.AlphaCut = OpaquePrepass` + `BillboardMode.FixedY` 已配置 |
| 贴图缺失导致崩溃 | 所有加载点都有 `ResourceLoader.Exists` 检查 + 占位回退 |
