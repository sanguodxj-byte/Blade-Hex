# 技能星盘布局方案（最终敲定版）

> 本文是技能星盘节点分布的**权威规格**，当前实现由 `scripts/generation/generate_skill_tree_star_chart.py` 生成 `skill_tree_layout.json` / `skill_tree_content.json`，运行时由 `SkillTreeLayoutLoader` 加载。
> 取代旧版（N=9 环 / 45 槽 / 大节点 3→4 那套草案）。
> 核心：六扇区**同一套配额**，各自沿扇区三角形**中轴左右镜像对称**，**填满整个扇区**。

---

## 一、已锁定决策

| 维度 | 决策 |
|------|------|
| 扇区形状 | 边长 20 的等边三角（中心为顶点、外缘为底边），**400 三角格** |
| 起点 | `r1` = 六扇区共用的「启程」六边形的一瓣，**无属性**；扇区内容在 r2–r20 |
| 配额 | 六扇区**同一套** size 数量 |
| 对称 | 每扇区沿自己的中轴（主轴 `Dirs[sectorIdx]`）**左右镜像** |
| 填充 | **填满整个扇区**（r2–r20 全部 399 格铺满） |
| size 调色板 | **{1, 2, 3, 4, 6, 12} 格**（新增 1 格 pip + 12 格巨型） |
| 链接 | 中轴主干采用从 `r1` 指向 `r16` 的**直线链接/定位轴**；全盘仍密铺三角瓦片，不引入独立菱形格 |

---

## 二、几何模型

- 扇区 = 边长 20 等边三角，沿中轴从内到外 20 环，**第 r 环含 `2r−1` 格**。
- 累计 `Σ(2r−1) = 20² = 400 格`（上三角 210 + 下三角 190）。
- 六区 ×400 = 2400 = 半径 20 六边形的全部三角格，无缝铺满。
- **4 格单元视角**：边长 20 三角 = 100 个「边长 2 单元」（4 格大三角），10 个单元行，第 j 行 `2j−1` 个单元。布局算法可按单元切分。
- **每环对称槽**：第 r 环 `2r−1` 格 = **轴上 1 格 + 离轴 (r−1) 对**。轴上节点须自身左右对称；离轴节点须成镜像对。

### 坐标系（沿用现有 `SkillTreeCoord`）
- 每个 hex 格点 `(q,r)` 关联 2 个三角瓦片：`t=0` ▽、`t=1` ▲，编码 `EncodeTile(q,r,t)=Vector2I(q*2+t, r)`。
- 共边邻接 `GetTileNeighbors`（每三角 3 个共边邻居）。
- 瓦片环号 = 三顶点 hex 距离的最大值（见 `SkillNodeShape.TileRadialDistance`）。

## 三、三段规划带

> 内 / 中 / 外是节点摆位规划带，不是互斥瓦片分账；`r8` 是内中交界。实际可点内容仍从 `r2` 开始，硬约束仍是 `r2–r20` 共 **399** 格全填满。

| 段 | 规划环位 | 说明 |
|------|------|------|
| 内环 | r1–r8 | 含中心启程瓣；主动 ×2、被动 ×2，节点中心靠近本段中心半径 r4–r5 |
| 中环 | r8–r15 | 主动 ×2、被动 ×4，节点中心靠近本段中心半径 r11–r12 |
| 外环 | r16–r20 | 巨型 ×1、主动 ×2、被动 ×2，节点中心靠近本段中心半径 r18 |
| **内容合计** | r2–r20 | **399** 格，含大件、小点和 pip 收缝 |

## 四、节点配额（每扇区，六区同一套）

| size | 形状 | 语义 | 数量 | 小计格 |
|------|------|------|------|------|
| 12 | 巨型 | 终极节点 | 1 | 12 |
| 6 | 六角冠 keystone | keystone | 3 | 18 |
| 4 | 大三角 active_kite | 主动技能 | 6 | 24 |
| 3 | 三连 passive_triad | 被动技能 | 8 | 24 |
| 2 | 菱形 | 标准属性点（**主体**） | 156 | 312 |
| 1 | 单三角 pip | **仅填缝**/廉价微点 | 9 | 9 |
| **合计** | | | **183 节点** | **399** |

- **小点(2 格菱形)是绝对主体**；**pip(1 格)只在密铺收口、轴线落单处补缝**，不主动堆量。
- `2×156 小点 + 9 pip = 321`（399 − 大件 78）。

## 五、大件摆位骨架（每扇区，✦=轴上，⇆=左右对称对）

### 内环 r1–8（含起点；内容中心靠近 r4–r5）
- ✦ keystone ×1 @ r5
- ⇆ 主动 ×2 @ r4–r5
- ⇆ 被动 ×2 @ r4–r5
- 其余内容用菱形小点为主体，pip 只作收缝

### 中环 r8–15（内容中心靠近 r11–r12）
- ⇆ keystone ×2 @ r11–r12
- ⇆ 主动 ×2 @ r11–r12
- ⇆ 被动 ×4 @ r11–r12
- 其余内容用菱形小点为主体，pip 只作收缝

### 外环 r16–20（内容中心靠近 r18）
- ✦ **巨型 ×1 @ r18 居中**
- ⇆ 主动 ×2 @ r18
- ⇆ 被动 ×2 @ r18
- 其余内容用菱形小点为主体，pip 只作收缝

**校验**：被动 2+4+2 = 8 ✓｜主动 2+2+2 = 6 ✓｜keystone 1+2 = 3 ✓｜巨型 1 ✓｜剩余格由小点 + pip 填满至 399 ✓

## 六、够到成本（点亮门槛）

沿中轴每向外推进 1 环 ≈ **2 格**（▲▽ 交替，密铺下最细的线）；环 R 首次出现在第 `2R−2` 格。
点亮某节点最低投入 ≈ `2R−2`（pip 桥） + 该节点本体格数。成本 = 格数，与用大节点还是小点搭桥无关（2 格/环是硬地板）。

| 目标 | 环位 | 最低投入 |
|------|------|------|
| 内环 keystone | r5 | ~14 点 |
| 中环 keystone ×2 | r11 | ~26 点 |
| 外环巨型(12 格) | 外环中心 r18 附近 | ~42–44 点 |

梯度：内/中 keystone 门槛约 14 / 28，巨型约 43 收顶；节点中心按所属规划带中心半径放置，不再把巨型中心硬压到 r19。

## 七、规模

- 单扇区 **183 节点 / 399 点全清**。
- 六扇区 **1098 内容节点 / 2394 内容点**；加中心 `start` 后为 **1099 节点 / 2400 三角瓦片**。约 6 倍扩容，PoE 式巨树。
- 配 120 级、约 1–2 点/级，玩家只点得动一小块。

## 八、实现要点（已落地）

1. **新增 12 格巨型 size**：
   - `SkillNodeData.NodeType.Giant` 已加入，`GetRequiredTileCount` 返回 12。
   - `SkillNodeShape.BuildTemplateTiles` / `SkillNodeFigureCatalog` 支持 6 种巨型模板：`apex_sunburst_12`、`apex_arrowhead_12`、`apex_bastion_12`、`apex_crystal_12`、`apex_hourglass_12`、`apex_crown_12`。
2. **固定 JSON 布局**：
   - 旧 `ConstellationBuilder` 已移除；布局由生成器写入显式瓦片坐标。
   - 生成器按"三角扇区划分 → r1 启程瓣 → 中轴直线链接到 r16 → 外环巨型模板 → 左右镜像大件 → 小点密铺 → pip 收缝"产出每扇区 399 内容瓦片。
   - 中轴主干是从 `r1` 沿每个扇区三角形中线指向 `r16` 的**直线链接/定位轴**；底层节点仍由三角瓦片组成，表现上相邻三角经常形成一串菱形过渡节点，但不要求用三角瓦片共边 BFS 来定义这条主干。
   - 到达外环后先放置 12 三角瓦片巨型模板；巨型中心靠近外环中心半径 r18，并沿扇区中线居中。
   - 主动、被动、keystone 从中轴主干向左右侧成对扩展；离轴大件必须有镜像对，轴上大件必须自身关于中线对称。
3. **pip 收缝**：每扇区 9 个 1 格 pip；只给风格池下限属性，不带机制。
4. **链接/密铺**：中轴主干按直线链接定位；三角瓦片仍全扇区密铺，节点自身瓦片保持共边连通，避免破碎形状。
5. **前端**：`SkillTreeUI` 读取 `GridPosition` + `GetTiles`，使用显式瓦片绘制节点、命中区域和边界；扇区背景使用中心到六边形相邻顶点的三角形。
6. **过渡区**：最终方案不生成 `trans_*` 过渡节点。

---

## 九、校验（布局算法须过）

复用/扩展 `SkillTreeLayoutTests`：
1. **无重叠**：所有节点瓦片集两两不相交。
2. **填满**：r2–r20 共 399 格全被占用（含 pip 收缝），无空格。
3. **中轴链接**：每扇区存在从 `r1` 指向 `r16` 的直线中轴链接，固定大件围绕该中轴左右镜像。
4. **在界内**：所有瓦片在 R=20 六边形内。
5. **对称**：每扇区节点关于中轴镜像（轴上节点自对称、离轴成对）。
6. **配额**：每扇区恰好 1 巨型 / 3 keystone / 6 主动 / 8 被动，小点+pip 填满余下。
7. **数据一致**：layout/content ID 唯一且集合完全一致。
8. 现有 Godot headless 测试不回归（`unit_test_runner.tscn`）。

## 十、每角色差异化（A 档 · 固定大件）

> 目标：让不同角色技能盘"千人千面"，但**不破坏固定对称几何、不增加平衡风险**。

### 已锁定决策
- 采 **A 档**（固定几何 + 内容微差），**不做** C 档随机几何。
- **中心放射树没有"入口差异"**：所有角色从正中启程，六扇区对称、access 完全相同，旋转/换入口无意义。差异**只能来自内容**。
- **出身**：只决定**初始属性 + 初始物品**，**不碰技能盘**（无折扣 / 无预点 / 无亲和梯度）。
- **大件固定**：每扇区 `1 巨型 + 3 keystone + 6 主动 + 8 被动` 全部**策划手搓、人人相同**。
- **仅小点/pip 的属性 payload 因角色而异**：按角色 `seed` 在**所属扇区的风格池**内加权采样；几何位置、节点数人人相同，变的只是"同一个小点给什么属性"。
- **风格约束**：所有内容（含采样结果）必须留在扇区风格内，**绝不跨风格**（STR 永远力量近战味，等）。

### 架构落地
- 几何 `SkillTreeData` 仍**全局共享**（位置、大件、节点结构不变）。
- per-character payload = `seed` 生成的 `nodeId → 加成表`；`CharacterSkillTree` 应用小点属性时读该表，替代直接读共享节点的 `StatBonuses`（大件仍直接读共享节点，固定不变）。
- 存档只多存一个 `seed`，读档时**确定性重算** payload，与现有"只存节点 ID"的轻量存档完全兼容。
- 生成器用 `System.Random(seed)`（Core 纯逻辑，**勿用全局 `GD.Randf`**，否则不可复现、坏档）。

### 落地次序
1. 先把固定对称巨树（大件 + 小点默认属性）跑通。
2. 再加 `seed` 采样层（小点 payload）。
3. 大件也随机抽取（方案乙：扇区技能池抽 N 个上盘）**留作以后增强，现在不做**。

## 十一、六扇区风格池（"特定风格"权威定义）

> 大件固定取自此风格；小点/pip 的 seed 采样也限定在此风格的"属性流池"内。

| 扇区 | 属性流池（小点/pip 采样范围） | 技能主题（大件设计基调） |
|------|------|------|
| STR 力量 | 近战命中 / 近战伤害 / HP / AC / 暴击 | 近战爆发 + 生存：旋风斩、连击、嗜血、战吼 |
| DEX 灵巧 | 远程命中 / 远程伤害 / 先攻 / 速度 / 暴击 | 远程 + 机动刺杀：瞄准、连珠、箭雨、潜行 |
| CON 体魄 | HP / AC / 全豁免 / 治疗 | 坦克续航：盾击、铁壁、生命之环 |
| INT 智力 | 法力上限 / 法术命中 / 法术伤害 / AC / 全豁免 | 法术输出：法术研习、法术反射、魔力汲取 |
| WIS 感知 | 暴击 / 法力上限 / 法力回复 / 全豁免 | 刺客暴击：弱点穿刺、暗杀、影刃涂毒 |
| CHA 魅力 | 友军加成 / 先攻 / HP | 指挥光环：指挥、集结号令、英雄号召 |

> 大件内容由 `技能星盘节点内容设计.md` 维护；小点/pip 的合法属性键与权重以该文档第三节为准。
