# UI设计（深度优化版）

## 设计原则
- **不暴露骰子**：玩家看到概率、结果、伤害数字，不看到"1d20"等骰子术语
- **信息层级清晰**：核心信息一瞥可知，详情按需展开
- **反馈即时**：每次操作都有视觉+文字反馈
- **图像UI预留**：所有视觉属性通过 `UITheme` 集中管理，未来替换图像资源时只需修改主题单例
- **组件工厂模式**：所有UI组件通过 `UIFactory` 统一创建，切换渲染方式时无需修改业务代码

## 八、主题系统与图像UI预留

### 8.1 UITheme 设计令牌

| 类别 | 数量 | 说明 |
|------|------|------|
| 颜色 | 80+ | 背景色/边框色/文字色/HP/MP/士气/稀有度/区域/学派/种族 |
| 字号 | 8级 | xs(10) ~ title(28) |
| 间距 | 8级 | xs(2) ~ xxxl(32) |
| 圆角 | 5级 | sm(2) ~ round(12) |
| 尺寸 | 12项 | 按钮高度/条高度/图标尺寸/面板宽度/头像尺寸 |
| 动画 | 4级 | fast(0.15s) ~ very_slow(0.6s) |

### 8.2 图像资源占位

| 占位字段 | 当前值 | 未来替换 |
|---------|--------|---------|
| btn_normal_texture | null | 按钮正常态贴图 |
| btn_hover_texture | null | 按钮悬停态贴图 |
| panel_bg_texture | null | 面板背景贴图 |
| icon_atlas | null | 图标图集 |
| portrait_frame | null | 头像框贴图 |
| skill_node_active_texture | null | 技能盘激活节点贴图 |
| terrain_icon_atlas | null | 地形图标图集 |

### 8.3 切换流程

```
1. 准备图像资源
2. 在 UITheme 中设置对应的 Texture2D 字段
3. UIFactory 检测到图像资源后自动切换渲染方式
   (例如：Button → TextureButton, StyleBoxFlat → NinePatchRect)
4. 无需修改任何业务代码
```

## 九、交互流程

### 9.1 选中角色

```
点击角色 → 高亮移动范围(蓝) → 点击格子移动
移动后 → 高亮攻击范围(红) → 点击目标攻击
悬停敌方 → 显示命中率预览浮窗
悬停空地 → 显示地形信息提示
```

### 9.2 法术施放

```
点击[法术] → 法术选择面板 → 选择法术 → 
  定向法术 → 点击目标
  范围法术 → 点击中心 → 确认范围(AOE紫色高亮)
  锥形法术 → 点击方向 → 确认
```

### 9.3 物品使用

```
点击[物品] → 物品栏 → 选择消耗品 → 
  治疗药水 → 点击自身/友方
  火油瓶 → 点击目标格
  磨刀石 → 直接使用(战斗外)
```

### 9.4 大地图交互

```
左键点击地图 → 队伍移动到目标
鼠标滚轮 → 缩放地图
WASD → 移动摄像机
空格 → 暂停/恢复
接近城镇 → 进入城镇界面
接近敌人 → 触发战斗
```

## 十一、UI 工程规约（防错清单）

> 本节是写新 UI 必须对照的"防错检查清单"。每条都源自实际踩过的坑——违反任何一条，提交前回退修复。
> 适用范围：所有派生自 `Control` 的面板、自定义 `_Draw` 渲染、模态/覆盖式 UI。

### 11.1 自定义 Canvas 绘制文本（`Control._Draw` + `DrawString`）

**适用场景**：在 `Control` 子类的 `_Draw()` 中调用 `DrawString` / `DrawArc` / `DrawRect` 直接绘图（如 `SkillTreeUI`、`RadialMenu`、自定义图表等）。

**铁律：先 box 后 text，永远不要"反推 box"。**

✅ **正确做法**：
1. 先用 `font.GetStringSize(text, HorizontalAlignment.Left, -1, fontSize)` 测量文本真实尺寸
2. 用 `textSize + padding*2` 计算 `boxSize`，再确定 `boxPos`（左上角）
3. 画 box（背景 / 边框 / 高光）
4. 画文字时：
   - `position.X = boxPos.X`（box 左边缘）
   - `width = boxSize.X`（box 宽度）
   - `HorizontalAlignment.Center` 才会真正在 box 内水平居中
   - `position.Y = boxPos.Y + padY + font.GetAscent(fontSize)` 让 baseline 落在 box 上下内边距之间

❌ **常见错误**：
- 把 `pos = boxCenter` 直接传给 `DrawString` 并用 `width = -1`：`HorizontalAlignment.Center` 此时退化成"从 pos 向右画"，文字漂在 box 右半边或溢出
- 用 `pos = textCenter - stringSize/2` 试图手动居中：忽略了字体 ascent/descent 的不对称，文字在 box 内垂直偏移
- 用 `font.GetStringSize(... Center ...)` 测量后又画 box——`HorizontalAlignment` 参数对 `GetStringSize` 在 `width=-1` 时无效，返回的是文本固有尺寸，等同 `Left`

✅ **代码模板**：

```csharp
var font = ThemeDB.FallbackFont;
int fs = Math.Max(8, (int)(baseFontSize * zoom));
Vector2 textSize = font.GetStringSize(text, HorizontalAlignment.Left, -1, fs);
float ascent = font.GetAscent(fs);

float padX = 8.0f * zoom, padY = 4.0f * zoom;
Vector2 boxSize = new(textSize.X + padX * 2, textSize.Y + padY * 2);
Vector2 boxPos = anchor - new Vector2(boxSize.X * 0.5f, 0); // 以 anchor 为水平中点

// 1) 先画 box
DrawRect(new Rect2(boxPos, boxSize), bgColor, true);
// 2) 再画文字（pos.X = box 左边、width = box 宽度，让 Godot 在 box 内居中）
Vector2 textPos = new(boxPos.X, boxPos.Y + padY + ascent);
DrawString(font, textPos, text, HorizontalAlignment.Center, boxSize.X, fs, fgColor);
```

### 11.2 RichText / Label 在固定宽度容器内的换行

**适用场景**：`RichTextLabel` / `Label` 放在 `PanelContainer`、`VBoxContainer` 等有宽度约束的父节点里（如右侧详情栏、卡片内容）。

**铁律：内容受容器约束的文本必须显式开启换行，并禁止"按内容撑大"。**

✅ **必须设置**（针对受约束的 RichTextLabel）：
```csharp
rt.AutowrapMode = TextServer.AutowrapMode.WordSmart;  // 按词软换行
rt.FitContent = false;     // 不按内容撑开容器
rt.ScrollActive = true;    // 内容超长时面板内可滚动
rt.SizeFlagsHorizontal = SizeFlags.Fill;  // 填满父宽度，不外扩
```

✅ **针对 Label**（多行场景）：
```csharp
lbl.AutowrapMode = TextServer.AutowrapMode.WordSmart;
lbl.CustomMinimumSize = new Vector2(parentWidth, 0);
```

✅ **父容器侧**（防止被 HBox 拉伸破坏固定宽度）：
```csharp
panel.SizeFlagsHorizontal = SizeFlags.ShrinkBegin;  // 严格保持固定宽度
panel.ClipContents = true;                           // 兜底裁切任何子节点越界
```

❌ **常见错误**：
- 直接 `new RichTextLabel()` 而不设 `AutowrapMode`：长文本不换行、横向溢出
- 保持 `FitContent = true`（这是 `UIFactory.CreateRichText` 的默认值）：文本会撑大父 VBox，把同行其他控件挤偏
- 父 `PanelContainer` 用 `SizeFlags.ExpandFill`：固定 `MinSize` 失效，被 HBox 拉伸到不可控宽度

✅ **改造 `UIFactory.CreateRichText`**（推荐统一约定）：在 panel 里调用后立即设置上述 4 项，或者在 factory 内提供 `CreateConstrainedRichText(minSize)` 默认就开启 WordSmart + FitContent=false + ScrollActive。

### 11.3 模态 / 全屏面板的输入拦截

**适用场景**：覆盖在场景之上的全屏面板（技能盘、ESC 菜单、对话框、商店、组合背包等）。下层场景必须感知不到键鼠事件，否则会出现"打开面板按 W 角色还在走""滚轮缩放穿透到大地图"等 bug。

**铁律：模态面板可见时，所有键鼠事件必须在它的 `_UnhandledInput` 里被消费或显式吃掉。**

**相机阻断使用引用计数（`PushInputBlock` / `PopInputBlock`）：**

```csharp
// 打开面板时
_camera.PushInputBlock();   // 计数 +1，相机停止响应

// 关闭面板时
_camera.PopInputBlock();    // 计数 -1，归零时相机恢复

// 多个面板同时打开不冲突：
//   PartyPanel.Open → Push (count=1)
//   SkillTree.Open  → Push (count=2)
//   SkillTree.Close → Pop  (count=1, 相机仍阻断)
//   PartyPanel.Close→ Pop  (count=0, 相机恢复)
```

不要直接赋值 `ExternalControl = true/false`——这会破坏引用计数。

✅ **实现规则**：

1. **键盘**：用**事件驱动**（`_UnhandledInput` 捕获 InputEventKey），不要用 `Input.IsKeyPressed` 全局轮询
   - 维护内部 `_moveUp/Down/Left/Right` bool 状态
   - 在 `_Process` 里用这些状态算移动，不再读 `Input`
   - 关闭面板（ESC / 关闭按钮 / Visible=false）时**必须清零所有 held 状态**，避免下层场景收到"卡住的按住"残留

2. **滚轮**：在 `_UnhandledInput` 全局拦截，不要只依赖 `GuiInput`（GuiInput 只在 hover 时触发，鼠标在面板边缘外的滚轮就漏了）
   - 在 `_drawContainer` 矩形内：执行缩放
   - 在矩形外：什么也不做，但仍 `SetInputAsHandled()` 阻止穿透

3. **每条拦截**最后必须调用 `GetViewport().SetInputAsHandled()`，否则事件继续传给下层场景

✅ **代码模板**：

```csharp
public override void _UnhandledInput(InputEvent e)
{
    if (!Visible) return;

    // 1. ESC：关闭并清零状态
    if (e is InputEventKey { Pressed: true, Keycode: Key.Escape })
    {
        Visible = false;
        _moveUp = _moveDown = _moveLeft = _moveRight = false;
        EmitSignal(SignalName.CloseRequested);
        GetViewport().SetInputAsHandled();
        return;
    }

    // 2. WASD/方向键：维护状态 + 吃掉事件
    if (e is InputEventKey ke && !ke.Echo)
    {
        bool handled = true;
        switch (ke.Keycode)
        {
            case Key.W: case Key.Up:    _moveUp = ke.Pressed; break;
            case Key.S: case Key.Down:  _moveDown = ke.Pressed; break;
            case Key.A: case Key.Left:  _moveLeft = ke.Pressed; break;
            case Key.D: case Key.Right: _moveRight = ke.Pressed; break;
            default: handled = false; break;
        }
        if (handled) { GetViewport().SetInputAsHandled(); return; }
    }

    // 3. 滚轮：在画布内缩放，画布外吃掉防穿透
    if (e is InputEventMouseButton mb && mb.Pressed
        && (mb.ButtonIndex == MouseButton.WheelUp || mb.ButtonIndex == MouseButton.WheelDown))
    {
        var rect = _drawContainer.GetGlobalRect();
        if (rect.HasPoint(mb.GlobalPosition)) { /* zoom logic */ }
        GetViewport().SetInputAsHandled();
        return;
    }
}
```

❌ **常见错误**：
- `_Process` 里用 `Input.IsKeyPressed(Key.W)`：全局轮询，下层场景的 `_Process` 同样读到
- 只在 `_drawContainer.GuiInput` 里处理滚轮：鼠标移到面板外（标题栏、信息栏）时滚轮事件漏出去
- 关闭面板没清零按住状态：用户按住 W 然后按 ESC，下层场景从此一直认为 W 被按着
- 拦截了事件但忘记 `SetInputAsHandled()`：事件继续传给下一个 `_UnhandledInput` 处理者

### 11.4 通用 Control 设置默认

| 场景 | 必须设置 | 说明 |
|------|---------|------|
| 模态面板根节点 | `SetAnchorsAndOffsetsPreset(LayoutPreset.FullRect)` | 全屏覆盖、自适应分辨率 |
| 任何吸收点击的面板 | `MouseFilter = Stop` | 防止点击穿透到下层 3D 场景 |
| 自定义 `_Draw` 容器 | `ClipContents = true` | 平移/缩放时绘制内容不越出边界 |
| 固定尺寸侧栏 | `SizeFlagsHorizontal = ShrinkBegin` + `CustomMinimumSize` | 不被 HBox 拉伸破坏布局 |
| 富文本/长文 Label | `AutowrapMode = WordSmart` | 防止水平溢出 |
| 轮询替代品 | 用 `_UnhandledInput` + 内部状态 | 不要用 `Input.IsKeyPressed` |
