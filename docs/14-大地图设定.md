# 大地图设定

> **关于本文档**
> 大地图在每次开局时**程序化生成**（见 `docs/overworld_generation_design.md` 与 `docs/overworld_generation_supplement.md`）。本文档不再列出固定的城镇、河流、关隘清单，而是定义**程序化生成必须遵守的世界设定约束**：画风、视角、可达边界、迷雾规则、出生地分配等。
>
> 任何"具体的城镇名 / 河流名 / 关隘名"都不再是设定，而是**生成产物**。
>
> 📐 距离 / hex 换算见 [31-比例尺与距离体系.md](./31-比例尺与距离体系.md)（SSOT）。
> 1 hex ≈ 250 m，原型尺寸 80×60 hex ≈ 20 km × 15 km；正式版可扩展至 200×150 hex ≈ 50 km × 37 km。

---

## 一、地图总览

### 视角与画风（恒定，与种子无关）
- **视角**：3D HD-2D 风格，斜俯视（约 45~60 度）。3D 六边形地块上叠加 2D billboard 立牌（树/山/城）
- **底色**：羊皮纸（暖黄褐色，hex 级 stochastic tiling，整张大地图共用一张材质）。详见 `.limcode/design/overworld-parchment-map-design.md`
- **画风核心**：古朴、粗粝、低饱和，拒绝光滑和华丽；中世纪 Mappa Mundi 古地图 + 战场兄弟大地图的视觉语言
- **整体色调**：以大地色系为主——赭石、焦褐、枯黄、苔绿、铁灰
- **地形渲染**：
  - 山是手绘墨线三角立牌，不是 3D 山丘
  - 森林用墨线树冠立牌，密集铺在森林 hex 上
  - 水域用手绘波纹，灰蓝/铅色
  - 城镇/建筑是小而精致的 billboard 剪影
  - 道路：泥土色虚线
  - 角色图标：小圆底座 + 纸片人立绘剪影
  - UI 叠加层：羊皮纸面板、蜡封图标、铁锈色边框
- **绝对避免**：日系明亮色彩、卡通渲染、高光反射、渐变天空、华丽粒子效果

### 程序化生成保证的"恒定特征"
无论种子如何，以下永远成立：
1. **大陆被海洋环绕**（高程 falloff），玩家始终在内陆开局
2. **存在至少 4 类区域**：寒冷高海拔区、茂密森林区、温带平原区、贫瘠高危区（详见 §2）
3. **存在至少 1 条贯穿陆地的河流**（贝塞尔雕刻）
4. **存在至少 3~4 座人类 Town/Castle 作为道路网络枢纽**
5. **至少 1 座龙巢出现在最高雪山**
6. **至少 1 个暗影教团据点出现在远离城镇的偏远地带**

> 这些是**生成器的 invariant**，不是地理设定。生成器测试必须验证它们。

## 二、区域类型（生成器输出语义类别）

每次开局会按 `RegionRegistry` 的预设原型，结合本次种子的高程/温度/湿度，实例化若干语义区域。每个区域获得：
- 一个程序化命名（来自 `GeographicNameGenerator`）
- 一份地形偏好（哪几种 TerrainType 优先填充）
- 一个危险等级（影响 POI 密度、遭遇强度）
- 一个 POI 密度系数

### 默认六类区域（数据见 `RegionRegistry.cs`）

| 类型 | 偏好地形 | 危险度 | POI 密度 | 典型生成命名 |
|------|---------|-------|---------|------------|
| 高山带 | Mountain / MountainSnow / Snow / Hills | 0.7 | 0.6 | "霜冠群峰" / "铁砧山脉" / "苍穹高地"（生成器随机） |
| 茂密森林 | Forest / DenseForest | 0.3 | 0.9 | "银叶森林" / "暮光林地" / "古木树海" |
| 中央农业区 | Plains / Grassland | 0.1 | 1.2 | "金穗平原" / "丰饶旷野" / "麦浪沃土" |
| 干旱高危带 | Sand / Savanna / Wasteland | 0.8 | 0.7 | "焦土荒原" / "灰烬死地" / "末日不毛之地" |
| 沼泽湿地 | Swamp / Bog | 0.5 | 0.8 | "蛮荒沼泽" / "黑水湿地" / "瘴气泥沼" |
| 丘陵草原 | Savanna / Hills / Plains | 0.4 | 0.8 | "雷蹄旷野" / "赤铜丘陵" |

> 这些类型在每次开局**至少各出现一次**（通过 `WorldGenerator` 的保底逻辑）。位置和大小因种子而异。
>
> ⚠️ 若新增/删除区域类型，必须同步修改：
> - `BladeHexCore/src/Map/RegionRegistry.cs`（数据）
> - `BladeHexCore/src/Strategic/WorldRegionRegistry.cs`（位置约束）
> - `docs/01-世界观.md`（语义对接）
> - 本表

## 三、政治格局（程序化产物）

不再有固定数量的王国/城邦。生成器根据 POI 网络的连通性，**在生成完成后**推断政治格局：

### 推断规则
- **人类王国**：温带平原区中相互连通的 Town/Castle 群构成一个王国；多个不连通的群可能是多个王国（典型 1~3 个）
- **精灵王庭**：每个茂密森林区中心生成一个"森林深处" POI（不可进入或需要许可），归属于该森林的精灵阵营
- **矮人城邦**：每个高山带生成 1~3 个矮人 POI（地下入口），构成一个矮人联盟
- **半兽人部落**：每个丘陵草原 / 干旱带生成游牧式 Settlement，互相独立
- **暗影教团**：在远离任何 Town（>15 hex）的偏远区域生成 1~2 个 Settlement（ShadowCult）
- **野外部落**：哥布林/狗头人/牛头人作为 Settlement 散布在偏好地形上

### 阵营关系矩阵
| | 人类王国 | 精灵 | 矮人 | 半兽人 | 哥布林等 | 暗影教团 |
|---|---|---|---|---|---|---|
| 人类王国 | 内部诸侯有摩擦 | 冷淡通商 | 友好 | 敌对 | 敌对 | 敌对（多数王国不知教团规模） |
| 精灵 | — | 友善 | 互相尊重 | 戒备 | 敌对 | 仇恨 |
| 矮人 | — | — | 城邦联盟内部联合 | 敌对 | 敌对 | 敌对 |
| 半兽人 | — | — | — | 部落间内斗频繁 | 互不干扰 | 警惕 |
| 哥布林等 | — | — | — | — | 互相吞并 | 被操纵 |
| 暗影教团 | — | — | — | — | — | — |

### 非国家势力（散布于全图，由生成器钉在合适的 POI）
| 势力 | 钉在 | 作用 |
|------|------|------|
| 雇佣兵公会总部 | 最大的 Town（Prosperity 最高） | 招募中心、委托枢纽 |
| 银塔法师会塔 | 平原中段 Town 或 Outpost | 法术学习/卷轴交易 |
| 商队 | 道路上随机生成 | 中立经济实体 |

## 四、水系与地形（生成方式）

### 4.1 河流
- **算法**：贝塞尔地形雕刻（见 `overworld_generation_supplement.md` §S2）
- **数量**：1~3 条主河 + 0~2 条支流（随种子）
- **生成约束**：
  - 源头：`elevation > 0.65` 的可通行格
  - 入海口：邻接浅水的陆地格
  - 主河长度 > `RIVER_MIN_LENGTH`（约 30 hex）
  - 河谷雕刻强制降低高程（视觉上"劈"出峡谷）
  - 河流两侧三角洲偏好放置 Village
- **作用**：人类王国的运输动脉；天然边界；城镇布点的强力吸引子

### 4.2 海岸
- **位置**：地图边缘（高程 falloff 强制沉海）
- **特征**：岩质 + 沙滩混合
- **港口（Port）**：生成器在沿海平原 hex 上放置 1~2 座
- **海域**：玩家无法渡海，海岸线是绝对边界

### 4.3 山脉与关隘
- **算法**：高程图（噪声 + falloff）→ 高程 > 0.85 即山
- **关隘**：由道路生成器（A* 寻路）穿越山脊时自动产生，标记为"通行点"
- **不可通行**：纯山脊与雪峰（道路绕行或代价极高）

### 4.4 森林
- **算法**：温度+湿度满足"温带湿润"或"温带潮湿"的 hex 自动归为 Forest/DenseForest
- **精灵核心点**：森林面积最大块的几何中心 hex 被钉为"长老议会所在地"

## 五、POI 系统（程序化生成）

### 5.1 POI 类型（来自 `OverworldPOI.POIType`）

| 类型 | 含义 | 生成偏好 |
|------|------|---------|
| Town | 大型城镇，完整设施 | 中央平原，密集 |
| Village | 小村庄，基础设施 | 平原 / 森林边缘 |
| Castle | 城堡 | 丘陵 / 区域交界 |
| Port | 港口 | 沿海平原 |
| Outpost | 前哨站 | 道路要冲 / 边境 |
| Tavern | 旅店 | 道路途中 |
| Mine | 矿场 | 高山 / 丘陵 |
| Farm | 农场 | 平原密集区 |
| Shrine | 神龛 | 森林 / 山脉深处 |
| Settlement | 外族聚落（含子类型 SettlementRace） | 见外族表 |
| Lair | 巢穴（含子类型 LairType） | 见巢穴表 |

### 5.2 外族 Settlement 子类型

| SettlementRace | 偏好地形 / 区域 | 距最近 Town |
|----------------|----------------|-----------|
| Goblin | 森林边缘 / 沼泽 | > 8 hex |
| Kobold | 高山 / 丘陵地下 | > 10 hex |
| Minotaur | 荒漠 / 荒原 / 稀树草原 | > 12 hex |
| ShadowCult | 偏远（不限地形） | > 15 hex |

### 5.3 Lair 子类型

| LairType | 偏好地形 / 区域 |
|----------|----------------|
| DragonLair | 雪山高峰（elevation > 0.80） |
| AncientTomb | 沼泽 / 荒原远离城镇 |
| Ruins | 任意偏远地形 |
| GolemForge | 荒原边缘 |

### 5.4 命名
所有 POI 在生成时由命名生成器赋名（见 `BladeHexCore/src/Character/GeographicNameGenerator.cs`）：
- Town：[英雄名/家族姓] + [Land/burg/ford]，例如"塞德里克堡"
- Village：[地理特征] + [村/镇]，例如"绿溪村"
- Castle：[功能] + [石/堡]，例如"磐石要塞"
- Settlement / Lair：固定语义名（"哥布林营地"/"龙巢"）+ 地理修饰词

## 六、道路与商路（生成方式）

### 6.1 主干道
- **算法**：A* 连接 `Settlement` 类型为 Town/Castle 的 POI，使用代价权重（平原 1.0，丘陵 2.0，森林 3.0，山脉 8.0，河流 12.0）
- **桥梁**：道路穿越河流时标记为 `is_bridge`，渲染时叠加桥梁纹理（见 `overworld_generation_supplement.md` §S3）
- **效果**：×1.5 移速，低遭遇

### 6.2 商路
- **算法**：从 Town 出发延伸到周围 Village 的连接线，代价权重略低
- **效果**：×1.2 移速，中遭遇

### 6.3 小径
- **不主动生成**，但 A* 寻路允许在山地/森林/沼泽中以高代价通行
- **效果**：×0.8 移速，高遭遇

### 6.4 不可通行边界
| 边界 | 类型 | 说明 |
|------|------|------|
| 海洋 | 高程 < 0.2 | 玩家无船时不可渡海；港口提供"船运到另一座港口"作为非空间跳转 |
| 雪峰 / 高山 | 高程 > 0.85 | 道路绕行，玩家可硬闯（高代价 + 寒冷debuff） |
| 深沼泽 | 特殊 Tile flag | 移动代价极高，潜在毒沼伤害 |

## 七、地图探索系统（战争迷雾）

### 7.1 设计理念
参考文明系列：
- 玩家开局只看到**出身地周围一小圈**
- 其他区域被**神话生物插画**覆盖（海蛇/巨龙/奇美拉/利维坦），标注"传说中…"或问号
- 只有亲自探索或购买地图后，对应区域才揭示
- 已揭示的区域永久可见（文明式永久揭示）

### 7.2 迷雾层级

| 层级 | 视觉表现 | 游戏效果 |
|------|---------|---------|
| 未探索 | 羊皮纸上覆盖神话生物插画，地名标注为"传说中…" | 地形不可见，POI 不可见，无法移动到该区域 |
| 已揭示 | 正常地形显示，但无实时信息 | 地形可见，POI 可见（静态），敌方实体不可见 |
| 视野内 | 正常地形 + 实时动态 | 一切可见：移动中的 AI 实体、战斗、围攻等 |

### 7.3 出生时已知区域（按种族）

每次开局时，根据玩家出身种族，生成器寻找最匹配的区域类型作为起点周围的初始已知区域：

| 种族 | 寻找的初始区域 | 起点 POI 选择 |
|------|--------------|------------|
| 人类 | 中央农业区中最大的连通块 | 该区域内最大的 Town |
| 精灵 | 茂密森林区 | 森林边缘的 Village 或 Tavern |
| 矮人 | 高山带 | 山麓的矮人 POI |
| 半兽人 | 丘陵草原 | 草原边缘的 Outpost |
| 半精灵 | 中央农业区与森林接壤处 | 边境 Village |

> 起点 POI 由 `WorldGenerator` 在分析完图后推荐；具体物品奖励见 `origin_questions.json`。

### 7.4 揭示方式

| 方式 | 揭示范围 | 条件 |
|------|---------|------|
| 亲自探索 | 以玩家为中心 `VISION_RANGE`（≈ 600 px / 4 hex）的圆形区域 | 玩家队伍移动到该区域 |
| 购买地图 | 揭示特定区域（如"沼泽地图"揭示某沼泽区轮廓） | 在城镇商店花费金币 |
| NPC 情报 | 揭示单个 POI 或小范围 | 完成委托后 NPC 提供，或酒馆对话 |
| 侦察技能 | 视野半径 ×1.5 | 队伍中有游侠/斥候 |
| 间谍网络 | 揭示指定势力的所有领土 | 领主阶段（Phase 5），通过金钱和关系建立 |

### 7.5 未探索区域的神话插画
按未探索区域的**主导地形**选择对应贴图：

| 主导地形 | 神话插画 | 标注文字 |
|---------|---------|---------|
| 海洋 / 海岸 | 巨大海蛇缠绕船只 | "此处有海兽出没" |
| 山脉 | 巨龙盘踞山峰 | "龙族领地，凡人勿入" |
| 森林 | 巨树间幽光 + 怪异人脸 | "精灵的禁地" |
| 荒原 / 荒漠 | 牛头人骨架半埋沙中，远处火山 | "死亡之地" |
| 沼泽 | 沼泽中浮出眼球 + 骸骨 | "瘴气弥漫，有去无回" |
| 通用 | 手指从迷雾中伸出，罗盘指向不明 | "未知领域" |

插画风格：褐色墨水线条，与羊皮纸底色融合，中世纪版画质感。

### 7.6 技术实现要点

```
迷雾系统按 cell_size（≈ 20 px）分块存储:
  explored_grid: byte[Y, X]  // 0=未探索, 1=已揭示, 2=当前视野

每帧更新:
  1. 清除所有"当前视野"标记回退为"已揭示"
  2. 以玩家位置为中心，VISION_RANGE 内的格子标记为"当前视野"
  3. 这些格子永久标记为"已揭示"

视觉渲染（FogOverlay3D）:
  - 未探索: 绘制神话生物贴图（按主导地形选择）
  - 已揭示: 正常绘制地形，但不显示动态实体
  - 当前视野: 正常绘制一切（含 AI 实体动画）

存档:
  - explored_grid 随存档保存，探索进度永久保留
```

## 八、Chunk 流式生成（详见 `26-Chunk流式生成与遭遇模拟.md`）

大地图采用 Chunk 流式生成：
- Chunk = 16×16 hex
- 加载半径 = 2（19 个 chunk 同时活跃）
- 玩家远离的 chunk 进入休眠，存档只存差异
- 遭遇按 chunk 槽位条件触发（不模拟全局 AI）

## 十、设计原则速查

写作或扩展本文档时遵循：

1. **不写绝对方位**：用"中央农业区"而非"中部平原"，用"高山带"而非"北域山脉"
2. **不写固定地名**：举例时永远标注【示例】或"程序化命名为'XXX'"
3. **不写固定数量**：用"若干"或"1~3 个"而非"4 个王国"
4. **写约束而非实例**：写"必有至少 1 个雪山龙巢"而非"霜冠峰有古龙"
5. **数据来源指向代码**：当一个设定能在 `RegionRegistry.cs` 等代码里查到时，文档应指向代码而非复制数值
