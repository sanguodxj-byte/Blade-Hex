# 战略大地图程序化生成 (PCG) 架构方案

为了在六边形大地图上实现类似《Minecraft》或《文明》系列那样真实、自然且具备探索价值的广袤世界，我们需要摒弃单一的随机噪声，转而采用基于**物理气候模型（多层复合噪声）**的生成体系。

本方案旨在设计一套完整的战略层大地图生成逻辑，确保雪地、沙漠、森林、山脉和河流不仅存在，而且它们的分布遵循地理常识（例如：沙漠不会直接和雪地接壤，河流从高山流向大海）。

---

## 1. 核心模型：多层噪声与气候矩阵

要生成真实的生物群系（Biome），我们需要至少生成三张不同基底的 2D 噪声图（使用 `FastNoiseLite` 的 Simplex 或 Perlin 噪声），并将它们叠加计算。

### 1.1 三大基础特征图 (Feature Maps)

1. **高程图 (Elevation Map)**
   - 决定地势的高低。
   - **分层**：深水 < 浅水 < 沿海平地 < 内陆平原 < 丘陵 < 高山 < 雪峰。
   - **海岛化处理 (Falloff)**：为了让世界是一个被海洋包围的大陆，需要将高程图与一张中心高、四周低的渐变遮罩（Radial Falloff Map）相乘，迫使地图边缘沉入海底。

2. **温度图 (Temperature Map)**
   - 决定气候的冷暖。
   - **纬度影响**：以地图中轴线为赤道（最热），向南北两极递减，再叠加噪声产生自然的不规则波动。高海拔地区（来自高程图）会自动降低温度。
   - **分层**：极寒 < 寒冷 < 温带 < 炎热。

3. **湿度图 (Moisture/Rainfall Map)**
   - 决定植被的茂密程度。
   - **分层**：干旱 < 干燥 < 湿润 < 潮湿。

### 1.2 惠特克生物群系矩阵 (Whittaker Biome Diagram)

在确定了一个格子的（温度，湿度）后，我们通过查表来决定它的地表植被，最后结合高程决定最终瓦片类型。

**查表逻辑示例：**

| 温度 \ 湿度 | 干旱 (0-0.25) | 干燥 (0.25-0.5) | 湿润 (0.5-0.75) | 潮湿 (0.75-1.0) |
| :--- | :--- | :--- | :--- | :--- |
| **炎热** (0.75-1.0) | 荒漠 (Desert) | 稀树草原 (Savanna) | 丛林 (Jungle) | 雨林/沼泽 (Swamp) |
| **温带** (0.35-0.75) | 荒原 (Wasteland) | 草原 (Grassland) | 森林 (Forest) | 茂密森林 (Dense Forest)|
| **寒冷** (0.15-0.35) | 岩石荒地 (Rocky) | 针叶林 (Taiga) | 针叶林 (Taiga) | 冻土沼泽 (Bog) |
| **极寒** (0.0-0.15) | 冰原 (Ice) | 雪地 (Snow) | 雪地 (Snow) | 极地雪原 (Snow) |

**结合高程最终定型：**
- 高程 > 0.85：如果温度极寒则是“雪山 (Snow Mountain)”，否则是“裸岩高山 (Mountain)”。
- 高程在 0.6~0.85 之间：转为对应气候的“丘陵 (Hills)”。
- 高程 < 0.2：深水区 (Deep Water)。
- 高程在 0.2~0.25：浅水区/近海 (Shallow Water)。

## 2. 线性地理特征生成

地形铺好后，需要加入贯穿大陆的线状地形。

### 2.1 真实的河流系统 (River Networks)
河流的生成对于地图的视觉表现至关重要。我们采用**“贝塞尔地形雕刻 (Bezier Terrain Carving)”**算法，以兼顾河流形态的艺术美感与地理的合理性。

**贝塞尔地形雕刻算法：**
1. **寻找起止点**：在内陆高海拔区域（山脉/高丘陵）随机选择作为“源头”，在海岸线或浅水区选择作为“入海口”。
2. **生成控制点**：在源头和入海口之间，随机生成 1 到 2 个带有横向偏移的控制点，构建一条贯穿大陆、蜿蜒优美的 S 型或 U 型三次贝塞尔曲线。
3. **坐标光栅化**：沿着生成的贝塞尔曲线进行等距采样，将连续的二维浮点坐标转换为离散的六边形网格坐标 (Axial Coordinates)。
4. **地形雕刻 (Carving)**：
   - 将曲线上采样的六边形瓦片强制转换为 `RIVER` 瓦片，并**强制降低其高程**到极低值。
   - **河谷平滑**：不仅修改河流本身，还要对河流两侧的相邻瓦片进行降级雕刻（例如山脉强制降为丘陵，丘陵降为平原，平原可能变为沙滩），在视觉和数值上硬生生在大陆上“劈”出一条峡谷。
   - 这种做法使得河流永远流淌在“低地”之中，完美吻合河流侵蚀的地理常识，同时在俯视视角下拥有无可匹敌的优美弧线。

### 2.2 道路系统 (Road Networks)
- **生成时机**：必须在城镇（POI）生成之后进行。
- **算法**：使用带权重的 A* 寻路连接各个主要城镇。
- **寻路偏好**：
  - 极度偏好平原、草地、荒漠等平坦地形（移动消耗极低）。
  - 排斥穿过高山和深水。
  - 遇到河流时，产生高昂的代价（模拟造桥），但一旦跨过，将该点标记为“桥梁”瓦片。

## 3. 地图平滑与修饰 (Smoothing & Polish)

程序生成的噪声难免会产生一些“孤岛像素”（例如一大片雪地中莫名其妙出现一格沙漠）。

### 3.1 元胞自动机平滑 (Cellular Automata Smoothing)
- 遍历全图，如果一个瓦片周围 6 个邻居中有 5 个都是另一种地形，则强制同化该瓦片。这能抹平极度突兀的单格异常。

### 3.2 海岸线修饰 (Beach Generation)
- 在大陆生成完毕后，遍历所有紧邻浅水区（Shallow Water）的陆地瓦片。
- 如果它是草地或平原，有较高概率将其替换为“沙滩 (Sand)”，形成自然柔和的海岸线过渡。

---

## 4. POI 与阵营分布逻辑

大地图不仅要有好看的风景，还得合理地散布村庄、城堡和地下城。

### 4.1 聚落分布 (Settlement Distribution)
- **繁荣区**：人类或主流种族的城镇和村庄总是倾向于生成在**温带、临近水源（河流/海岸）且平坦**的草地或平原上。
- **险要区**：城堡/要塞倾向于生成在丘陵或者扼守狭窄山谷隘口的战略位置。

### 4.2 敌对巢穴分布 (Lair Distribution)
- **哥布林营地**：喜欢生成在温带森林的边缘。
- **兽人部落**：倾向于生成在严酷的荒原、沙漠或雪地。
- **亡灵墓穴**：绝对不会生成在靠近城镇的地方，通常位于地图边缘的沼泽或荒漠废墟中。
- **龙巢**：只生成在最高峻、最难以到达的山脉深处。

## 5. 代码流水线执行顺序 (Execution Pipeline)

在接下来的重构中，`HexOverworldGenerator.gd` 的 `generate()` 方法应重构为以下严谨的步骤：

```gdscript
func generate() -> HexOverworldGrid:
    var grid = HexOverworldGrid.new()
    
    # 1. 基础物理层
    _generate_elevation(grid)    # 高程与大陆轮廓
    _generate_temperature(grid)  # 纬度温度与高海拔降温
    _generate_moisture(grid)     # 湿度噪声
    
    # 2. 地表类型映射
    _apply_whittaker_biomes(grid) # 查表得出基础 TerrainType
    
    # 3. 修饰层
    _smooth_terrain(grid)        # 抹平单格孤岛
    _generate_beaches(grid)      # 柔化海岸线
    
    # 4. 水系
    _carve_bezier_rivers(grid)   # 贝塞尔地形雕刻生成优美河流
    
    # 5. 区域划分与 POI
    _define_regions(grid)        # 划分势力板块
    _place_settlements(grid)     # 放置城镇与村庄
    
    # 6. 交通网络
    _build_roads(grid)           # 连接城镇的道路
    
    # 7. 隐秘地点
    _place_dungeons_and_lairs(grid) # 放置怪物巢穴
    
    return grid
```

通过引入气候模型和自然物理模拟（水流下溯），生成的地图将彻底摆脱目前“一片混乱拼凑”的视觉感，呈现出像 Minecraft 一样大片连续的雪原、广袤的沙漠、连绵的山脉和蜿蜒入海的江河。
