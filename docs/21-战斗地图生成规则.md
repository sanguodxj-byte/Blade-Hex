# 战斗地图生成规则

> 📐 距离 / 尺寸换算见 [31-比例尺与距离体系.md](./31-比例尺与距离体系.md)（SSOT）。
> 战斗规模档位 + 战斗 cell 数 = `1 + 3·N·(N+1)`，六边形地图。

## 概述

战斗地图是战术战斗的舞台。每场战斗的地图不是手动设计的关卡，而是由 `BattleMapGenerator` 根据大地图遭遇环境或模板权重**程序化生成**的。本设计确保每场战斗的地图有足够的变化性，同时忠实反映遭遇发生的大地图地形。

当前代码基础：`BattleMapGenerator.gd` 已完整实现（~1392行），包含两种生成路径、模板系统、地形分布算法和部署区生成。

---

## 一、生成路径

### 1.1 路径A：大地图驱动（主路径）

当战斗在大地图上触发时（从 OverworldScene 切入），地图生成器从 `HexOverworldGrid` 采样遭遇点周围的地形，将其展开为战术级别的六边形网格。

```
大地图遭遇点坐标 (ow_x, ow_y)
    │
    ▼
采样半径 R 内的所有大地图格子
    │
    ▼
每个大地图格子 → 映射为战术地图的一组格子（放大）
    │
    ▼
保留地形类型、高程、河流、道路
    │
    ▼
添加部署区和装饰
```

### 1.2 路径B：模板随机（备用路径）

当没有大地图数据时（测试场景/QuickCombat），从预定义的模板库中按权重随机选择一个模板，再在模板范围内随机变化。

## 二、战斗规模与地图尺寸

> 详见 [31-比例尺与距离体系.md](./31-比例尺与距离体系.md)。本节只列与本系统相关的快速对照。

战斗地图为**六边形**，半径 N 决定 cell 数 = `1 + 3·N·(N+1)`：

| 规模（BattleSize） | 阶段 | 战斗 hex 半径 N | cell 数 | 实际宽度 (1 cell ≈ 20m) |
|---------------------|------|----------------|---------|------------------------|
| MERCENARY | 雇佣兵 | 7 | 169 | 300 m |
| KNIGHT | 骑士 | 8 | 217 | 340 m |
| LORD | 领主 | 11 | 397 | 460 m |
| STRONGHOLD | 攻城 / boss | 14 | 631 | 580 m |

部署区上限：MERCENARY ≈ 6 单位 / 边，KNIGHT ≈ 4+4，LORD ≈ 4+7，STRONGHOLD ≈ 攻防双方按场景定。

> Feature flag：`BattleMapGenerator.UseHexagonalShape = false` 可回退到旧矩形 W×H（仅调试用）。

## 四、地形分布算法

### 4.1 主地形填充

每个模板定义一个**主地形**和占比权重：

```
步骤1: 用主地形填充整个地图（如 55% 为森林）
步骤2: 按权重随机分配次要地形（如 20% 平地, 15% 丘陵, 10% 道路）
步骤3: 平滑过渡（2次迭代）：遍历每个格子，如果与邻居不同，按概率同化
```

### 4.2 地形聚集特征

为避免地形分布过于随机（棋盘格效应），生成器使用**簇生成**算法：

| 特征 | 代码参数 | 说明 |
|------|----------|------|
| 树木簇 | `tree_cluster_count: Vector2i(2,5)` | 2-5个树木聚落 |
| 树木簇半径 | `tree_cluster_radius: Vector2i(2,4)` | 每个聚落半径2-4格 |
| 密林核心 | `dense_tree_core_chance: 0.3` | 30%概率在树木簇中心生成密林 |
| 废墟结构 | `ruin_structure_count: Vector2i(1,3)` | 1-3处废墟 |
| 墙壁段 | `wall_segment_count: Vector2i(0,2)` | 0-2段独立墙壁 |
| 墙壁长度 | `wall_segment_length: Vector2i(2,5)` | 每段2-5格长 |

### 4.3 高程生成

高程由模板的 `elevation_bias` 参数控制：

| 偏置值 | 地形倾向 |
|--------|----------|
| -0.5 | 低地为主（沼泽/河流地形） |
| 0.0 | 混合高程 |
| +0.5 | 高地为主（丘陵/山地地形） |

高程分配规则：
- 丘陵/山地格子 → 平地或高地
- 其他格子 → 低地或平地
- 部署区保证在低地或平地

| forest_ambush | 森林 | 密林、平地 | 密林核心多 | 无 |

| swamp_fight | 沼泽 | 浅水、平地 | 无 | 暴风雨 |
| river_crossing | 平地 | 浅水、道路 | 河流 | 无 |
| mountain_pass | 山地 | 丘陵、道路 | 墙壁段 | 无 |

### 5.2 模板数据结构

```gdscript
class BattleMapTemplate:
    var template_name: String
    var primary_terrain: BattleCellData.TerrainType   # 主地形
    var primary_weight: float                         # 主地形占比
    var secondary_terrains: Dictionary                # TerrainType → weight
    var has_river: bool                               # 是否有河流
    var has_road: bool                                # 是否有道路
    var elevation_bias: float                         # 高程偏置
    var special_features: Array                       # 特殊地形
    var environment_event: String                     # 环境事件ID
    var tree_cluster_count: Vector2i                  # 树木簇数量范围
    var tree_cluster_radius: Vector2i                 # 树木簇半径范围
    var dense_tree_core_chance: float                 # 密林核心概率
    var ruin_structure_count: Vector2i                # 废墟数量范围
    var wall_segment_count: Vector2i                  # 墙壁段数量范围
    var wall_segment_length: Vector2i                 # 墙壁段长度范围
    var smoothing_passes: int                         # 平滑迭代次数
```

## 六、部署区生成

### 6.1 部署规则

每张战斗地图包含两个部署区（玩家/敌方），由 `DeploymentZone` 类管理：

| 规则 | 说明 |
|------|------|
| 位置 | 玩家在地图一侧，敌方在对面 |
| 距离 | 两方部署区边缘最少相隔4格（MERCENARY）~ 6格（LORD） |
| 大小 | 根据单位数量动态调整 |
| 地形限制 | 部署区内不可有墙壁、深水、不可通行地形 |
| 对称性 | 非对称部署（大地图驱动时），模板随机时可选对称 |

### 6.2 部署区类型

| 类型 | 说明 | 触发条件 |
|------|------|----------|
| 标准部署 | 双方各在一侧 | 正常遭遇 |
| 包围部署 | 玩家在中央，敌方四面 | 被伏击/被包围 |
| 伏击部署 | 伏击方散布在地图各处，被伏击方集中 | 伏击事件 |
| 攻城部署 | 攻方在一侧，守方在城墙后 | 攻城战（Phase 5） |
| 护送部署 | 护送目标在中间，护卫在两侧 | 护送任务 |

### 6.3 伏击部署的特殊规则

| 被伏击方 | 伏击方 |
|----------|--------|
| 集中在地图中央小区域 | 散布在地图边缘和隐蔽位置 |
| 首回合全员"措手不及"（AC-2） | 首回合可正常行动 |
| 阵型混乱（随机朝向） | 已占据有利地形 |

## 七、特殊战场

### 7.1 室内战斗（矿洞/遗迹/墓穴）

| 规则 | 说明 |
|------|------|
| 墙壁 | 大量墙壁格子限制视线和移动 |
| 无高程 | 室内无高程差 |
| 狭窄通道 | 强制瓶颈，限制战术展开 |
| 无环境事件 | — |
| 无飞行 | 室内禁止飞行单位 |
| 无骑乘 | 室内禁止骑乘 |

### 7.2 攻城战（Phase 5）

| 规则 | 说明 |
|------|------|
| 地图特征 | 城墙、城门、箭塔 |
| 守方部署 | 城墙后+箭塔上 |
| 攻方部署 | 地图一侧远处 |
| 攻城器械 | 攻城锤、云梯、投石车 |
| 破坏性 | 城墙可被破坏 |

## 八、大地图采样细节

### 8.1 采样半径

| 战斗规模 | 大地图采样半径 | 说明 |
|----------|---------------|------|
| MERCENARY | 2格 | 小范围，地形较单一 |
| KNIGHT | 3格 | 中等范围 |
| LORD | 4格 | 大范围，地形可能多变 |

### 8.2 采样映射

```
大地图格子 (ow_x, ow_y) → 战术地图格子 (t_x, t_y)

映射规则:
    每个大地图格子 → 展开为 tactical_cells_per_ow × tactical_cells_per_ow 的区块
    具体数值取决于战斗规模和采样半径

    MERCENARY (12×10 地图, 2格采样):
        采样区域 ≈ 5×5 = 25个大地图格
        每个大格 → ~5个战术格
        25 × 5 = 125 ≈ 120格 (12×10)

    LORD (20×16 地图, 4格采样):
        采样区域 ≈ 9×9 = 81个大地图格
        每个大格 → ~4个战术格
        81 × 4 = 324 ≈ 320格 (20×16)
```

### 8.3 地形保真度

| 地图元素 | 是否保留 | 说明 |
|----------|----------|------|
| 基础地形 | ✅ | 森林→森林，山地→山地 |
| 高程 | ✅ | 低地/平地/高地映射 |
| 河流 | ✅ | 浅水/深水格子 |
| 道路 | ✅ | 道路格子 |
| 建筑物 | 部分 | 村庄→废墟格子，城堡→墙壁格子 |
| 其他实体 | ❌ | AI实体/玩家不映射到战术地图 |
