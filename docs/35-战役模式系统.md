# 战役模式系统

## 概述

战役模式是独立于大地图战略层的**线性关卡挑战模式**。玩家率领一支雇佣兵团，依次挑战10个预设关卡，每关之间可进行部队整备、雇佣、商店交易和医官救治。

**核心定位**：
- 无需大地图，直接体验战斗核心玩法
- 适合短时间游戏Session（单关10-20分钟）
- 作为大地图模式的"快速战斗"替代入口

**设计理念**：
- 每关都是独立的战术挑战
- 队伍成长贯穿全程（等级、装备、人员）
- 经济压力驱动决策（雇佣vs救治vs装备）
- 死亡/重伤有代价但不致命（重伤系统）

## 二、准备阶段

每关开始前进入准备阶段，玩家可进行以下操作：

### 2.1 界面布局

```
┌─────────────────────────────────────────────────────┐
│  关卡 3/10 — 荒野猎场     [描述文本]      💰 850    │  ← 顶栏
├─────────────────────────────────────────────────────┤
│                                                     │
│              [关卡插图 - 半透明背景]                  │  ← 中部
│                                                     │
├─────────────────────────────────────────────────────┤
│  [返回] [管理部队] [商店] [雇佣] [技能盘] [医官] [⚔开始] │  ← 底栏
└─────────────────────────────────────────────────────┘
```

### 2.2 功能模块

| 按钮 | 功能 | 说明 |
|------|------|------|
| 管理部队 | 打开PartyPanel | 完整的装备/属性/技能盘管理 |
| 商店 | 打开战役商店 | 买卖装备，繁荣度随关卡提升 |
| 雇佣 | 打开雇佣面板 | 招募新佣兵补充队伍 |
| 技能盘 | 打开技能树 | 管理角色技能点分配 |
| 医官治疗 | 打开救治面板 | 治疗重伤单位 |
| 开始战斗 | 进入战斗 | 加载战役战斗场景 |

### 2.3 角色命名（首关）

首关开始时，玩家需要输入：
- **队长名字**：玩家主角，死亡则战役失败
- **副官名字**：初始队友

系统生成2个1级随机角色，并配发初始装备。

## 三、队伍系统

### 3.1 PartyRoster（复用大地图）

战役模式复用大地图的 `PartyRoster` 系统，管理队伍成员：

| 属性 | 说明 |
|------|------|
| Leader | 队长（玩家主角），死亡=战役失败 |
| Members | 所有成员列表（含队长） |
| Capacity | 队伍上限（默认6） |

### 3.2 成员状态

| 状态 | 条件 | 效果 |
|------|------|------|
| 正常 | HP > 0 且 IsWounded = false | 可出战 |
| 重伤 | IsWounded = true | 不可出战，需医官救治 |
| 死亡 | 队长HP=0 | 战役失败 |

### 3.3 重伤机制

**触发条件**：战斗中HP降至0（非队长）

**效果**：
- 标记 `IsWounded = true`
- HP设为0
- 下一关**不可部署出战**
- 需要花费金币请医官救治

**设计意图**：
- 惩罚存在但不永久（不像大地图的永久死亡）
- 鼓励玩家保护所有队员
- 创造经济决策（救治vs雇佣新人）

### 3.4 队伍上限

默认6人。玩家可通过以下方式扩充：
- 雇佣新佣兵（金币消耗）
- 队伍满时无法雇佣

## 四、雇佣系统

### 4.1 可雇佣角色生成

每关开始时生成2-4个可雇佣角色：

```
雇佣角色等级 = 当前关卡敌方等级 - 1（至少1级）
装备等级 = 按CR换算的装备等级
难度系数 = 关卡难度（easy/normal/hard）
```

### 4.2 雇佣费用

由 `CampaignPricingService.GetHireCost()` 计算：

```
费用 = RecruitPricingService.GetRecruitCost(unit, poiTier, prosperity)
```

其中：
- `poiTier` = BattleSize + Difficulty（0-3）
- `prosperity` = 35 + 关卡×6 + 难度×8 + 规模×4

**费用特点**：
- 等级越高越贵
- 装备越好越贵
- 关卡推进后费用上升

### 4.3 雇佣流程

1. 点击"雇佣"按钮打开面板
2. 查看可雇佣角色列表（名字/种族/等级/属性/装备/费用）
3. 点击"雇佣"按钮
4. 扣除金币，角色加入队伍
5. 队伍满时按钮置灰

## 五、商店系统

### 5.1 商店繁荣度

由 `CampaignPricingService.GetShopProsperity()` 计算：

```
繁荣度 = 35 + 关卡×6 + 难度×8 + 规模×4 + (Boss?8:0)
范围: 30-95
```

**繁荣度影响**：
- 物品买卖价格
- 库存品质门槛
- 随关卡推进逐步提升

### 5.2 商店库存

由 `MarketStockService.GenerateStock(prosperity)` 生成：
- 繁荣度越高，物品品质越好
- 包含武器、防具、消耗品

### 5.3 价格计算

复用大地图的 `TradePricingService`：
- 买入价 = 基础价 × 繁荣度系数
- 卖出价 = 基础价 × 0.5

## 六、医官救治系统

### 6.1 救治费用

由 `CampaignPricingService.GetMedicTreatmentCost()` 计算：

```
费用 = 可支配金币 × 周期系数 + HP加成
周期系数 = 1.4 + 等级×0.45 + 难度×0.30
HP加成 = max(0, 最大HP-10) × 1.5
最低费用: 60金币
```

**设计意图**：
- 救治费用随等级和关卡难度上升
- 高HP角色救治更贵
- 创造"救治旧部vs雇佣新人"的决策

### 6.2 救治流程

1. 点击"医官治疗"按钮
2. 查看重伤角色列表
3. 点击"救治"按钮
4. 扣除金币，角色恢复满HP
5. IsWounded设为false

## 七、战斗结算

### 7.1 胜利结算

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 经验奖励 | 所有存活角色获得XP |
| 2 | HP恢复 | 存活角色恢复30%最大HP |
| 3 | 重伤标记 | HP=0的非队长角色标记IsWounded |
| 4 | 金币奖励 | 按关卡配置计算金币 |
| 5 | 升级检查 | 检查并应用升级 |
| 6 | 推进关卡 | CurrentLevel++ |

### 7.2 经验奖励

由 `CampaignPricingService.GetBattleXpReward()` 计算：

```
基础XP = 敌方等级×24 + 敌方数量×14
乘数 = 1.0 + 难度×0.20 + 规模×0.08 + (Boss?0.35:0)
最终XP = max(25, 基础XP × 乘数)
```

### 7.3 金币奖励

由 `CampaignPricingService.GetBattleGoldReward()` 计算：

```
基础奖励 = 平均委托奖励
规模系数 = 0.65 + 等级×0.10 + 数量×0.075 + 难度×0.25 + 规模×0.12
Boss加成 = Boss?1.35:0
最终奖励 = max(60, 基础奖励 × (规模系数 + Boss加成))
```

### 7.4 失败结算

- 队长HP=0 → 战役失败
- 延迟2秒后返回主菜单
- 战役状态重置

## 八、存档系统

### 8.1 检查点保存

**保存时机**：每关开始前（进入准备阶段时）

**保存路径**：`user://campaign_save.json`

**保存内容**：
```json
{
  "current_level": 3,
  "gold": 850,
  "is_active": true,
  "roster": { /* PartyRoster序列化 */ },
  "inventory": { /* PartyInventory序列化 */ },
  "equipment": [ /* 每个成员的装备详情 */ ]
}
```

### 8.2 装备序列化

由于PartyRoster不序列化装备，战役模式额外保存：

| 字段 | 说明 |
|------|------|
| primary_main | 主武器 |
| secondary_main | 副武器 |
| primary_off | 副手 |
| armor | 护甲 |
| helmet | 头盔 |
| boots | 靴子 |
| gauntlets | 手套 |
| shield | 盾牌 |

### 8.3 存档加载

**加载时机**：从主菜单选择"继续战役"

**加载流程**：
1. 检查存档文件是否存在
2. 解析JSON
3. 恢复关卡进度、金币、队伍、装备
4. 进入对应关卡的准备阶段

## 十、与大地图模式的关系

### 10.1 共享系统

- 战斗逻辑（CombatManager/CombatResolver）
- 角色属性/装备系统
- 技能树系统
- UI组件（PartyPanel/CombatUI）

### 10.2 独立系统

| 维度 | 大地图 | 战役模式 |
|------|--------|----------|
| 进度 | 自由探索 | 线性关卡 |
| 经济 | 日薪+食物 | 无关卡间消耗 |
| 死亡 | 永久死亡 | 重伤可救治 |
| 存档 | 完整存档 | 检查点存档 |
| 地图 | 程序生成 | 预设模板 |

### 10.3 设计决策

**为什么复用PartyRoster而不是新建数据结构？**
- 减少代码重复
- 统一装备/属性计算逻辑
- 方便未来扩展（如战役→大地图的队伍导入）

**为什么独立经济系统？**
- 大地图有日薪/食物压力，战役不需要
- 战役节奏更快，经济更简单
- 避免大地图经济复杂度影响战役体验

## 十二、已知限制与未来扩展

### 12.1 当前限制

- 关卡固定10关，无随机性
- 无难度选择（固定曲线）
- 无成就系统
- 无多周目继承
- 敌方无技能盘（仅人形有）

### 12.2 可能的扩展

| 扩展方向 | 说明 |
|----------|------|
| 随机战役 | 每次随机生成关卡顺序/配置 |
| 难度选择 | 简单/普通/困难/噩梦 |
| 成就系统 | 通关条件/挑战目标 |
| 周目继承 | 二周目保留部分进度 |
| 支线关卡 | 可选的额外挑战 |
| Boss技能 | Boss关的独特机制 |
| 环境效果 | 特殊关卡的环境事件 |
