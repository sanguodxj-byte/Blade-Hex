# 特殊角色生成系统设计

## 1. 概述

世界初始化阶段（WorldCreator Stage 7.5）生成一批具有独立身份的特殊角色：**领主**和**冒险者**。这些角色在生成后立即收容到 `DormantEntityPool`，在运行时按需激活。

### 设计目标
- 世界感：每个国家有具名领主，玩家能在大地图上遇到有名有姓的 NPC
- 复用性：冒险者从池中复用，避免每次遭遇都是无名路人
- 成长性：领主和冒险者有等级和属性成长，随游戏天数推进

---

## 2. 角色分类

### 2.1 领主 (Lord)

| 属性 | 说明 |
|------|------|
| 绑定 | 每个领主绑定一个 POI（城堡/城镇） |
| 数量 | 每个主要国家 3~5 个，小势力 1~2 个 |
| 等级 | 初始 40~80 级，随游戏天数缓慢成长 |
| 行为 | 巡逻领地、回援被攻击的 POI、发动战争 |
| 命名 | 按 `naming_logic_design.md` 种族规则生成 |
| 性格 | Cautious / Balanced / Aggressive（影响 AI 决策） |

### 2.2 冒险者 (Adventurer)

| 属性 | 说明 |
|------|------|
| 绑定 | 无固定绑定，自由漫游 |
| 数量 | 世界初始 12~60 个（按世界大小缩放） |
| 等级 | 初始 5~80 级，随存活天数成长 |
| 行为 | 在城镇间移动、接取任务、与玩家遭遇时可交谈/交易/决斗 |
| 命名 | 按种族冒险者命名规则生成 |
| 类型 | Novice / Veteran / Elite / Outlaw |

## 3. 属性与成长规则

### 3.1 基础属性体系（SSOT: docs/战斗数值体系.md）

- 满级：**120 级**
- 6 项属性：STR, DEX, CON, INT, WIS, CHA
- 初始 25 点分配，每级 +1 点，单属性上限 40
- 专精加值：+2（1级）→ +10（97-120级）
- 伤害额外骰：1-20级 +0d20, 21-40级 +1d20, ..., 101-120级 +5d20

### 3.2 领主属性模板（初始等级 40~80）

领主是各国的军事/政治领袖，初始等级远高于玩家（玩家 1 级起步）。

| 种族 | 初始等级 | STR | DEX | CON | INT | WIS | CHA | 说明 |
|------|---------|-----|-----|-----|-----|-----|-----|------|
| 人类 | 40~80 | 18 | 14 | 16 | 14 | 16 | 20 | 均衡偏魅力（统帅） |
| 精灵 | 50~80 | 12 | 22 | 12 | 22 | 18 | 16 | 敏捷+智力（长寿=高等级） |
| 矮人 | 40~70 | 22 | 10 | 24 | 12 | 16 | 12 | 力量+体质（坚韧） |
| 兽人 | 30~60 | 26 | 14 | 22 | 8 | 10 | 14 | 力量+体质（蛮力） |

注：属性值 = 基础模板 + 等级带来的额外点数（每级 +1 分配到优势属性）

### 3.3 冒险者属性模板（初始等级 5~80）

冒险者等级分布更广，模拟世界中不同实力的旅行者。

| 类型 | 初始等级 | 属性倾向 | 说明 |
|------|---------|---------|------|
| Novice（新手） | 5~24 | 均匀分配 | 刚出道的冒险者 |
| Veteran（老手） | 25~50 | 偏向 STR/DEX | 有经验的战士/游侠 |
| Elite（精英） | 50~80 | 明显偏科 | 专精某一方向的强者 |
| Outlaw（亡命） | 20~60 | 偏向 DEX/CHA | 通缉犯/赏金猎人 |

### 3.4 等级成长

```
每日成长概率：
  领主：0.5% / 天（约 200 天升 1 级）
  冒险者：1.5% / 天（约 67 天升 1 级）

等级上限：
  领主：120 级（满级）
  冒险者：100 级

每次升级：
  +1 属性点（分配到种族优势属性，概率 60% 优势 / 40% 随机）
  战力重算
  达到 5 级倍数时检查称号获取（按 naming_logic_design.md §8 规则）
```

### 3.5 战力计算

```
CombatPower = BaseStatPower + LevelPower + ArmyPower

BaseStatPower = (STR + DEX + CON) * 0.8
LevelPower = Level * Level * 0.1 + Level * 3  // 二次曲线，高等级差距拉大
ArmyPower = GarrisonSize * 2.0  // 领主专属

示例：
  30 级人类领主 (STR18+DEX14+CON16=48): 48*0.8 + 30*30*0.1 + 30*3 + 50*2 = 38.4 + 90 + 90 + 100 = 318
  10 级新手冒险者 (STR12+DEX14+CON12=38): 38*0.8 + 10*10*0.1 + 10*3 = 30.4 + 10 + 30 = 70
```

## 4. 生成数量缩放

| 世界大小 | 领主数 | 冒险者数 | 总计 |
|----------|--------|----------|------|
| 小型 (65k) | 15~22 | 16~20 | 31~42 |
| 中型 (221k) | 15~22 | 35~40 | 50~62 |
| 大型 (459k) | 15~22 | 60~70 | 75~92 |

计算公式：
```
lordCount = nations.Count(isMajor) * [3~5] + nations.Count(!isMajor) * [1~2]
adventurerCount = max(12, worldTileCount / 8000 + 8)
```

## 5. 生命周期

```
WorldCreator.CreateWorld()
  → Stage 7.5: GenerateSpecialCharacters()
      → 为每个有领土的国家生成领主
      → 生成自由冒险者
      → 全部存入 DormantEntityPool

运行时：
  → 玩家进入领主领地 → 从池中激活领主实体
  → 遭遇系统需要冒险者 → 从池中复用（而非新建）
  → 领主/冒险者离开活跃区域 → 收容回池
  → 每日结算 → 池中角色也参与成长判定
```

## 6. 领主与战争事件

### 6.1 领主 AI 行为

| 状态 | 触发条件 | 行为 |
|------|---------|------|
| Patrolling | 默认 | 在绑定 POI 周围巡逻 |
| Reinforcing | 己方 POI 被攻击 | 带兵前往回援 |
| Attacking | 性格=Aggressive + 敌方弱 | 主动进攻敌方 POI |
| Recruiting | 兵力 < 50% | 返回城镇招募 |
| Besieging | 到达敌方 POI | 发起围攻 |

### 6.2 领主死亡/俘虏

- 领主战败：50% 逃跑（返回首都），50% 被俘
- 被俘领主：30 天后自动赎回（或玩家可交涉释放）
- 领主死亡：该国在 60 天后生成继承者（同家族姓，新名）

## 7. 数据结构扩展

### OverworldEntity 新增字段

```csharp
// 特殊角色标识
[Export] public bool IsNamedCharacter { get; set; } = false;
[Export] public string CharacterTitle { get; set; } = ""; // 称号
[Export] public string FamilyName { get; set; } = "";     // 家族姓

// 领主专属
[Export] public string BoundPoiName { get; set; } = "";   // 绑定的 POI 名称
[Export] public int RecruitCooldownDays { get; set; } = 0;

// 成长
[Export] public int LastLevelUpDay { get; set; } = 0;     // 上次升级的游戏天数
```

## 8. 实现步骤

1. 在 `WorldCreator` 中添加 Stage 7.5: `GenerateSpecialCharacters()`
2. 实现 `LordGenerator` — 按国家生成领主，绑定 POI
3. 实现 `AdventurerGenerator` — 生成自由冒险者
4. 扩展 `DormantEntityPool` — 支持按名称查找特定角色
5. 扩展 `OverworldEntityManager.OnDayPassed()` — 池中角色成长判定
6. 扩展 `EncounterEntitySpawner` — 冒险者遭遇优先从池中取具名角色
