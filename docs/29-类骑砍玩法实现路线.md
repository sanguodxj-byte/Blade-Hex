# 类骑砍/战场兄弟玩法实现路线

> 目标：从当前可跑通的"大地图 + 战斗原型"状态，迭代到完整的"雇佣兵团生意经"循环。
> 
> 单一玩法目标：**带一支雇佣兵团接活、打仗、招人、发饷、买装备、被歼灭、再招募**。

---

## 阶段总览

每个阶段都以**该阶段结束时玩家能做什么**作为切片，避免"大重构 6 个月不能玩"的陷阱。

```
阶段 0: 当前状态（已完成）
  ├─ 战术战斗原型可玩
  ├─ Chunk 大地图可生成
  └─ 玩家在母国出生

阶段 1: 雇佣兵团核心数据   → 可看到自己的队伍清单、能死人
阶段 2: 大地图战斗闭环     → 大地图遭遇 → 带队战斗 → 战利品/伤亡回灌
阶段 3: 招募系统            → 城镇招募新兵补充损失
阶段 4: 经济压力             → 工资/食物压迫感，逼着接活
阶段 5: 委托循环             → 完整的接委托→打→领赏闭环
阶段 6: 世界活力             → 实体真在动（村被袭、商队跑、领主出征）
阶段 7: 装备库存             → 战利品装到队员身上变强
阶段 8: 声望与外交           → 玩家行为影响国家关系、招募权限
阶段 9: 永久化与抛光         → 存档完整、UI 抛光、平衡数值
```

阶段 1-5 是 **MVP 必要项**，缺一项都跑不通骑砍循环。
阶段 6-9 是 **品质项**，让游戏从"能玩"变成"想玩"。

---

## 阶段 1：雇佣兵团核心数据（MVP 基石）

**结束时玩家能做：**
- 大地图角色信息面板里看到"队伍 X 人"，每人姓名、种族、等级、HP
- 战斗中我方部署的就是这些队员（不是凭空生成的）
- 战斗死亡 → 回大地图后清单里少一个

### 任务

#### T1-1：`PartyRoster` 数据类
- **文件**：`BladeHexCore/src/Strategic/Party/PartyRoster.cs`
- **职责**：持有 `List<UnitData> Members`，提供 Add/Remove/Get/Count/Serialize
- **字段**：
  - `string Leader` — 队长 unit_id（玩家本人）
  - `List<UnitData> Members` — 队员（含队长）
  - `int Capacity` — 队伍上限（初始 6，随等级提升）
- **接口**：
  - `Add(UnitData unit) → bool` — 满员则失败
  - `Remove(string unitId) → bool`
  - `GetMember(string unitId) → UnitData?`
  - `IsLeader(string unitId) → bool`
  - `LivingMembers / DeadMembers` — 按 HP 过滤
  - `Serialize() / Deserialize()` — `Godot.Collections.Dictionary`
- **持有方**：`OverworldScene.PlayerParty.Roster`
- **初始化**：玩家本人作为第一个 Member 加入（来自 `BuildInitialPlayerUnit`）

#### T1-2：`OverworldParty` 持有 Roster
- **文件**：`BladeHexFrontend/src/View/Strategic/OverworldParty.cs`
- **改动**：
  - 新增 `public PartyRoster Roster { get; set; }`
  - 玩家在大地图上的视觉仍然是"一个 sprite"，但 Roster 决定战斗时部署哪些人
- **接线**：`OverworldScene.InitPlayerParty` 创建 Roster 并 `Add(PlayerUnitData)`

#### T1-3：UI 显示队伍清单
- **文件**：`src/ui/overworld/PartyPanel.gd`（已存在）
- **改动**：
  - 从 `OverworldScene.PlayerParty.Roster.Members` 取列表
  - 每行：头像 + 名字 + 种族 + 等级 + HP/MaxHP + 状态（活/伤/亡）
  - 选中后显示属性详情（已有 `CharacterDetailPanel`）

#### T1-4：战斗加载 Roster 而非凭空生成
- **文件**：`BladeHexFrontend/src/Scenes/combat/CombatScene.cs`
- **改动**：
  - 新增 `public PartyRoster? PlayerRoster` 字段
  - `_Ready()` 检查 `PlayerRoster != null` → 从 Roster 取 Members 部署到我方区
  - `PlayerRoster == null`（QuickCombat 路径）→ 走原有的随机生成
- **接线**：`OverworldScene.gd._enter_combat_scene` 把 `PlayerParty.Roster` 传给 CombatScene

#### T1-5：战斗结果回写 Roster
- **文件**：`BladeHexFrontend/src/Scenes/combat/CombatScene.cs`
- **新增**：`BuildBattleOutcome()` 方法构造 `BattleOutcome`：
  - `SurvivedUnits: List<string unitId>`
  - `DeadUnits: List<string unitId>`
  - `WoundedUnits: Dictionary<string unitId, int currentHp>`
- **回写**：战斗结束 signal `combat_finished(victory, outcome)` 增加 `outcome` 参数
- **`OverworldScene._on_combat_finished`**：
  - 把 `outcome.DeadUnits` 从 Roster 移除
  - 把 `outcome.WoundedUnits[id]` 写回对应 UnitData.CurrentHp
  - 触发 UI 刷新

#### T1-6：永久死亡确认 UI
- **新增**：`PermadeathConfirmPanel`（简单弹窗）
  - 显示阵亡名单 + "X 已经永远离开了这个世界"
  - 玩家按确认后才回大地图（也可以做成自动消失 3s）

### 验收
- [ ] 新游戏后 `Roster.Members.Count == 1`（玩家本人）
- [ ] 触发战斗 → 我方只有 1 个单位（队长）
- [ ] 战斗失败队长死亡 → 弹出"游戏结束"
- [ ] 战斗胜利队长存活但 HP 减少 → 大地图 PartyPanel 显示伤员

## 阶段 2：大地图战斗闭环

**结束时玩家能做：**
- 在大地图遇到敌方实体 → 触发战斗 → 用当前 Roster 打 → 胜利后实体消失/失败后己方损失
- 战利品 / 经验 / 金币正确结算到玩家

### 任务

#### T2-1：`BattleContext` 完整化
- **文件**：`BladeHexCore/src/Strategic/BattleContext.cs`
- **新增字段**：
  - `PartyRoster PlayerRoster` — 玩家队伍快照（不是引用，避免战斗中改变原始数据）
  - `EncounterData Encounter` — 遭遇方信息（敌方单位列表、密度、AI 行为）
  - `Vector2I OverworldCoord` — 大地图坐标，用于 BattleMapGenerator 采样
  - `string EncounterRegionName` — 区域名（影响战斗模板）

#### T2-2：`EncounterData` 数据类
- **文件**：`BladeHexCore/src/Strategic/Encounter/EncounterData.cs`
- **字段**：
  - `string EncounterId`
  - `string EnemyFaction` — 敌方阵营 id
  - `List<UnitData> Enemies` — 敌方部署单位
  - `int Difficulty` — 1-10 难度等级
  - `RewardTable Rewards` — 战利品配置
- **生成**：`EncounterSpawner` 已存在，需要补全为它产生完整 EncounterData

#### T2-3：`OverworldEntity` → `EncounterData` 转换
- **文件**：`BladeHexCore/src/Strategic/EntityToEncounter.cs`（新）
- **职责**：把大地图上的实体（哥布林营地、领主军队）转成战斗用的 `EncounterData`
- **逻辑**：
  - 实体 unit_count → 敌方数量
  - 实体 unit_template → 敌方 UnitData 模板（用 UnitTemplateDB 实例化）
  - 实体所在 tile 地形 → 战斗模板选择

#### T2-4：触发遭遇 → 进入战斗的胶水
- **文件**：`BladeHexFrontend/src/Scenes/overworld/OverworldScene.Combat.cs`
- **方法**：`EnterBattleFromEncounter(OverworldEntity entity)`
- **流程**：
  1. 从 entity 构造 BattleContext（含 PlayerRoster + EncounterData + 大地图坐标）
  2. 暂停大地图时间
  3. `_enter_combat_scene(ctx)` 切场景
- **当前 GD 代码**：`OverworldScene.gd._enter_combat_scene` 已经有壳子，需要把 PlayerRoster 注入

#### T2-5：`BattleOutcome` 完整化
- **文件**：`BladeHexCore/src/Strategic/BattleOutcome.cs`
- **新增字段**：
  - `bool Victory`
  - `int GoldReward`
  - `List<ItemData> Loot` — 战利品物品
  - `int XpReward` — 总经验值
  - `Dictionary<string, int> XpPerMember` — 每个队员经验
  - `string EncounterId` — 用于 chunk 状态写回（已击败的标记）

#### T2-6：胜利后写回大地图
- **文件**：`OverworldScene.Combat.cs._on_combat_finished`
- **流程**：
  - `outcome.Victory == true` →
    - 大地图实体从 `OverworldEntityManager` 移除（已有 `EntityRemoved` 信号）
    - `EconomyManager.AddGold(outcome.GoldReward)`
    - 战利品加入 PartyInventory（阶段 7 实现，先存到内存）
    - `outcome.XpPerMember` → 每个 Member 加 XP，触发可能的升级
  - `outcome.Victory == false` →
    - Roster 全员死亡 → 游戏结束
    - 部分死亡 → 阵亡名单弹窗 + 回大地图
- **关键**：所有写回操作经过 `BattleOutcomeApplier`（新文件）统一处理，方便存档

#### T2-7：战利品配置表
- **文件**：`BladeHexCore/src/Strategic/Encounter/RewardTable.cs`
- **数据**：
  - 按敌方阵营 + 难度等级查表
  - 每条 entry：物品 id + 概率 + 数量范围
- **驱动**：`NationConfig.UniqueResources` 提供候选物品

### 验收
- [ ] 打野怪营地胜利后营地消失 + 金币增加 + 队员升级
- [ ] 打野怪营地失败 → 失败弹窗 + 队伍减员
- [ ] 战斗中我方部署人数 = Roster 当前活着人数
- [ ] 同一个 encounter 不会被重复触发（chunk 写回成功）

## 阶段 3：招募系统

**结束时玩家能做：**
- 进城镇酒馆 → 看到一份可招募列表 → 花钱招进队伍 → 这个新队员立刻参战
- 不同种族/国家提供不同的兵种类型

### 任务

#### T3-1：`RecruitPool` 数据类
- **文件**：`BladeHexCore/src/Strategic/Recruit/RecruitPool.cs`
- **字段**：
  - `string PoiId` — 所属 POI
  - `List<RecruitableUnit> Available` — 可招募列表
  - `int LastRefreshDay` — 上次刷新天数
  - `int RefreshIntervalDays = 7`
- **结构 `RecruitableUnit`**：
  - `UnitData Unit` — 已生成的完整 UnitData（种族/属性/特质/等级 1-3）
  - `int Cost` — 招募费用
  - `int WeeklyWage` — 周薪
  - `string SourceTemplate` — 兵种模板 id（用于刷新时重新生成同类）

#### T3-2：刷新算法
- **文件**：`BladeHexCore/src/Strategic/Recruit/RecruitPoolGenerator.cs`
- **职责**：每周根据 POI 所属 NationConfig 生成可招募列表
- **逻辑**：
  - POI.OwningFaction → NationConfig.RecruitPool（字符串列表，如 ["knight", "spearman", "archer"]）
  - 城镇规模决定数量：村庄 2-3 人，城镇 4-6 人，首都 6-10 人
  - 每个候选用 `CharacterGenerator.GenerateCharacter(race, level)` 生成完整属性
  - 价格 = 基础价 × (1 + 0.5 × level) × 种族系数

#### T3-3：兵种模板 SSOT
- **文件**：`BladeHexCore/src/Data/MercenaryTemplate.cs`
- **数据**：每个 RecruitPool 字符串对应的兵种定义
  - `string TemplateId` — "knight" / "ranger" / ...
  - `string Race` — 限制种族
  - `int LevelMin / LevelMax`
  - `int BaseCost / BaseWage`
  - `string[] InitialEquipment` — 初始装备 item_id 列表
  - `string PreferredClass` — 职业倾向（影响属性分配偏好）

#### T3-4：招募 UI 接通
- **文件**：`src/ui/overworld/RecruitPanel.gd`（已存在）
- **改动**：
  - 进入 POI 时：`var pool = RecruitPoolService.GetOrCreate(poi.PoiId)`
  - 显示 `pool.Available` 每条：肖像/名字/种族/等级/属性概览/价格/周薪
  - 招募按钮：`EconomyManager.SpendGold(cost)` + `Roster.Add(unit)` + `pool.Available.Remove(unit)`
  - 队伍满员时按钮置灰

#### T3-5：起始队伍配置
- **文件**：`OverworldScene.Init.cs`
- **改动**：新游戏时，根据玩家种族/出身（OriginSelect 选择）给一支初始小队
  - 默认：玩家本人 + 1-2 个新手队员（同种族，等级 1）
  - 快速游戏：玩家 + 3 个随机队员
- **理由**：游戏开局如果只有玩家一个人，第一场战斗就死了，体验糟

#### T3-6：兵种装备初始化
- **文件**：`BladeHexCore/src/Strategic/Recruit/MercenaryEquipper.cs`（新）
- **职责**：根据 `MercenaryTemplate.InitialEquipment` 给新招募的队员穿好装备
- **依赖**：装备系统（已有 `ItemData / EquipmentSlot`）

### 验收
- [ ] 新游戏开始有 2-4 人小队
- [ ] 进城镇酒馆能看到 4-6 个候选
- [ ] 花钱招一个 → 队伍 +1，候选 -1，金币减
- [ ] 一周后再进同一城镇 → 候选已刷新（不同人）
- [ ] 不同国家的城镇候选种族/兵种不同

## 阶段 4：经济压力

**结束时玩家能做：**
- 每天工资从金库扣 → 钱不够 → 队员逐渐流失
- 食物消耗 → 没食物 → 全员士气掉
- 必须主动接活赚钱才能维持队伍

### 任务

#### T4-1：`WageSystem`
- **文件**：`BladeHexCore/src/Strategic/Economy/WageSystem.cs`
- **职责**：每日结算工资
- **逻辑**：
  - 每个 Member 按 `RecruitableUnit.WeeklyWage / 7` 扣
  - 队长（玩家）不扣工资
  - 总额 > 当前金币 → 触发"欠饷"事件
- **触发时机**：`EconomyManager.OnDayPassed` 信号

#### T4-2：欠饷处理
- **文件**：`BladeHexCore/src/Strategic/Economy/MoraleSystem.cs`
- **逻辑**：
  - 欠饷 1 天：所有队员士气 -10
  - 连续 3 天：随机一个队员离队（最低士气先走）
  - 连续 7 天：50% 队员集体离队
- **`UnitData` 加字段**：`Morale` (0-100，初始 50)

#### T4-3：食物消耗细化
- **现状**：`EconomyManager.ConsumeFood(deltaHours * 0.1f)` 太抽象
- **改动**：
  - 食物消耗 = `Roster.Members.Count × 0.5 / 天`（每人每天 0.5 单位）
  - 没食物 → 每天士气 -5 + HP 不能恢复
- **相关**：`EconomyManager` 增加 `FoodPerMemberPerDay` 配置

#### T4-4：商人/酒馆补给
- **改动**：城镇酒馆/商店增加"补给食物"选项
  - 价格随城镇繁荣度浮动
  - 战时国家食物贵 50%（阶段 8 接通）

#### T4-5：财务面板
- **文件**：`src/ui/overworld/EconomyPanel.gd`（新）
- **显示**：
  - 当前金币、每日收支
  - 食物存量、每日消耗、能撑几天
  - 队员工资明细
  - 警告：欠饷 X 天 / 食物不足 Y 天

### 验收
- [ ] 闲逛 7 天不接活 → 金币显著下降
- [ ] 金币耗尽后第 3 天 → 弹窗"X 因为欠饷离开了队伍"
- [ ] 食物耗尽 → 队员 HP 不再自动恢复
- [ ] 经济面板能看到下一周的财务预测

## 阶段 5：委托循环

**结束时玩家能做：**
- 进城镇 → 看任务板 → 选委托（杀某营地/护送商队/给村庄送物资）→ 大地图上有目标点 → 完成 → 回交付者领赏

### 任务

#### T5-1：`Quest` 数据类完整化
- **文件**：`BladeHexCore/src/Quest/QuestData.cs`（已存在则补字段）
- **字段**：
  - `string QuestId`
  - `QuestType Type` — `Kill / Escort / Deliver / Investigate`
  - `string ClientPoiId` — 委托发布者 POI
  - `string TargetPoiId` 或 `Vector2I TargetCoord`
  - `int RewardGold`
  - `List<ItemData> RewardItems`
  - `int ReputationReward` — 与发布者势力的声望
  - `int DaysUntilExpire`
  - `QuestStatus Status` — Available / Accepted / Completed / Failed / Expired

#### T5-2：任务生成器
- **文件**：`BladeHexCore/src/Quest/QuestGenerator.cs`
- **职责**：城镇任务板每日生成新委托
- **逻辑**：
  - 每个城镇维护一个 QuestPool（最多 5 个 active）
  - 按 NationConfig + 周边敌方 entity 生成：
    - 杀任务：选 200 格内的敌方 entity，悬赏 = entity 等级 × 50
    - 护送：选另一个城镇作为目的地
    - 送货：选另一个城镇 + 一类物资
  - 每天有概率刷新

#### T5-3：任务板 UI
- **文件**：`src/ui/overworld/QuestBoardPanel.gd`（已存在）
- **改动**：
  - 显示该城镇的可接任务列表
  - 每条：标题/描述/奖励/期限
  - 接受按钮 → `QuestManager.Accept(questId)`
  - 队伍已接 5 个 → 不能再接

#### T5-4：任务追踪
- **文件**：`BladeHexFrontend/src/View/Quest/QuestManager.cs`（已存在）
- **改动**：
  - `ActiveQuests` 列表
  - 每帧检查目标坐标 vs 玩家位置 → 接近触发完成判定
  - 时间到期 → 标记 Failed

#### T5-5：任务目标视觉
- **文件**：`QuestTargetVisual.gd`（已存在）
- **改动**：在 chunk 模式下根据 ActiveQuests 渲染图标

#### T5-6：任务完成闭环
- **文件**：`QuestManager.OnQuestCompleted`
- **流程**：
  - `Kill` 类：战斗胜利 → 检查死亡 entity 是否匹配任务目标 → 标记完成
  - `Escort` 类：到达目标 POI → 标记完成
  - `Deliver` 类：到达目标 POI 且物品在背包 → 标记完成
- **领赏**：回原 POI → 触发"领赏"对话 → `EconomyManager.AddGold(reward)` + 声望

#### T5-7：任务面板（已接任务）
- **文件**：`src/ui/quest/QuestLog.gd`（已存在）
- **显示**：当前所有 active quest，目标位置，剩余时间

### 验收
- [ ] 进城镇任务板看到 3-5 个委托
- [ ] 接一个杀哥布林营地的任务 → 大地图上目标点亮
- [ ] 灭掉营地后任务标记完成
- [ ] 回原城镇领赏 → 金币 + 声望增加
- [ ] 同时进行 5 个任务上限生效

## 阶段 6：世界活力

**结束时玩家能做：**
- 看到大地图上其他实体在动：商队走商路、领主军队出征、哥布林袭击村庄
- 玩家不接任务时世界仍在变化（村庄被劫掠、城镇陷落）
- 这些变化产生新的任务（"我们村被烧了，求人帮忙"）

### 任务

#### T6-1：`ChunkAStar` 实现
- **文件**：`BladeHexCore/src/Map/Chunk/ChunkAStar.cs`
- **现状**：chunk-migration-plan 提到了，没实现
- **职责**：
  - 分层寻路：chunk 级粗路径 + tile 级精确路径
  - 跨 chunk 自动加载邻居 chunk

#### T6-2：实体寻路接通
- **文件**：`BladeHexFrontend/src/View/Strategic/OverworldEntityManager.cs`
- **现状**：chunk 模式下 TODO 没接 ChunkAStar
- **改动**：
  - `EntityMgr.SetChunkNavigation(chunkManager, chunkAStar)`
  - `MovementProcessor.TickMovement` 用 ChunkAStar 找路

#### T6-3：村庄被袭事件链
- **流程**：
  1. 哥布林营地（Settlement POI）每 N 天有概率派出 RaidParty 实体
  2. RaidParty 寻路到最近的村庄（OverworldPOI.Village）
  3. 到达后触发 OnVillageAttacked 事件
  4. 村庄 Prosperity 下降 + 守军战斗
  5. 玩家附近时弹窗提示"X 村正在被袭击"
- **代码**：`OverworldEntity.cs` 已有 RaidParty 概念，需要把 trigger 接通

#### T6-4：商队系统

## 阶段 8：声望与外交

**结束时玩家能做：**
- 帮某国做任务 → 该国声望升 → 解锁高级招募/降低城镇税
- 抢劫某国商队 → 该国敌对 → 进城被通缉
- 国家之间互相宣战，玩家选边站

### 任务

#### T8-1：`ReputationTracker`
- **文件**：`BladeHexCore/src/Strategic/Reputation/ReputationTracker.cs`
- **字段**：`Dictionary<string nationId, int reputation>`（-100 ~ +100）
- **接口**：`AddReputation(nationId, delta)`、`GetReputation(nationId)`

#### T8-2：声望影响
- **进城**：声望 < -50 → 城门拒绝玩家进入；< -75 → 守军主动攻击
- **招募**：声望 < 0 → RecruitPool 减半；< -25 → 完全没有招募
- **价格**：声望 +25 起商店折扣 -10%；-25 起涨价 +20%
- **任务**：声望 +50 才解锁该国"领主任务"（高奖励）

#### T8-3：声望来源
- 完成该国任务：+5
- 击败该国敌人：+1
- 攻击该国实体：-10
- 抢劫该国商队：-25
- 杀该国 NPC：-50
- 战胜后释放战俘：+5

#### T8-4：国家外交
- **文件**：`BladeHexCore/src/Strategic/Diplomacy/DiplomacyManager.cs`
- **字段**：`Dictionary<(string a, string b), int relation>`（-100 ~ +100）
- **逻辑**：
  - 战争状态：< -50
  - 中立：-50 ~ +50
  - 盟友：> +50
  - 关系受边境冲突、贸易、玩家干预影响
- **影响**：
  - 战争 → 双方互发军队（阶段 6）
  - 战争 → 双方城镇互相不接对方公民
  - 盟友 → 共享道路、贸易增加

#### T8-5：通缉/逃亡
- **逻辑**：声望 < -75 时进入"通缉"状态
- **行为**：该国领土内每 N 分钟随机刷一队捕快追玩家
- **解除**：跨国境逃到敌国 / 时间过去（30 天）

### 验收
- [ ] 帮人类王国做 5 个任务后，进精灵国声望上升（人类的盟友）
- [ ] 抢一次商队后下次进城被警告，连续抢被通缉
- [ ] 国家间宣战后大地图能看到军队对峙

## 阶段间依赖图

```
阶段 1 (PartyRoster)
   ├─→ 阶段 2 (大地图战斗闭环 — 必须有 Roster 才能传给战斗)
   │      ├─→ 阶段 3 (招募 — 必须有 Roster 才能加人)
   │      │      └─→ 阶段 4 (经济 — 必须有 Roster 才能扣工资)
   │      │             └─→ 阶段 5 (委托 — 必须有经济压力才有意义)
   │      │                    └─→ 阶段 6 (世界活力 — 委托循环之上的活水)
   │      │                           └─→ 阶段 8 (声望/外交 — 需要持续行为积累)
   │      └─→ 阶段 7 (装备库存 — 需要战斗产出战利品)
   │
   └─→ 阶段 9 (持久化 — 持续渗透到所有阶段)
```

**MVP 路径**：阶段 1 → 2 → 3 → 4 → 5 = 完整可玩的骑砍循环  
**完整体验**：再加 6、7、8 = 接近战场兄弟的世界感  
**发布版本**：再加 9 = 可上 Steam 抢先体验
