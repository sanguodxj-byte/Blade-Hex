# 对话系统

## 概述

对话系统驱动大地图上所有的NPC交互——从酒馆招募外族英雄到村庄长老委托任务，从暗影教团叛教者的忏悔到龙族的远古低语。对话不是纯文本展示，而是**游戏机制的前端**——检定、招募、交易、任务触发都通过对话流程完成。

当前代码基础：`DialoguePanel.gd` 已存在作为UI容器，`InteractionManager.gd` 管理交互入口，`InteractionType.gd` 定义了15种交互类型。本设计补全对话树数据结构和流程逻辑。

---

## 一、对话触发场景

| 场景 | 触发方式 | 典型内容 |
|------|----------|----------|
| 酒馆招募 | 点击酒馆中的NPC → 选择"交谈" | 外族招募检定、性格对话、加入条件 |
| 村庄委托 | 点击布告栏/村长 → 选择"委托" | 任务描述、奖励谈判、难度提示 |
| 城镇商店 | 点击商人 → 选择"交易" | 商品推荐、讨价还价、特殊商品解锁 |
| 野外遭遇 | 接近AI实体 → 弹出交互面板 | 战斗/回避/交涉/勒索 |
| 势力外交 | 接近领主/使者 → 选择"交谈" | 结盟、贿赂、威胁、情报交换 |
| 同伴互动 | 营地/城镇中点击队友 | 个人故事、关系发展、冲突事件 |
| 剧情 | 特定地点/事件触发 | 暗影教团线索、龙族传说、古代遗迹解读 |

## 二、对话树数据结构

### 2.1 核心节点类型

对话树由三种节点组成：

| 节点类型 | 说明 | 用途 |
|----------|------|------|
| **对话节点（DialogueNode）** | NPC说一段话 | 铺陈剧情、提供信息 |
| **选择节点（ChoiceNode）** | 玩家从2-5个选项中选择 | 分支决策 |
| **结果节点（ResultNode）** | 执行游戏逻辑 | 检定、奖励、触发事件 |

### 2.2 对话节点（DialogueNode）

```gdscript
# 对话节点数据
class_name DialogueNodeData
extends Resource

@export var node_id: String = ""              # 唯一标识
@export var speaker_name: String = ""         # 说话者名字（空=旁白）
@export var speaker_portrait: String = ""     # 头像ID（对应立绘资源）
@export var text: String = ""                 # 对话文本（支持变量插值）
@export var choices: Array[DialogueChoice] = [] # 可选回应
@export var on_show_effects: Array[DialogueEffect] = [] # 显示时执行的效果
@export var next_node_id: String = ""         # 无选择时自动跳转的下一节点
```

### 2.3 选择项（DialogueChoice）

```gdscript
class_name DialogueChoice
extends Resource

@export var choice_id: String = ""            # 选择ID
@export var text: String = ""                 # 选项文本
@export var condition: DialogueCondition = null # 显示条件（null=始终显示）
@export var next_node_id: String = ""         # 选择后跳转的节点ID
@export var on_select_effects: Array[DialogueEffect] = [] # 选择时执行的效果
@export var is_default: bool = false          # 默认选项（无特殊条件时高亮推荐）
```

### 2.4 对话条件（DialogueCondition）

条件决定选项是否可用/可见：

| 条件类型 | 参数 | 说明 |
|----------|------|------|
| 属性检定 | attribute, threshold | `CHA >= 14` → 解锁"说服"选项 |
| 声望阈值 | reputation_type, min_value | 声望≥30 → 解锁高级委托 |
| 势力关系 | faction, min_relation | 关系≥50 → 解锁结盟选项 |
| 物品检查 | item_id, count | 持有"暗影教团密信" → 解锁追问选项 |
| 种族检查 | race_id | 玩家/队伍中有精灵 → 解锁精灵语对话 |
| 技能盘检查 | node_id | 已点亮"交涉"节点 → 解锁折扣选项 |
| 任务状态 | quest_id, status | 已完成"矿洞调查" → 解锁后续对话 |
| 对话标记 | flag_id | 已选择"答应帮助" → 解锁感谢对话 |
| 阶段检查 | phase | 当前阶段≥2 → 解锁骑士相关对话 |
| 金币检查 | min_gold | 金币≥500 → 解锁贿赂选项 |

### 2.5 对话效果（DialogueEffect）

效果在对话流程中即时执行：

| 效果类型 | 参数 | 说明 |
|----------|------|------|
| 属性检定 | attribute, dc, success_node, fail_node | `CHA检定DC15 → 成功跳说服线/失败跳拒绝线` |
| 设置标记 | flag_id, value | 设置对话变量（持久化到存档） |
| 给予物品 | item_id, count | NPC给玩家物品 |
| 移除物品 | item_id, count | 扣除玩家物品 |
| 给予金币 | amount | 正数=获得，负数=扣除 |
| 调整声望 | reputation_type, amount | +5声望 |
| 调整势力关系 | faction, amount | +10关系 |
| 触发任务 | quest_id, status | 开始/完成任务 |
| 触发战斗 | enemy_template_id | 对话后立即进入战斗 |
| 招募英雄 | hero_unit_data | 英雄加入队伍 |
| 解锁商品 | shop_id, item_ids | 商店新增可购买物品 |
| 离开对话 | — | 结束对话，返回交互界面 |

## 三、对话流程示例

### 3.1 外族精灵招募（完整流程）

```
[D1] 酒馆老板: "角落那位精灵看起来不太好相处，你要试试？"
  → [进入对话] → D2

[D2] 精灵·艾拉瑞尔: "……人类。你想做什么？"
  ├─ [条件: CHA ≥ 12] "听说你是银叶来的？我在绿溪村见过你们的游侠。"
  │   → [D3] 亲和路线
  ├─ "我需要一位弓箭手。报酬丰厚。" → [D5] 直接路线
  ├─ [条件: 队伍中有半精灵] "我们队里有半精灵，你应该不会太排斥我们。"
  │   → [D4] 混血桥路线
  └─ "没什么，打扰了。" → [离开]

[D3] 艾拉瑞尔: (态度软化) "绿溪村……那些人类至少懂得尊重边界。"
  → "也许你不是我想象中的那种人类。"
  → [CHA检定 DC12]
  ├─ 成功 → [D6] "好吧，我跟你走。"
  └─ 失败 → [D7] "不，还不够。你先去银叶帮我送封信，证明你的诚意。"
       → [触发任务: "精灵的考验"]

[D4] 队友·半精灵: "嘿，同胞，我们不全是坏家伙。"
  → 艾拉瑞尔: (犹豫) "……半血。你至少还保留了一部分我们祖先的敏锐。"
  → [D6] 直接跳到同意

[D5] 艾拉瑞尔: "哼，用钱打动我？有意思。"
  → [效果: 招募价格×1.5] → [D6]

[D6] 艾拉瑞尔: "好吧，但我有三个条件：第一，不砍伐古树；第二，不攻击无辜的精灵；第三，每天给我一小时独处。"
  ├─ "同意。" → [效果: 招募加入，价格200金] → [离开]
  └─ "条件太多。" → [D8] "那就没什么好谈的了。"

[D7] 艾拉瑞尔: "银叶森林月影哨站的埃尔隆德会告诉你怎么做。"
  → [效果: 触发任务"精灵的考验"] → [离开]

[D8] 艾拉瑞尔: (转身继续喝酒) → [离开]
```

### 3.2 村庄委托（标准模板）

```
[D1] 村长: "冒险者！你来得正好。"
  → [描述问题] → D2

[D2] 村长: "[随机: 哥布林偷粮/狼群袭扰/矿洞异响/失踪的猎人]"
  ├─ "交给我吧。报酬怎么算？" → [D3] 接受
  ├─ [条件: CHA ≥ 14] "报酬能不能再加点？这种活很危险。" → [CHA检定 DC12]
  │   ├─ 成功 → [D4] "好吧……再加50金。"
  │   └─ 失败 → [D3] "不行，村子已经很穷了。"
  └─ "抱歉，我现在抽不开身。" → [离开]

[D3] 村长: "完成后来找我领赏。[效果: 触发任务]" → [离开]
[D4] 村长: "行……你救了我们全村，这钱值得。[效果: 触发任务(奖励+50金)]" → [离开]
```

## 四、对话文本系统

### 4.1 变量插值

对话文本支持动态插值，使用 `{变量名}` 语法：

| 变量 | 示例 | 来源 |
|------|------|------|
| {player_name} | "阿尔萨斯" | 主角名字 |
| {speaker_name} | "艾拉瑞尔" | 当前说话者 |
| {gold} | "2,450" | 当前金币 |
| {hero_count} | "5" | 队伍人数 |
| {faction_name} | "铁炉堡联盟" | 关联势力名 |
| {quest_target} | "黑牙营地" | 任务目标名 |

示例：`"你好，{player_name}。我听说你的队伍已经有{hero_count}人了，是吗？"`

### 4.2 条件文本

同一段对话根据条件显示不同文本：

```
@if(cha >= 14)[你以优雅的姿态向对方致意。]@else()[你笨拙地点了点头。]@endif
```

## 六、对话标记持久化

### 6.1 标记系统

对话中的选择和结果通过**标记（Flags）**持久化到存档：

| 标记类型 | 作用域 | 示例 |
|----------|--------|------|
| 全局标记 | 整个游戏进程 | `helped_elven_village = true` |
| NPC标记 | 针对特定NPC | `elariele_recruitment = "accepted"` |
| 任务标记 | 针对特定任务 | `mine_investigation_clue_found = 3` |
| 队友标记 | 针对特定队友 | `dorn_backstory_revealed = true` |

### 6.2 标记影响链

对话标记不是孤立的——它们影响后续所有相关对话：

```
标记 "saved_village_from_goblins" = true
  → 村长后续对话新增感谢内容
  → 村庄商店价格-10%
  → 解锁高级委托
  → 该村庄被攻击时，玩家收到紧急通知
```

## 七、同伴对话系统

### 7.1 同伴对话触发

| 触发条件 | 对话类型 |
|----------|----------|
| 每隔5-10个游戏日 | 闲聊（随机抽取） |
| 进入特定区域 | 区域联想（精灵进银叶森林） |
| 战斗后（有人濒死） | 关心/担忧 |
| 队友种族冲突 | 争论/和解 |
| 声望晋升 | 祝贺/评论 |
| 同伴忠诚度变化 | 抱怨/鼓励 |
| 完成同伴相关任务 | 个人故事推进 |

### 7.2 同伴性格标签

每个同伴在生成时获得1-2个性格标签，影响对话语气：

| 性格标签 | 对话风格 | 对选择的影响 |
|----------|----------|------------|
| 勇敢 | 直接、自信 | 支持战斗选项 |
| 谨慎 | 小心、犹豫 | 反对冒险行为 |
| 贪婪 | 务实、计较 | 关注金钱选项 |
| 忠诚 | 坚定、护卫 | 支持保护弱者 |
| 愤世 | 冷嘲、挖苦 | 负面评论一切 |
| 好奇 | 多问、兴奋 | 支持探索选项 |

## 八、原型阶段

### 必须实现
- 基础对话树（线性 + 2-3个分支选择）
- DialoguePanel UI 显示文本 + 选项
- 属性检定条件（CHA检定）
- 设置对话标记
- 酒馆招募对话流程
- 村庄委托对话流程

### 原型之后逐步扩展
- 完整条件系统（物品/种族/任务状态）
- 对话效果系统（奖励/触发战斗/招募）
- 同伴对话（性格标签 + 触发条件）
- 变量插值
- 条件文本
- 对话日志回看
