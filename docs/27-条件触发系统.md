# 27-条件触发系统设计

## 概述

大地图采用 **条件触发式生成**，不模拟全局世界状态。所有世界事件（遭遇、POI 变化、环境事件）通过统一的条件触发框架驱动。

## 核心原则

1. **玩家中心** — 只处理玩家感知范围内的事件
2. **确定性** — 相同种子 + 相同条件 → 相同结果
3. **惰性计算** — 不预计算，玩家到达时才判定
4. **零全局 tick** — 没有 `_Process` 里的世界模拟循环

## 触发类型分类

| 类型 | 触发时机 | 举例 |
|---|---|---|
| **空间触发** | 玩家进入新 Chunk | 野生怪物、资源点、悬念事件 |
| **交互触发** | 玩家主动接触 | POI 进入、遭遇标记接触、NPC 对话 |
| **时间触发** | 天数推进（玩家操作驱动） | POI 繁荣恢复、掠夺队刷新、季节变化 |
| **连锁触发** | 前置事件完成 | 击败巢穴守卫 → 宝箱出现、完成任务 → 新 POI 可见 |
| **环境触发** | chunk 地形 + 季节判定 | 暴风雪、山崩、洪水 |

## 触发流程

```
玩家操作（移动/休息/交互）
    │
    ├─→ 空间触发判定
    │     chunk 变化 → 遭遇槽位检查 → 放置/移除遭遇标记
    │
    ├─→ 时间触发判定
    │     天数+1 → POI.OnDayPassed() → 掠夺队刷新检查
    │                → 区域事件检查 → 天气/季节
    │
    ├─→ 交互触发判定
    │     接触标记 → EncounterData 生成 → 战斗/事件
    │     接触 POI → POI 交互面板
    │
    └─→ 连锁触发判定
          事件完成 → 检查后续条件 → 解锁新内容
```

## 模块设计

### TriggerCondition — 条件谓词

```
TriggerCondition {
    Id: string                    // 条件唯一 ID
    Type: TriggerType             // 触发类型
    Priority: float               // 优先级（高先判定）
    Chance: float                 // 基础概率 (0~1)
    RequiredRegions: string[]     // 限定区域（空=任意）
    RequiredTerrains: TerrainType[] // 限定地形
    MinPlayerLevel: int           // 最低玩家等级
    MinDay: int                   // 最早触发天数
    MaxDay: int                   // 最晚触发天数（0=无限）
    CooldownDays: int             // 触发后冷却天数
    PrerequisiteIds: string[]     // 前置触发 ID（全部满足才可触发）
    MutuallyExclusive: string[]   // 互斥触发 ID（任一已触发则跳过）
}
```

### TriggerResult — 触发结果

```
TriggerResult {
    TriggerId: string
    Triggered: bool
    Encounter: EncounterData?     // 遭遇数据（如果有）
    UnlockedIds: string[]         // 解锁的后续触发 ID
    Narrative: string             // 叙事文本
}
```

### TriggerContext — 触发上下文

```
TriggerContext {
    WorldSeed: int
    CurrentDay: int
    PlayerLevel: int
    PlayerWorldPos: Vector2I
    PlayerChunk: Vector2I
    CurrentChunk: ChunkData
    CurrentRegion: string
    CurrentTerrain: TerrainType
    History: TriggerHistory       // 已触发的记录
}
```

### TriggerHistory — 触发历史

记录已触发的条件，用于冷却和连锁判定。

```
TriggerHistory {
    _triggered: Map<string, int>  // id → 触发天数
    
    IsTriggered(id) → bool
    IsOnCooldown(id, currentDay, cooldownDays) → bool
    GetTriggerDay(id) → int?
    Record(id, day)
    Serialize() → Dictionary
    Deserialize(Dictionary)
}
```

## 五种触发 Handler

### 1. SpatialTriggerHandler — 空间触发

玩家进入新 Chunk 时判定。用于：遭遇生成、资源点、悬念事件。

Evaluate 在 ChunkManager.UpdateChunks 返回新 Chunk 后调用。每个新 chunk 的每个可触发位置逐一判定。

判定逻辑：
1. 检查 RequiredRegions / RequiredTerrains 过滤
2. 用 (WorldSeed ^ coordHash) 做确定性随机
3. Chance * DangerModifier * DayModifier = 最终概率
4. 判定通过 → 设置 Chunk.EncounterSlot = Available

### 2. InteractionTriggerHandler — 交互触发

玩家主动接触时判定。用于：POI 交互、遭遇战斗、NPC 对话。

不做概率判定 — 交互触发是确定的。只检查前置条件（等级/天数/连锁）。

### 3. TimeTriggerHandler — 时间触发

天数推进时判定（由玩家操作驱动，不是全局 tick）。用于：POI 繁荣恢复、掠夺队刷新、季节事件。

Evaluate 在 WorldGenerator.AdvanceDay() 中调用。遍历所有已知 POI，不遍历全局实体。

### 4. ChainTriggerHandler — 连锁触发

前置事件完成后判定。用于：任务链、解锁新区域、隐藏内容。

检查 PrerequisiteIds 是否全部在 History 中。检查 MutuallyExclusive 是否在 History 中。

### 5. EnvironmentTriggerHandler — 环境触发

chunk 地形 + 季节 + 天数综合判定。用于：暴风雪、山崩、洪水。

判定逻辑：季节 = f(month)，恶劣天气概率 = f(season, terrain, elevation)，效果 = 移动减速 / 视野缩小 / 强制战斗。

## 调用时机（由 Frontend 驱动）

```
OverworldScene._Process():
    if (player moved to new chunk):
        newChunks = WorldGenerator.UpdatePlayerPosition(...)
        foreach chunk in newChunks:
            triggerEngine.EvaluateSpatial(ctx)
    
    if (player contacts encounter marker):
        result = WorldGenerator.TriggerEncounter(coord, ...)
        triggerEngine.EvaluateInteraction(ctx)
    
    if (player rests / time advances):
        WorldGenerator.AdvanceDay()
        triggerEngine.EvaluateTime(ctx)
```

## 优先级规则

空间触发覆盖 80% 的场景。时间触发仅用于：
- POI 繁荣度恢复
- 掠夺队刷新周期
- 季节变更

不需要时间触发的：
- 遭遇生成 → 空间触发
- 资源点 → 空间触发
- 环境事件 → 空间/季节判定
- 连锁事件 → 前置事件驱动
