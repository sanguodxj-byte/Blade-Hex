# 技能盘系统策划案

> 基于 Path of Exile 技能树模型重新定义的连通网络式技能盘系统

---

## 一、设计理念

### 1.1 核心隐喻

本技能盘采用 PoE 式连通网络模型，而非传统树状分支。其核心张力来自一个简单事实：**你不可能什么都要。**

- 20 级角色仅能点亮全盘约 25-30% 的节点
- 每一个属性点的分配都是在"想要的"与"付得起的"之间做选择
- 路径上的小节点不是奖励，而是**到达大节点的过路费**——这恰恰是选择的重量

### 1.2 PoE 模型特征映射

| PoE 特征 | 本项目对应 |
|---|---|
| 巨型连通网（~1300 节点） | 精简版连通网（~184 节点） |
| 所有职业共享同一棵树 | 所有角色共享同一盘，起点统一在"启程" |
| 起点位置不同决定"职业" | 无固定职业，起点统一在"启程" |
| 小节点 = 数值过路费 | 小节点 = HP+5、AC+1 等，到达大节点的路径成本 |
| 大节点/Keystone = 能力质变 | 大节点 = 连击、旋风斩等主动/核心被动 |
| 区域性聚集（近战区/法术区） | 6 个属性方向 = 技能盘上的 6 个区域 |
| 到远处大节点的路径成本 = build 深度 | 同理：想拿法术区的"快速施法"得走过一堆 INT 小节点 |
| 环路 = 多条路径到达同一目标 | 技能盘应有环路，提供路线选择 |

## 二、规模参数

### 2.1 节点总量

| 参数 | 数值 | 推导 |
|---|---|---|
| 可点亮节点数 | ~35 个 | 19 属性点 + 4 次跳跃带来的跨区推进 |
| 盘总节点数 | ~184 个 | 35 × (3.5~4) 倍冗余 |
| 其中大节点 | 35-40 个 | 文档已定义 33 个 + CHA 方向补齐 |
| 其中小节点 | 80-110 个 | 路径成本节点 |

### 2.2 为什么是 4-5 倍冗余

一个 20 级角色只能点亮全盘的 25-30%。这正是 PoE 的核心体验：**不可能全点亮 = 选择有意义。** 若冗余不足，选择沦为"全都要"。

---

## 三、拓扑结构：区域化连通网

### 3.1 整体布局

```
                      ┌─────────────────┐
                      │   STR 区域      │
                      │  (近战/力量)    │
                      │  ~20 节点      │
                    ┌─┤                │
                    │ └─────────────────┘
  ┌──────────────┐  │  ┌──────────────┐
  │  CON 区域    │  │  │  DEX 区域    │
  │ (防御/体魄)  ├──┼──┤ (远程/灵巧)  │
  │  ~20 节点   │  │  │  ~20 节点   │
  └──────┬───────┘  │  └──────┬───────┘
         │     ┌────┴────┐    │
         │     │  启 程   │    │
         │     │  (中心)  │    │
         │     └────┬────┘    │
  ┌──────┴───────┐  │  ┌──────┴───────┐
  │  CHA 区域    │  │  │  INT 区域    │
  │ (指挥/魅力)  ├──┴──┤ (法术/智力)  │
  │  ~20 节点   │     │  ~20 节点   │
  └──────────────┘     └──────────────┘
                       ┌──────────────┐
                       │  WIS 区域    │
                       │ (治疗/感知)  │
                       │  ~20 节点   │
                       └──────────────┘
```

### 3.2 关键原则

- 区域之间**不是墙**，而是有自然连接路径
- 相邻区域的边缘只需 2-3 个过渡小节点即可连通
- 区域核心大节点之间距离遥远——这是"深度投入"的含义
- **6 个区域共享同一张盘，一角色一盘**

---

## 四、六大区域定义

### 4.1 区域概览

| 区域 | 缩写 | 主题 | 核心大节点示例 | 区域规模 |
|---|---|---|---|---|
| 力量 | STR | 近战/伤害爆发 | 连击、旋风斩、猛击 | ~20 节点 |
| 灵巧 | DEX | 远程/暴击闪避 | 快速射击、暴击精通 | ~20 节点 |
| 体魄 | CON | 防御/生存 | 铁壁、坚韧体质 | ~20 节点 |
| 智力 | INT | 法术/奥术 | 快速施法、法术穿透 | ~20 节点 |
| 感知 | WIS | 暴击/刺杀 | 爆头突袭、刺客本能 | ~24 节点 |
| 魅力 | CHA | 指挥/社交 | 战吼、鼓舞士气 | ~20 节点 |

### 4.2 潜行归属

潜行作为 **DEX 区域的子方向**，不独立成区。DEX 区域内部自然分化出"远程射击"与"潜行暗杀"两条子路径。

---

## 五、区域间连接与过渡

### 5.1 过渡路径

| 过渡路径 | 过渡主题 | 过渡小节点类型 |
|---|---|---|
| STR → CON | 近战→防御 | "近战生存"类：近战伤害时恢复 HP、被攻击时 AC+1 |
| STR → DEX | 力量→灵巧 | "战斗技巧"类：命中+1、暴击率+2% |
| DEX → INT | 灵巧→智力 | "精准施法"类：施法速度+1、法术命中+1 |
| INT → WIS | 智力→感知 | "神秘学"类：魔力+3、法术豁免+1 |
| WIS → CHA | 感知→魅力 | "精神领袖"类：治疗量+1、士气+2 |
| CHA → CON | 魅力→体魄 | "鼓舞防御"类：友军 AC+1、自身 HP+3 |

### 5.2 过渡节点的设计意义

过渡节点是**两个方向都能受益的混合加成**。走到区域交界处时，你的 build 自然获得了相邻方向的一些好处——这正是 PoE 的魅力所在。

- 过渡不是"二选一"，而是"经过即获得"
- 区域边缘的小节点 = 与邻居共享的加成池
- 过渡节点的存在使得"纯方向 build"和"混合 build"都可行

---

## 六、环路设计

### 6.1 为什么需要环路

环路意味着从 A 到 B 有**多条路径**，每条路径上的小节点不同。玩家可以选择：

- 直走：经过 A 方向的小节点
- 绕路：经过 B 方向边缘的小节点，可能更"划算"

### 6.2 环路位置

| 环路类型 | 位置 | 示例 |
|---|---|---|
| 同区域内部 | 入门大节点 → 进阶大节点之间 | STR 区内"连击"到"旋风斩"有 2-3 条可选路径 |
| 区域交界 | 两三个区域边缘形成三角环路 | STR-CON-DEX 三区交界处形成可选绕行 |
| 中心辐射 | 启程出发的 6 条路径形成 6 个环 | 走 STR 方向再折返 CON 方向 = 环路 |

### 6.3 环路的策略价值

- **同区域环路** = 同方向内的细分流派选择
- **跨区域环路** = "借道"邻居方向获得额外加成
- 环路让每一点分配都有"走哪条路"的决策

---

## 七、跳跃机制

### 7.1 定位重定义

在当前星盘模型下，跳跃 = **在远处属性区域建立新的推进锚点，用来改变职业路线与专属技能获取方向**。

跳跃不直接改变职业，也不免费完成节点图形；它只让角色开始推进其他属性区域。职业变化仍然只发生在职业图形节点完整完成之后。

### 7.2 跳跃价值表

| 跳跃用途 | 玩法价值 | 场景 |
|---|---|---|
| 同区域内跳到未开启图形 | 更快深化主方向 | 强化单属性职业深度 |
| 跳到相邻区域边缘图形 | 开启第二属性涉足 | 形成双属性职业与专属技能 |
| 跳到对角区域边缘图形 | 开启远距离职业转向 | 近战转法术、远程转指挥等混合 build |
| 跳到对角区域核心图形 | 高风险职业重塑 | 极端复合职业路线 |

### 7.3 跳跃次数与时机

- 全程共 4 次跳跃机会
- 5 级时获得第一次跳跃（此时角色只走了 5-6 个节点）
- 每次跳跃消耗 1 次跳跃机会，并点亮目标图形的 1 片瓦片
- **跳跃让跨属性职业路线变成可能，但职业本身仍由完成图形决定**

---

## 八、启程节点与初始路径

### 8.1 启程节点

- 所有角色 1 级解锁，统一起点
- 位于盘的中心
- 6 条初始路径从这里出发

### 8.2 初始小节点

| 方向 | 初始小节点（启程→入门大节点之间） | 设计意图 |
|---|---|---|
| STR | HP+3、近战命中+1 | 走近战的第一步就获得少量生存+攻击 |
| DEX | AC+1、速度+1 | 走灵巧的第一步是更灵活 |
| CON | HP+5、全豁免+1 | 走防御的第一步是更硬 |
| INT | 魔力+3、法术位+1(戏法) | 走法术的第一步获得施法能力 |
| WIS | 暴击率+2%、WIS 检定+1 | 走暴击流的第一步是感知和直觉 |
| CHA | 士气+2、CHA 检定+1 | 走指挥的第一步是社交 |

### 8.3 初始路径的开放性

第一步方向**不锁定**角色。你可以走两步到 STR 区域再折返回来——只是消耗了属性点在过路费上。这种"浪费"就是选择的代价。

---

## 九、代价型大节点（Keystone）

### 9.1 设计理念

PoE 的 Keystone 不只是增益——有些带有负面效果，让选择更有重量。代价型大节点让"强但有限制"成为策略核心。

### 9.2 代价型大节点定义

| 大节点 | 增益 | 代价 |
|---|---|---|
| 狂暴之力 | 近战伤害+50% | AC-3，不能使用盾牌 |
| 绝对专注 | 法术 DC+4 | 不能学习其他体系的法术 |
| 铁壁 | 受到物理伤害-5 | 速度-2，不能冲锋 |
| 幽灵步伐 | 永久半隐身 | HP 上限-20% |

### 9.3 代价型节点在盘上的位置

代价型大节点位于各区域**最深处**。拿到它意味着你已经深度投入该方向，而代价让"全都要"变得不可能。

---

## 十、小节点：路径成本详解

### 10.1 小节点 = 过路费

小节点不是独立的奖励——它是你为了到达大节点而必须经过的路径。每一点加成都微不足道，但路径上积累的数值构成了 build 的底座。

### 10.2 小节点密度

**密集。** 这是 PoE 的核心机制。

- 同区域内部：大节点之间间隔 2-4 个小节点
- 区域交界：过渡小节点 2-3 个
- 小节点的密度确保"不可能全点亮"

### 10.3 小节点数值参考

| 区域 | 典型小节点加成 |
|---|---|
| STR | 近战伤害+1、近战命中+1、HP+3 |
| DEX | 远程命中+1、AC+1、暴击率+1% |
| CON | HP+5、AC+1、全豁免+1 |
| INT | 法术伤害+1、魔力+2、法术位+1 |
| WIS | 暴击率+2%、WIS 检定+1、法力+3 |
| CHA | 士气+1、CHA 检定+1、友军加成+1 |

## 十二、职业称号与技能盘联动

### 12.1 职业称号系统

技能盘上的投资方向会自然形成**职业称号**（详见 `docs/05-角色与职业.md` 职业称号系统章节）。称号是纯粹的标签层，不影响技能盘机制。

- 在某属性区域完成 ≥1 个职业图形节点（Big / Keystone）= **涉足**该属性
- 已涉足的属性组合决定职业名（共 63 种组合 + 1 种无涉足状态「无名者」）
- 每次完成新的职业图形节点后动态重新判定
- 单片瓦片、小节点和未完成图形只影响属性/进度，不影响职业
- 六大单属性职业：战士(STR)、游侠(DEX)、守卫(CON)、法师(INT)、刺客(WIS)、诗人(CHA)
- 双属性及以上组合产生复合职业名，如魔剑士(STR+INT)、圣骑士(STR+WIS)等

### 12.2 称号与技能盘拓扑的关系

- 称号不改变技能盘的连通网络结构
- 称号不限制可点亮节点
- 称号不修改数值——它是"你已经成为什么"的描述，不是"你只能做什么"的限制
- AI推荐路径可参考当前称号辅助玩家决策，但不强制

## 十三、设计决策清单

| # | 决策项 | 方案 |
|---|---|---|
| 1 | 拓扑模型 | 区域化连通网，6 个主题区域 + 过渡路径 + 环路 |
| 2 | 盘规模 | ~184 个节点（大节点 57 + 小节点 121 + 过渡 19 + 启程 1） |
| 3 | 小节点密度 | 密集——路径成本，PoE 核心机制 |
| 4 | 跨分支 | 自然连通，相邻区域通过过渡小节点连接 |
| 5 | 跳跃定位 | 跨属性迁移锚点，用于改变职业路线 |
| 6 | 环路 | 必须有，提供路径选择 |
| 7 | Keystone 代价 | 引入代价型大节点，增加选择重量 |
| 8 | 潜行归属 | DEX 区域子方向，不独立 |
| 9 | 职业称号 | 纯标签层，由已完成职业图形节点的属性组合动态决定（63种），不限制机制 |

## 十四、与旧模型的核心差异

| 旧模型结论 | PoE 模型修正 |
|---|---|
| 6 条路径独立，只能跳跃跨接 | 错。整个盘是连通图，区域之间有自然连接路径 |
| 跳跃是唯一的跨分支手段 | 错。整个盘自然连通；跳跃是跨属性迁移锚点，用来改变职业路线 |
| 小节点应该稀疏 | 错。小节点必须密集——路径成本是核心张力 |
| 分支间完全隔离 | 错。隔离程度由距离和路径成本决定，不是硬隔离 |

---

*本策划案基于 PoE 技能树模型重新定义技能盘系统。核心原则：连通而非隔离，密集而非稀疏，代价而非免费。选择即取舍，不可能什么都要。*
