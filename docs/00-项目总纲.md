# 项目总纲：剑与六芒星 (Sword & Hex) — 魔幻战术RPG

## 一句话定义
一款融合骑砍式大地图战略、HOMM式军队系统和CRPG深度的六边形战棋RPG。从雇佣兵团长成长为割据一方的领主。

## 核心体验
- **三幕进化**：雇佣兵(小队RPG) → 骑士(英雄+士兵) → 领主(HOMM式军团)
- **战术层**：六边形网格回合制战斗，英雄+兵种堆叠，命运检定驱动
- **战略层**：骑砍式大地图行军，招兵买马，攻城掠地，势力博弈
- **RPG深度**：六维属性/检定/豁免系统，120级成长曲线，63种属性组合职业称号，8学派法术与装备驱动身份

## 技术栈
- **引擎**：Godot 4.6+ (.NET 8)
- **语言**：C#（主力，逻辑/系统/场景控制）+ GDScript（UI 布局层）
- **解决方案结构**：
  - `BladeHexCore` — 核心逻辑库（数据模型、地图生成、战略层、技能树）
  - `BladeHexFrontend` — Godot 节点层（View/场景控制器/UI组件，引用 Core）
  - `src/` — GDScript UI 层（纯视觉布局，无业务逻辑）
- **分辨率**：1920×1080
- **视觉风格**：HD-2D（3D六棱柱网格 + 2D Billboard 纸片人），完美支持高程和视线遮挡
- **架构**：
  - 数据驱动设计，C# class + JSON 作为数据容器
  - Autoload 单例管理全局系统（EventBus, GlobalState, AudioManager, SkillTreeManager, DebugConsole）
  - EventBus 事件总线解耦系统间通信
  - 命令模式记录战斗操作（CommandHistory）
  - 状态机驱动战斗流程（CombatStateMachine）
  - AI 策略模式（4种策略：Tactical/Cautious/Reckless/Instinct）

### 语言职责划分
| 层级 | 语言 | 职责 |
|------|------|------|
| 核心逻辑 | C# (BladeHexCore) | 数据模型、规则引擎、地图生成、寻路、AI决策 |
| Godot集成 | C# (BladeHexFrontend) | 场景控制器、View节点、系统管理器、事件总线 |
| UI布局 | GDScript (src/ui/) | 纯视觉构建、控件布局、动画，不含业务逻辑 |
| 渲染辅助 | GDScript (少量) | Shader辅助脚本、渲染器（FogOfWarRenderer等） |

## 法律合规

### 知识产权立场
- 本项目使用通用游戏机制（属性检定、回合制战斗、等级成长等），这些属于**不受版权保护的游戏规则**
- 所有术语使用中文RPG圈通用词汇或原创命名
- 所有世界观、角色、法术描述均为原创
- 不使用、不引用、不暗示任何第三方IP

### 命名风格
- 采用CRPG中文本地化通用风格（参考正义之怒/奇迹时代/英雄无敌）
- 术语自然、直观，不做生造词
- 骰子机制隐藏于引擎后台，玩家面对概率与决策

## 设计哲学
1. **规则即世界** — 战斗规则即世界物理法则，不做"游戏性例外"
2. **涌现而非脚本** — 系统交互产生叙事，而非脚本叙事
3. **惩罚有意义** — 失败是游戏的一部分，角色可以永久死亡
4. **复杂度分层** — 核心循环简单，深度由组合涌现
5. **骰子隐于幕后** — 随机性在后台运作，玩家面对的是概率与决策
