# 职业称号命名集

> **剑与六芒星（Sword & Hex）** 职业名对照表
>
> 角色没有固定职业。星盘上已完成的职业图形节点自然形成当前职业称号；称号不限制技能盘选择，但会决定当前生效的职业专属技能。
> 由已完成职业图形节点（Big / Keystone）的属性组合决定，动态变化。
> 单片瓦片只提供属性，不参与职业判定；仅点亮「启程」、小节点或未完成图形时，称号为 **无名者 / Nameless**。
>
> **本文档以代码为权威源（`BladeHexCore/src/SkillTree/ClassTitleResolver.cs`），并按当前世界观做诠释。**

## 世界观基调（低魔西幻）

- **魔法存在但不泛滥**：魔法是少数受训者掌握的稀有技艺，不存在召唤神迹的"神圣魔法"，也不存在传送门、复活术等高魔奇观。所有施法者依靠的是后天学习的**秘法 / 秘术 / 符文**，而非神授。
- **宗教仅是信仰**：阿瓦隆尼亚的神殿、圣典、骑士誓言只是文化与精神纽带，神不会直接显灵，"圣战者""圣盾使"这类称号代表的是受**信仰驱动**的人，而非神职法师。所谓奇迹，多半是热情、训练或运气的产物。
- **属性即专业**：六大属性对应六种现实中的人类素养，没有"灵性"或"神性"维度。
- **多属性≠强**：技能盘资源有限，每多涉足一个属性，意味着原有路径被稀释。称号代表**广度**，不代表**深度**。

### 属性定位

| 缩写 | 中文 | 战斗与生活含义 |
|---|---|---|
| STR | 力量 | 近战伤害、重甲耐受、力量检定（破门、推搡） |
| DEX | 敏捷 | 远程命中、闪避、先攻、灵巧操作 |
| CON | 体质 | HP、耐力、抗病、长途行军 |
| INT | 智力 | 法术强度与数量、知识、逻辑推断 |
| WIS | 感知 | **暴击与抗暴击**、洞察弱点、侦察、警觉、意志 |
| CHA | 魅力 | 士气、指挥、谈判、领导力 |

> **注**：WIS 不再承载"灵性 / 神圣"这类玄学色彩。「先知」「圣战者」等称号体现的都是普通人的高感知 + 信念，而非神授能力。

### 命名原则

- 中文凝练贴近古风/史诗，英文保持辨识度。
- 避免直接使用"神圣 / Divine / Holy"作为机制描述（保留为信仰隐喻）；保留"圣"字时仅作为人格描述（圣盾使的"圣"是被守护的人尊称的"圣"，不是神授的"圣"）。
- 已存在的代码命名一律保留——本文档随代码更新。

## 命名规则说明

### 判定逻辑

1. **涉足定义**：在某个属性区域完成 ≥1 个职业图形节点（Big / Keystone）= 涉足该属性
2. **主属性标记**：已由完成图形涉足的属性中，职业图形节点数最多的为「主属性」（并列取角色属性值最高的），UI 中排在组合首位
3. **动态变化**：每次完成新的职业图形节点后重新判定；未完成图形和单片瓦片不会改变职业
4. **无涉足**：仅点亮「启程」、小节点或未完成图形时，称号为 **无名者 / Nameless**

### 与游戏系统的关系

- **不锁定技能盘**：职业名不限制可点亮节点，不阻止后续转向其他属性组合
- **专属技能**：当前职业称号对应一个当前生效的专属技能，由 `CareerSkillRegistry` 提供，详见 `职业专属技能.md`
- **属性倾向共存**：属性倾向（生成时由属性分布决定）是先天底色；职业称号是后天选择
- **UI 位置**：角色卡、技能盘界面、战斗单位信息栏
