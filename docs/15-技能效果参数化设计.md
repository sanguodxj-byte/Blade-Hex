# 技能效果参数化设计

> 本文档为技能盘系统中全部 42+ 大节点设计统一的参数化数据结构，使技能效果可由数据驱动、可扩展、可直接转化为 GDScript 代码。

---

## 一、概述与设计原则

### 1.1 目标

将技能盘中所有大节点的效果逻辑，从硬编码转化为**参数化数据描述**。每条技能效果 = 一组 `SkillEffectData` 资源，由战斗引擎统一解析执行。

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| **数据驱动** | 效果逻辑通过参数组合表达，新增技能不需要写新代码 |
| **正交分解** | 每个效果拆解为「触发条件 × 修正项 × 状态附加 × 装备约束」四个正交维度 |
| **骰子表达式** | 所有数值公式使用统一骰子语法 `NdS+M`，由 `DiceParser` 统一解析 |
| **装备联动优先** | 每个效果必须声明装备条件，战斗引擎在执行前先检查装备是否满足 |
| **边界明确** | 每条参数表都包含边界情况处理规则，不留给实现者猜测 |

### 1.3 骰子表达式语法

```
表达式格式: [系数*]NdS[+M] | @属性修正[*倍率] | 纯数字

示例:
  1d8+3        → 掷1个8面骰加3
  2d6          → 掷2个6面骰
  @WIS*3       → WIS修正值×3
  @STR*1.5     → STR修正值×1.5（向下取整）
  1d8+@WIS     → 掷1d8加WIS修正
  @LEVEL       → 当前角色等级
  @PROF        → 当前专精加值
```

## 二、SkillEffectData 数据结构设计

### 2.1 核心枚举定义

```gdscript
## 效果类型
enum EffectType {
    ACTIVE,         ## 主动技能 — 玩家手动触发
    PASSIVE,        ## 被动技能 — 永久生效的修正
    TRIGGERED,      ## 触发型 — 满足触发条件时自动执行

    NONE,           ## 无消耗（被动/光环）
```

```gdscript
## 战斗运行时：激活效果的实例
class_name ActiveEffectInstance
extends RefCounted

var source_effect: SkillEffectData    ## 来源效果数据
var source_unit: Node                 ## 来源单位（光环/触发用）
var remaining_duration: int = -1      ## 剩余回合（-1=永久）
var remaining_cooldown: int = 0       ## 剩余冷却
var uses_this_battle: int = 0         ## 本场战斗已使用次数
var uses_this_round: int = 0          ## 本回合已触发次数

func is_on_cooldown() -> bool:
    return remaining_cooldown > 0

func can_use(battle_context: BattleContext) -> bool:
    if is_on_cooldown(): return false
    if source_effect.uses_per_battle >= 0 and uses_this_battle >= source_effect.uses_per_battle:
        return false
    if trigger_uses_per_round >= 0 and uses_this_round >= trigger_uses_per_round:
        return false
    return true
```

## 三、效果类型分类体系

### 3.1 五大效果类型

| 类型 | 触发方式 | 典型场景 | 实现要点 |
|------|----------|----------|----------|
| **ACTIVE** | 玩家手动选择使用 | 连击、旋风斩、治疗 | 需要目标选择 UI，消耗行动，检查冷却 |
| **PASSIVE** | 永久生效，不需要激活 | 基础剑术+1命中、铁壁减伤 | 在角色属性计算链中作为常驻修正 |
| **TRIGGERED** | 满足条件自动触发 | 反击（被攻击时）、坚韧意志（致命伤害时） | 需要注册事件监听，每回合触发次数限制 |
| **AURA** | 以施法者为中心持续影响 | 统帅光环、君临天下 | 每回合开始扫描范围内单位，进入/离开范围时增减效果 |
| **MODIFIER** | 修改其他系统参数 | 外交手腕改价格、法术穿透降抗性 | 不参与战斗计算，在对应子系统中查询 |

### 3.2 类型与战斗系统的对接方式

```
┌──────────────────────────────────────────────────────────────┐
│                    SkillEffectResolver                        │
│                                                              │
│  resolve(effect: SkillEffectData, context: BattleContext)    │
│                                                              │
│  ┌─ ACTIVE ──→ 1.检查装备条件                                  │
│  │             2.检查冷却/次数                                  │
│  │             3.选择目标（UI交互）                              │
│  │             4.执行伤害/状态/治疗                              │
│  │             5.进入冷却                                      │
│  │                                                           │
│  ├─ PASSIVE ─→ 1.角色初始化时注册到属性计算链                    │
│  │             2.每次计算属性时查询所有PASSIVE效果                │
│  │             3.叠加修正值                                    │
│  │                                                           │
│  ├─ TRIGGERED → 1.注册到事件系统                               │
│  │             2.事件触发时检查条件                              │
│  │             3.执行效果（自动，不需玩家确认）                    │
│  │             4.累加触发次数                                   │
│  │                                                           │
│  ├─ AURA ────→ 1.回合开始时扫描范围                             │
│  │             2.范围内单位获得修正                              │
│  │             3.离开范围的单位失去修正                          │
│  │                                                           │
│  └─ MODIFIER → 1.子系统初始化时查询所有MODIFIER效果             │
│                2.在对应计算中应用修正                            │
└──────────────────────────────────────────────────────────────┘
```

### 3.3 效果执行流程（主动技能）

```
玩家选择技能
    │
    ▼
检查1: 装备条件满足？ ──No──→ 提示"当前装备无法使用"
    │Yes
    ▼
检查2: 冷却完毕？ ──No──→ 提示"冷却中(N回合)"
    │Yes
    ▼
检查3: 使用次数未超？ ──No──→ 提示"本场战斗已用完"
    │Yes
    ▼
检查4: 行动点足够？ ──No──→ 提示"行动点不足"
    │Yes
    ▼
进入目标选择模式
    │
    ▼
确认目标后执行:
    ├── 若有伤害 → 计算命中 → 计算伤害 → 应用伤害
    ├── 若有状态 → 豁免检定 → 应用状态
    ├── 若有治疗 → 计算治疗量 → 恢复HP
    ├── 若有推开 → 移动目标
    └── 若有额外效果 → 执行额外逻辑
    │
    ▼
后处理:
    ├── 进入冷却
    ├── 消耗行动点
    └── 累加使用次数
```

### 3.4 效果执行流程（触发型技能）

```
战斗事件发生（如：被近战攻击命中）
    │
    ▼
事件系统广播 → 所有TRIGGERED效果收到通知
    │
    ▼
每个效果检查:
    ├── 触发条件匹配？（ON_MELEE_HIT）
    ├── 装备条件满足？
    ├── 本回合触发次数未超？
    └── 其他自定义条件？
    │All Yes
    ▼
自动执行效果:
    ├── 反击伤害 = 基础武器伤害 × 50%
    ├── 不需要玩家确认
    └── 累加触发次数
```

### 3.5 装备条件检查规则

```gdscript
func check_equip_conditions(effect: SkillEffectData, unit: Unit) -> bool:
    var weapon: WeaponData = unit.get_current_weapon()
    var armor: ArmorData = unit.get_current_armor()
    var has_shield: bool = unit.has_shield_equipped()

    # 检查需要的装备
    var requires = effect.equip_requires
    if requires & EquipCondition.MELEE_WEAPON and not weapon.is_melee:
        return false
    if requires & EquipCondition.RANGED_WEAPON and not weapon.is_ranged:
        return false
    if requires & EquipCondition.SHIELD and not has_shield:
        return false
    if requires & EquipCondition.STAFF and not weapon.is_staff:
        return false
    if requires & EquipCondition.SPELLBOOK and not unit.has_spellbook():
        return false
    if requires & EquipCondition.LIGHT_WEAPON and not weapon.is_light:
        return false

    # 检查禁止的装备
    var forbids = effect.equip_forbids
    if forbids & EquipCondition.NO_SHIELD and has_shield:
        return false
    if forbids & EquipCondition.NO_HEAVY_ARMOR and armor.is_heavy:
        return false

    return true
```

### 8.2 效果注册流程

```gdscript
func register_unit_effects(unit: Unit) -> void:
    for node_data: SkillNodeData in unit.skill_disk.get_unlocked_big_nodes():
        for effect: SkillEffectData in node_data.effects:
            match effect.effect_type:
                EffectType.PASSIVE:
                    _register_passive(unit, effect)
                EffectType.TRIGGERED:
                    _register_triggered(unit, effect)
                EffectType.AURA:
                    _register_aura(unit, effect)
                EffectType.MODIFIER:
                    _register_modifier(unit, effect)
                EffectType.ACTIVE:
                    _register_active_skill(unit, effect)

func _register_passive(unit: Unit, effect: SkillEffectData) -> void:
    # 直接将修正加入属性计算链
    unit.stat_calculator.add_modifier(
        effect.modifier_target,
        effect.modifier_value,
        effect.modifier_formula,
        effect.equip_requires,
        effect.equip_forbids,
        effect.source_node_id
    )

func _register_triggered(unit: Unit, effect: SkillEffectData) -> void:
    # 注册到事件总线
    event_bus.subscribe(effect.trigger, func(args: Dictionary):
        if not check_equip_conditions(effect, unit):
            return
        var instance = _get_or_create_instance(unit, effect)
        if not instance.can_use(battle_context):
            return
        _execute_triggered_effect(unit, effect, args)
        instance.uses_this_round += 1
    )
```

### 8.3 属性计算链

```gdscript
## 属性计算链：所有修正按优先级叠加
class StatCalculator:

    var _modifiers: Dictionary = {}  # target -> Array[ModifierEntry]

    func get_value(target: ModifierTarget, base_value: int, context: Dictionary = {}) -> int:
        var result = base_value
        for entry in _modifiers.get(target, []):
            if not _check_equip(entry, context):
                continue
            if not _check_condition(entry, context):
                continue
            if entry.is_multiplier:
                result = int(result * entry.value / 100.0)  # 百分比乘法
            else:
                result += _resolve_formula(entry.formula, context) if entry.formula else entry.value
        return result

    ## 计算顺序：加法修正先叠加，乘法修正最后计算
    func get_final_value(target: ModifierTarget, base: int, context: Dictionary = {}) -> int:
        var add_sum = base
        var mult_factors: Array[float] = [1.0]  # 乘法因子列表

        for entry in _modifiers.get(target, []):
            if not _check_equip(entry, context): continue
            if not _check_condition(entry, context): continue
            if entry.is_multiplier:
                mult_factors.append(entry.multiplier)
            else:
                add_sum += _resolve_formula(entry.formula, context) if entry.formula else entry.value

        # 应用乘法
        var result = float(add_sum)
        for factor in mult_factors:
            result *= factor
        return int(result)
```

### 8.4 主动技能执行流程

```gdscript
func execute_active_skill(unit: Unit, effect: SkillEffectData, targets: Array[Unit]) -> bool:
    # 1. 前置检查
    if not check_equip_conditions(effect, unit):
        _show_message("装备条件不满足")
        return false

    var instance = _get_instance(unit, effect)
    if instance == null or not instance.can_use(battle_context):
        _show_message("冷却中或已用完")
        return false

    # 2. 消耗行动点
    match effect.action_cost:
        ActionCost.MAIN_ACTION:
            if not unit.has_main_action():
                _show_message("主行动不足")
                return false
            unit.consume_main_action()
        ActionCost.MINOR_ACTION:
            if not unit.has_minor_action():
                _show_message("次要行动不足")
                return false
            unit.consume_minor_action()

    # 3. 对每个目标执行效果
    for target in targets:
        # 3a. 伤害
        if effect.has_damage or effect.damage_is_weapon:
            var hit_result = _resolve_attack(unit, target, effect)
            if hit_result.is_hit:
                var damage = _calculate_damage(unit, target, effect, hit_result)
                target.take_damage(damage, effect.damage_type)

        # 3b. 治疗
        if effect.is_heal:
            var heal = _resolve_dice(effect.heal_formula, unit)
            heal = unit.stat_calculator.get_final_value(
                ModifierTarget.HEAL_BONUS, heal, {unit = unit}
            )
            target.heal(heal)

        # 3c. 状态附加
        if effect.apply_status != StatusEffect.POISON or effect.status_duration > 0:
            _apply_status(unit, target, effect)

        # 3d. 推开
        if effect.is_push:
            _push_target(target, effect.push_distance, unit)

        # 3e. 临时HP
        if effect.grants_temp_hp:
            var temp = _resolve_dice(effect.temp_hp_formula, unit)
            target.grant_temp_hp(temp)

        # 3f. 驱散
        if effect.dispels_negative:
            target.dispel_negative_effects()

    # 4. 后处理
    instance.remaining_cooldown = effect.cooldown
    instance.uses_this_battle += 1

    # 5. 触发后续效果（连击第2击等）
    for bonus in effect.bonus_actions:
        _execute_bonus_action(unit, bonus, targets)

    return true
```

### 8.6 回合管理中的效果处理

```gdscript
func on_turn_start(unit: Unit) -> void:
    # 1. 状态效果结算（中毒/燃烧/再生等）
    for status in unit.active_statuses:
        _process_status_tick(unit, status)

    # 2. 触发 ON_TURN_START 事件
    event_bus.emit(TriggerCondition.ON_TURN_START, {unit = unit})

    # 3. 重置每回合触发次数
    for instance in _get_instances_for(unit):
        instance.uses_this_round = 0

    # 4. 重置移动标记（坚守用）
    unit.has_moved_this_turn = false

func on_turn_end(unit: Unit) -> void:
    # 1. 触发 ON_TURN_END 事件（生命之泉等）
    event_bus.emit(TriggerCondition.ON_TURN_END, {unit = unit})

    # 2. 冷却递减
    for instance in _get_instances_for(unit):
        if instance.remaining_cooldown > 0:
            instance.remaining_cooldown -= 1

    # 3. 状态持续回合递减
    for status in unit.active_statuses:
        status.remaining_duration -= 1
        if status.remaining_duration <= 0:
            _remove_status(unit, status)
```

### 9.2 装备条件判定优先级

```
1. 武器类型（近战/远程/法杖/魔导书）→ 由当前持有的主武器决定
2. 盾牌 → 由盾牌槽是否有装备决定
3. 护甲类型（轻/中/重）→ 由身体装备槽决定
4. 双手武器 → 副武器槽不可用时视为双手

判定时机：
- 每次切换武器/装备时 → 立即刷新所有效果
- 每次行动开始前 → 再次检查（防止装备状态变化）
- AOE效果命中时 → 不检查（效果已经发动）
```
