# 32 - 上半身骨骼动画系统设计

## 概述

角色视觉采用"棋子"风格：腰部以下为静态底座（已实现 `BasePedestal`），腰部以上为骨骼驱动的上半身纸娃娃。骨骼动画与现有 6 层装备系统结合，实现换装不换动画的纸娃娃效果。

## 设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 动画方案 | 骨骼动画（程序化 Node3D 骨骼树） | 纸娃娃换装只需替换贴图，动画数据全职业共享 |
| 渲染方式 | 多 Sprite3D + Billboard FixedY | 与现有系统一致，无需 SubViewport 开销 |
| 动画驱动 | AnimationPlayer + .tres 资源 | Godot 原生，编辑器可视化 K 帧 |
| 身体范围 | 腰部以上（头、躯干、双臂、武器） | 腰以下由底座替代 |
| 朝向 | Billboard FixedY（面向摄像机） | 保持 HD-2D 风格 |

## 骨骼结构

### 节点树

```
CharacterRenderNode (Node3D)
 └── BodyRoot (Node3D)
      ├── BasePedestal (Sprite3D)          ← 静态底座，平放地面
      │     旋转: (-90°, 0, 0)
      │     位置: (0, 2, 0)
      │
      └── SkeletonRoot (Node3D)            ← 骨骼动画根节点
           │  位置: (0, BasePedestalTop, 0)  ← 底座顶部
           │
           ├── Bone_Torso (Node3D)          ← 躯干骨骼
           │    │  默认位置: (0, 0, 0)
           │    │  动画轨道: position.y, rotation_degrees.x (前倾/后仰)
           │    │
           │    ├── Sprite_Body (Sprite3D)   ← Body 层贴图
           │    │     Billboard: FixedY
           │    │     纸娃娃: 躯干基础体型
           │    │
           │    ├── Sprite_Costume (Sprite3D) ← Costume 层贴图（护甲外观）
           │    │     Billboard: FixedY
           │    │     SortOffset: -0.15
           │    │
           │    ├── Bone_Head (Node3D)       ← 头部骨骼
           │    │    │  相对位置: (0, TorsoHeight, 0)
           │    │    │  动画轨道: rotation_degrees.z (左右摇摆), position.y (点头)
           │    │    │
           │    │    ├── Sprite_Head (Sprite3D)    ← 头部/发型贴图
           │    │    │     Billboard: FixedY
           │    │    │     SortOffset: -0.25
           │    │    │
           │    │    └── Sprite_Helmet (Sprite3D)  ← 头盔贴图
           │    │          Billboard: FixedY
           │    │          SortOffset: -0.32
           │    │
           │    ├── Bone_ArmL (Node3D)       ← 左臂骨骼
           │    │    │  相对位置: (-ShoulderWidth, ShoulderY, 0)
           │    │    │  动画轨道: rotation_degrees.z (摆臂)
           │    │    │
           │    │    └── Bone_ForearmL (Node3D)  ← 左前臂
           │    │         │  相对位置: (0, -UpperArmLen, 0)
           │    │         │  动画轨道: rotation_degrees.z
           │    │         │
           │    │         └── Sprite_ShieldMount (Sprite3D)  ← 盾牌/副手
           │    │               Billboard: FixedY
           │    │
           │    └── Bone_ArmR (Node3D)       ← 右臂骨骼
           │         │  相对位置: (ShoulderWidth, ShoulderY, 0)
           │         │  动画轨道: rotation_degrees.z (摆臂/挥砍)
           │         │
           │         └── Bone_ForearmR (Node3D)  ← 右前臂
           │              │  相对位置: (0, -UpperArmLen, 0)
           │              │  动画轨道: rotation_degrees.z
           │              │
           │              ├── Sprite_Hands (Sprite3D)  ← 手甲贴图
           │              │     Billboard: FixedY
           │              │     SortOffset: -0.30
           │              │
           │              └── Bone_Weapon (Node3D)  ← 武器挂载点
           │                   │  相对位置: (0, -ForearmLen, 0)
           │                   │  动画轨道: rotation_degrees.z (武器角度)
           │                   │
           │                   └── Sprite_Weapon (Sprite3D)  ← 武器贴图
           │                         Billboard: FixedY
           │                         SortOffset: -0.50
           │
           └── AnimationPlayer             ← 动画播放器
                 动画库: idle, attack_melee, attack_ranged, cast, hit, die, move
```

### 骨骼参数（像素坐标，PixelSize 转换后为世界坐标）

| 参数 | 值 (px) | 说明 |
|------|---------|------|
| BasePedestalTop | 40 | 底座顶部 Y 坐标，骨骼根起点 |
| TorsoHeight | 48 | 躯干高度（肩到腰） |
| ShoulderWidth | 24 | 肩宽（中心到肩关节） |
| ShoulderY | 40 | 肩关节相对躯干底部的 Y |
| UpperArmLen | 20 | 上臂长度 |
| ForearmLen | 18 | 前臂长度 |
| HeadOffset | 52 | 头部中心相对躯干底部的 Y |
| PixelSize | 0.5~1.0 | Sprite3D 像素到世界单位的换算 |

## 动画规格

### 动画列表

| 动画名 | 时长 | 循环 | 用途 |
|--------|------|------|------|
| `idle` | 2.0s | ✓ | 待机呼吸 |
| `attack_melee` | 0.5s | ✗ | 近战攻击 |
| `attack_ranged` | 0.6s | ✗ | 远程攻击（拉弓/投掷） |

## 纸娃娃贴图规格

### 各部位贴图尺寸

| 部位 | 贴图尺寸 | 锚点 | 说明 |
|------|----------|------|------|
| Body（躯干） | 64×64 | 底部中心 | 无腿，从腰到肩 |
| Costume（护甲） | 72×68 | 底部中心 | 覆盖躯干，略大 |
| Head（头部/发型） | 48×48 | 底部中心 | 脸 + 发型 |
| Helmet（头盔） | 56×56 | 底部中心 | 覆盖头部 |
| Hands（手甲） | 32×32 | 中心 | 挂在前臂末端 |
| Weapon（武器） | 32×80 | 握柄位置 | 挂在武器骨骼 |
| Shield（盾牌） | 40×48 | 中心 | 挂在左前臂 |

### 贴图制作要求

1. **透明背景**（alpha = 0）
2. **锚点标记**：贴图内需要明确的关节连接点位置
3. **朝向**：所有贴图默认面向右侧（facing right），左向通过 `FlipH` 实现
4. **风格统一**：与现有 class icon 的双色调墨绘风格一致，或采用新的像素风/手绘风
5. **分辨率**：原始制作 2x（128×128 躯干），运行时缩放

### 体型变体

为支持不同种族/体型，Body 层可有多套基础贴图：

| 体型 ID | 描述 | 适用 |
|---------|------|------|
| `body_standard` | 标准人类体型 | 人类、精灵 |
| `body_heavy` | 宽肩厚胸 | 矮人、兽人 |
| `body_slim` | 纤细 | 精灵法师、刺客 |
| `body_large` | 大型 | Boss、巨人 |

骨骼参数（ShoulderWidth、TorsoHeight 等）随体型变体调整。

### PlayAnimation 接口兼容

```csharp
// 现有调用方式完全不变
unit.PlayAnim("attack");  // → RenderNode.PlayAnimation("attack")

// CharacterRenderNode 内部改为:
public void PlayAnimation(string animName)
{
    if (_isDead && animName != "die") return;
    
    // 映射旧动画名到新骨骼动画
    string resolved = animName switch
    {
        "attack" => IsRangedUnit ? "attack_ranged" : "attack_melee",
        "default" => "idle",
        _ => animName
    };
    
    if (_animPlayer.HasAnimation(resolved))
        _animPlayer.Play(resolved);
    else
        _animPlayer.Play("idle");
}
```

## 动画事件系统

### 关键帧回调

AnimationPlayer 支持在特定帧触发方法调用：

| 动画 | 事件时间 | 回调 | 用途 |
|------|----------|------|------|
| `attack_melee` | 0.3s | `OnHitFrame()` | 触发伤害计算 |
| `attack_ranged` | 0.4s | `OnProjectileLaunch()` | 生成投射物 |
| `cast` | 0.5s | `OnSpellRelease()` | 触发法术效果 |
| `hit` | 0.0s | `OnHitFlash()` | 闪白特效 |
| `die` | 1.0s | `OnDeathComplete()` | 清理单位 |

### 与现有 Tween 动画的关系

现有的 Tween 动画（PlayAttackLunge、PlaySelectBounce 等）作用于 `_bodyRoot` 整体位移，与骨骼动画不冲突：

- **骨骼动画**：驱动各部件的相对旋转/位移（内部姿态）
- **Tween 动画**：驱动 `_bodyRoot` 的世界位置（整体位移）
- 两者叠加 = 角色挥砍的同时身体向前突进

## 制作工作流

### Phase 1：原型验证

1. 在 Godot 编辑器中手动创建 `UpperBodyRig.tscn`
2. 用简单色块作为各部位占位贴图
3. 用 AnimationPlayer 面板 K 出 idle 和 attack_melee
4. 验证骨骼旋转 + Billboard 的视觉效果
5. 确认不会出现 Sprite 层级穿透问题

### Phase 2：代码集成

1. 实现 `UpperBodySkeleton.cs` — 程序化构建骨骼树
2. 重构 `CharacterRenderNode.BuildAllLayers()` — 改为骨骼挂载
3. 加载 Animation .tres 资源到 AnimationPlayer
4. 验证 `PlayAnimation()` 接口兼容性

### Phase 3：贴图制作

1. 确定美术风格（像素风 / 手绘 / AI 生成）
2. 制作通用体型的 Body 贴图（3 种体型）
3. 制作各装备部位贴图（按现有装备数据）
4. 验证换装效果

### Phase 4：动画完善

1. 完成所有 7 个动画的 K 帧
2. 添加动画事件回调
3. 针对不同体型微调骨骼参数
4. 添加动画混合/过渡（AnimationTree，可选）

## 性能考量

| 项目 | 预估 | 说明 |
|------|------|------|
| 每单位节点数 | ~15 个 Node3D/Sprite3D | 比现有 6 个 Sprite3D 多约 9 个骨骼节点 |
| 战场最大单位数 | 20 | 总计 ~300 节点，Godot 轻松处理 |
| AnimationPlayer | 每单位 1 个 | 共享 Animation 资源，内存开销极小 |
| 贴图内存 | 与现有相同 | 换装贴图本身不变，只是挂载方式不同 |
| Draw Call | 每单位 6~8 次 | 与现有相同（每个可见 Sprite3D 一次） |

## 备选方案记录

### 为什么不用 Skeleton2D + SubViewport

- 每单位一个 SubViewport 有 GPU 开销（额外 render pass）
- 20 单位 = 20 个额外 viewport，在低端设备上可能成为瓶颈
- 程序化骨骼节点方案零额外 GPU 开销

### 为什么不用 Godot 的 Skeleton3D

- Skeleton3D 设计用于 3D 模型蒙皮（mesh deformation）
- 我们的需求是刚体部件跟随旋转，不需要蒙皮变形
- 用 Node3D 父子关系天然实现刚体骨骼层级，更简单直接

### 为什么不用帧动画 SpriteFrames

- 50+ 职业 × 多套装备 × 5 动画 × 4 帧 = 海量帧图
- 换装时需要为每件装备每个动画画完整帧序列
- 骨骼方案：换装只需替换一张静态贴图，动画数据完全复用
