# 33 - 运行时骨骼动画编辑器

## 概述

游戏内置的骨骼动画编辑器，允许玩家在运行时自定义角色动画。作为 mod 级功能，玩家可以修改待机、攻击、受击等动画的骨骼姿态和时间轴，保存为 JSON 文件供游戏加载。

## 数据格式

动画保存为 JSON，存储在 `user://custom_animations/` 目录：

```json
{
  "name": "attack_melee",
  "duration": 0.5,
  "loop": false,
  "keyframes": [
    {
      "time": 0.0,
      "bones": {
        "Torso": { "rotation_z": 0, "position_y": 0 },
        "Head": { "rotation_z": 0 },
        "ArmR": { "rotation_z": 0 },
        "ArmL": { "rotation_z": 0 },
        "ForearmR": { "rotation_z": 0 },
        "ForearmL": { "rotation_z": 0 },
        "Weapon": { "rotation_z": 0 }
      }
    },
    {
      "time": 0.15,
      "bones": {
        "Torso": { "rotation_z": -10 },
        "ArmR": { "rotation_z": 45 },
        "Weapon": { "rotation_z": 30 }
      }
    },
    {
      "time": 0.3,
      "bones": {
        "Torso": { "rotation_z": 15 },
        "ArmR": { "rotation_z": -60 },
        "Weapon": { "rotation_z": -45 }
      }
    },
    {
      "time": 0.5,
      "bones": {
        "Torso": { "rotation_z": 0 },
        "ArmR": { "rotation_z": 0 },
        "Weapon": { "rotation_z": 0 }
      }
    }
  ]
}
```

规则：
- 每个关键帧只需记录**有变化的骨骼**（省略的骨骼保持上一帧值）
- 帧间线性插值
- `rotation_z` 为主要编辑轴（2D 骨骼旋转）
- `position_y` 仅 Torso 可编辑（上下浮动）

## 组件拆分

### 1. AnimEditorData — 数据模型

纯数据类，不依赖 Godot 节点。

```
AnimEditorData.cs
├── AnimClip          — 一个完整动画（name, duration, loop, keyframes[]）
├── AnimKeyframe      — 一帧（time, Dictionary<boneName, BonePose>）
├── BonePose          — 单骨骼姿态（rotation_z, position_y?）
├── AnimClipSerializer — JSON 序列化/反序列化
└── AnimClipInterpolator — 给定 time，插值出当前所有骨骼姿态
```

### 2. AnimEditorTimeline — 时间轴 UI

底部时间轴条，显示关键帧菱形标记，可拖动播放头。

```
AnimEditorTimeline.cs (Control)
├── 播放头位置 (0~duration)
├── 关键帧菱形标记（可点击选中）
├── 播放/暂停/逐帧按钮
├── 添加帧/删除帧按钮
└── 信号: TimeChanged(float), KeyframeSelected(int)
```

### 3. AnimEditorBonePanel — 骨骼列表 + 属性面板

左侧面板，列出所有骨骼，选中后显示旋转滑条。

```
AnimEditorBonePanel.cs (Control)
├── 骨骼按钮列表（7个）
├── 选中骨骼的属性编辑
│   ├── HSlider: rotation_z (-90 ~ 90)
│   └── HSlider: position_y (-30 ~ 30)（仅 Torso）
├── 实时修改 → 更新预览
└── 信号: BonePoseChanged(string boneName, BonePose)
```

### 4. AnimEditorPreview — 3D 预览区

复用 UpperBodySkeleton，实时应用当前帧的骨骼姿态。

```
AnimEditorPreview.cs (SubViewportContainer 或直接 Node3D)
├── UpperBodySkeleton 实例
├── ApplyPose(Dictionary<string, BonePose>) — 设置所有骨骼旋转
├── 播放模式: 按 AnimClipInterpolator 驱动
└── 编辑模式: 按当前帧静态显示
```

### 5. AnimEditorScene — 主场景控制器

组装以上组件，处理交互逻辑。

```
AnimEditorScene.cs (Node3D，挂载到 skeleton_preview.tscn)
├── 顶栏: 动画选择下拉、体型选择、保存/加载按钮
├── 持有 AnimClip 当前编辑数据
├── 协调 Timeline ↔ BonePanel ↔ Preview 三者同步
├── 保存: AnimClipSerializer.Save(clip, path)
├── 加载: AnimClipSerializer.Load(path) → 刷新 UI
└── ESC: 返回主菜单
```

## 交互细节

| 操作 | 行为 |
|------|------|
| 点击骨骼列表项 | 选中该骨骼，属性面板显示其当前帧数值 |
| 拖动旋转滑条 | 实时更新预览 + 修改当前帧数据 |
| 点击时间轴空白处 | 播放头跳到该位置 |
| 点击菱形标记 | 选中该关键帧，播放头跳到该帧时间 |
| 点击 [+帧] | 在播放头位置插入新关键帧（复制当前插值姿态） |
| 点击 [−帧] | 删除当前选中的关键帧 |
| 点击 [▶] | 从头播放动画（循环或单次取决于 loop 设置） |
| 点击 [▷\|] | 前进一帧（时间 += 1/30s） |
| 修改 duration | 所有关键帧时间等比缩放 |
| ESC | 若有未保存修改则提示，否则直接返回 |

## 约束

- 每个骨骼只编辑 `rotation_degrees.z`（2D 旋转），Torso 额外可编辑 `position.y`
- 不支持缩放、不支持 X/Y 轴旋转（保持 billboard 2D 风格）
- 插值方式固定为线性（不提供曲线编辑）
- 自定义动画优先级高于内置程序化动画
- 文件格式为人类可读 JSON，方便 mod 社区分享
